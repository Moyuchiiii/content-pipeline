Claude Code固有機能と差別化ポイント

CLAUDE.md
プロジェクトごとの永続指示ファイル。コーディングスタイル・禁止事項・タスク管理ルールを定義。
Cursor等は都度プロンプト入力が必要だが、Claude CodeはCLAUDE.mdで一度設定すれば永続的に反映される。
Hooks
特定のアクションが起きた際に自動実行されるスクリプト。コード保存時のフォーマット・テスト実行など。
完全自動・決定論的に動作。AIの判断を介さない確実なトリガー処理が可能。マルチエージェント自律化の要。
Skills（スキル）
SKILL.mdファイルで定義する再利用可能なAI機能。コンテキストに応じて自動発動。
2026年1月にスラッシュコマンドとSkillsが統合。プロジェクト横断で使い回せる知識ライブラリとして機能。
Sub-agents（サブエージェント）
専門特化したAIエージェントをタスク別に呼び出せる機能。レビュアー・リサーチャー・コーダーなど役割分担。
公式サポートで100+のOSSサブエージェントが公開中。agency-agentsは40,889スターを獲得（2026年3月）。
MCP Server / Client
Model Context Protocolで外部ツールと双方向連携。Jira・Slack・Google Drive・Notionなどに接続。
Claude CodeはMCPサーバーとしてもクライアントとしても動作できる唯一のツール。「agent in agent」パターンが可能。
GitHub Actions統合
@claudeメンションでPRへの自動コード修正・実装が動くCI/CDパイプライン。Anthropic公式機能。
非同期・クラウド実行でローカル環境不要。PRレビューコストを劇的に削減できる企業向けの強力機能。
Claude Code on Web
ブラウザだけで動作するクラウド版。2025年10月公開。各セッションは独立したサンドボックスで動作。
スマホだけでFlutterアプリ開発・リリースした事例あり。PCも開発環境も不要になったインパクトは大きい。
マルチエージェント（Swarms）
複数のClaudeエージェントが並列動作するオーケストレーション機能。2026年1月に発見されたフラグ機能。
Opus（リード・設計）+ Sonnet（ワーカー・実装）の役割分担でコスト50-60%削減が実証済み。
