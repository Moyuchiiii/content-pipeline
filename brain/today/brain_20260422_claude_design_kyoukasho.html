Day1 Claude Design を開いて、最初の 1 枚を生成する

まず、Day1 を読み終わった時点であなたに到達してほしいのは、「claude.ai/design を開いてスライドの叩き台を 30 秒で 1 枚作れる状態」。

所要時間 20 分。ここから 19 分、ついてきてほしい。

claude.ai/design を開く

ブラウザで claude.ai にログインして、URL の末尾を /design にする。

claude.ai は、Claude（Anthropic 社の AI アシスタント）の公式サイト。いつもチャットしてる画面と同じログイン情報で、/design を付けるだけで別画面のデザイン専用モードに切り替わる。

新しいアプリのインストールは不要。

画面が切り替わったら、左にチャット欄、右にキャンバス（生成された成果物が並ぶ大きな白い領域）が出てくる。これが Claude Design のエディタ。

プランの確認

Claude Design は Pro / Max / Team / Enterprise プランで利用可能。

・ Free プラン（無料）だと使えない

・ Pro は月 20 ドル（制限範囲内は追加料金なし）

・ すでに Claude Pro に入っている人は、追加費用ゼロで使える

Free プランの人は、Pro に上げるタイミング。副業で提案資料案件を 1 件でも受ければ、月 20 ドルは初回で回収できる。

架空の AI 副業コンサルのサービス紹介スライド、5 枚構成で作ってほしい。
1 枚目タイトル、2 枚目課題、3 枚目提供価値、4 枚目料金、5 枚目まとめ。

プロンプト 19 — レイアウト比率の調整

スライドの余白が狭すぎる。
上下左右のパディングを 2 倍にして、テキストが息をする感じにして。
全ページ一括で適用して。

プロンプト 20 — PPTX 出力向けの調整

この資料を PPTX でエクスポートするので、以下を調整して。
- フォントはゴシック系の一般的なもの（ヒラギノ角ゴ or メイリオ）に
- 画像は絶対配置でなく、テキストボックスに近い配置で
- アニメーションを想定した要素順に並べて

Day4 日本語 LP の崩れを 6 行 CSS で直す

Day4 のゴールは、「日本語 LP を Claude Design で作ったときに起きる 5 パターンの崩れを、プロンプト末尾に 6 行を足すだけで全部直せる状態」。

所要時間 20 分。

なぜ崩れるのか

Claude Design の生成結果は、デフォルトで英語向けの改行ルールが入ってる。具体的には CSS の以下が有効になってる。

・ word-break: auto-phrase（英語の単語単位で折り返す）

・ text-wrap: pretty（英語のタイポグラフィ最適化）

これが日本語だと全部逆効果になる。

結果として、日本語 LP では以下の 5 パターンの崩れが起きる。

・ 1 文字だけ次の行に追い出される（ぶら下がり）

・ 英数字が途中で改行される（例: Claude Co↵de）

・ ボタンの文字が折り返して見切れる

・ バッジが他要素と重なる

・ 句読点が行頭に来る（禁則処理違反）

でも、CSS を 6 行足すだけで全部直る。

コピペで使える 6 行 CSS

* {
  word-break: keep-all;
  line-break: strict;
  text-wrap: wrap;
  overflow-wrap: anywhere;
}
body { font-family: -apple-system, BlinkMacSystemFont, "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif; }
@supports (word-break: auto-phrase) { * { word-break: auto-phrase; } }

この 6 行が何をしてるかを、1 行ずつ簡単に。

・ word-break: keep-all — 単語の途中で改行しない

・ line-break: strict — 日本語の禁則処理を厳格に適用

・ text-wrap: wrap — 英語最適化の text-wrap: pretty を上書き

・ overflow-wrap: anywhere — どうしても入らない英単語は強制折り返し

・ font-family — 日本語に最適なゴシック指定

・ @supports — 対応ブラウザだけ auto-phrase に戻す（@supports はブラウザ互換のための条件分岐）

プロンプトへの差し込み方

Claude Design のプロンプト末尾に、以下の 1 文を追加する。

日本語 LP なので、以下の CSS を <style> タグ内に必ず入れて。
（上記 6 行をここに貼る）

これだけで、初回の生成から崩れない状態で出てくる。

すでに生成済みの LP を直したい場合は、「この LP のスタイルシートに、以下の CSS を先頭に追加して」とチャットで指示すれば上書きされる。

再生成してみる

Day2 のプロンプト 8（SaaS LP）に、この 6 行を末尾に追加して送ってみてほしい。

生成された LP を Export で .zip ダウンロードして、HTML をブラウザで開く。

崩れゼロで表示されているはず。

LP 制作案件を非エンジニア副業で受けるときの最大のボトルネックが、この日本語崩れだった。

それが 6 行で解決する。

Day5 副業 3 用途別の実戦フロー

Day5 のゴールは、「副業の案件が来たその場で、用途別のフローが頭の中で自動再生される状態」。

所要時間 45 分。

用途 1 提案資料・企画書案件のフロー

CW やランサーズで「企画書 作成」「提案書 テンプレ」「ピッチデック」と検索して応募する想定。

① 要件ヒアリング — 目的・ターゲット・枚数・色味・納期を 5 問でクライアントに確認（事前テンプレ化）

② プロンプト選択 — Day2 のプロンプト 1〜7 から近いものを選び、ヒアリング内容を差し込む

③ Claude Design 投入 — チャット欄に貼って送信・30 秒〜1 分待つ

④ プレビュー確認 — キャンバスで全ページを俯瞰・違和感あるスライドに印

⑤ 修正指示 — 「3 枚目の色を濃紺に」「5 枚目を簡略化」等を追加で投げる

⑥ Canva 送信 or PDF エクスポート — 納品形式に合わせて

⑦ Canva で最終仕上げ — ブランドカラー適用・フォント調整・画像差し替え

⑧ クライアント確認 — プレビュー URL か PDF を共有して方向性を確定

⑨ 微修正 → 納品

①〜⑤ は Claude Design で。⑥〜⑦ は Canva で。⑧〜⑨ はクライアントとのやり取り。

合計 15 分〜20 分。

用途 2 LP 制作案件の叩き台フロー

LP 制作は、完成品の前に「叩き台」を先に出す工程が効く。

① 要件ヒアリング — LP のテーマ・ターゲット・トーン・CTA を事前ヒアリング

② プロンプト選択 — Day2 のプロンプト 8〜12 から近いもの

③ 日本語 CSS を末尾に追加 — Day4 の 6 行を忘れず

④ Claude Design 投入 — 1 分待つ

⑤ スタンドアロン HTML でエクスポート — .zip ダウンロード

⑥ ローカルで HTML を開いて目視確認 — 崩れがないか

⑦ クライアントに「方向性確認用の叩き台です」として HTML プレビュー URL を共有

⑧ 方向性 OK → 本制作へ移る / NG → プロンプト修正して再生成

合計 30 分〜45 分。

本制作じゃなくて「方向性確認用」として出すのがコツ。これで「思ってたのと違う」を先に潰せる。

Day6 つまずきやすい 5 ポイントと回避策

Day6 のゴールは、「1 ヶ月間 Claude Design を副業で使い続けたときに踏みそうな落とし穴を、踏まずに避けられる状態」。

所要時間 20 分。

つまずき 1 日本語が崩れる

Day4 で解決済み。6 行 CSS をプロンプト末尾に入れる。

つまずき 2 カスタムテンプレの厳密な量産には向かない

「このクライアント専用のテンプレに沿って 20 枚量産してほしい」みたいな案件には、Claude Design は向かない。

理由は、自社デザインの雛形に厳密に合わせる機能が、まだ弱いから。多少は指示で寄せられるけど、「完全一致」レベルは出ない。

回避策は、こういう案件は Canva テンプレ + 手作業でやる。Claude Design は別の案件に使う。

向き・不向きを見切るのがプロ。

つまずき 3 Research Preview 段階の生成中断

混雑時は、途中で止まることがある。その場で「続きを生成して」と言えば動くけど、「明日納品」みたいな急ぎ案件で 1 発勝負するのは避けたほうが無難。

回避策は、余裕を持って 2 日前に叩き台を出しておく運用。急ぎ案件を取りに行くときは、他のツールと並行で回す。

つまずき 4 PPTX エクスポート時のレイアウト崩れ

キャンバス上では完璧でも、PPTX 形式でエクスポートすると 1〜2 枚レイアウトが崩れることがある。

回避策は 2 つ。

・ Day2 のプロンプト 20（PPTX 向け調整）を事前に投げておく

・ PPTX エクスポート後に、PowerPoint で 1 枚ずつ目視確認して崩れだけ直す

クライアントが PPTX 指定なら、最後の目視確認だけは手作業で入れる。

つまずき 5 Pro プランの利用制限到達

Claude Pro は月 20 ドル（約 3,000 円）で、制限内なら追加料金なし。でも、副業でガンガン使うと月末に制限に触れることがある。

回避策は 2 つ。

・ 重要な案件は月初に寄せる（使用量リセットが月初）

・ 制限に触れたら Max プラン（上位プラン）への一時アップグレードを検討

副業 1 件の報酬で、プラン代は 1〜3 ヶ月分回収できる。ここはケチらないほうが結果として時給が上がる。

Day7 副業ワークフローに組み込む

Day7 のゴールは、「Claude Design が、自分の副業ワークフローの定常運用に組み込まれている状態」。

所要時間 40 分。

Claude Design 単体で閉じない

Claude Design は強力だけど、単体で完結するツールじゃない。周辺ツールと組み合わせて初めて副業ワークフローとして回る。

自分が使ってる組み合わせは、以下の 3 つ。

・ Claude（ブラウザ版） — 要件ヒアリング・応募文作成・提案文ブラッシュアップ

・ Claude Design — 提案資料・LP・営業資料のデザイン生成

・ Canva — Claude Design からの仕上げ・ブランド調整

Claude Code（ターミナル版）は、より複雑な自動化タスク用。提案資料・LP 案件では不要な場面が多い。

案件タイプ別の使い分け

提案資料・企画書案件

LP 制作案件

営業資料の量産

月次の定常運用

自分の月次運用を具体的に書く。

① 月初 — 今月応募する業種 5〜10 個をリストアップ・営業資料を事前量産

② 週次 — 案件来た瞬間に Day5 のフローで動ける状態を維持

③ 月末 — Pro プラン使用量チェック・翌月の運用見直し

これだけで、副業の時間効率が大幅に変わる。

Claude Design × Canva で仕上げるフロー

Bonus1 のゴールは、「Claude Design で作った叩き台を、Canva に流し込んで納品レベルに仕上げる作業を、詰まりポイントなしで通せる状態」。

所要時間 15 分。

なぜ Canva と組み合わせるのか

Claude Design 単体でも PDF や PPTX を書き出せる。でも、副業案件で「クライアントのブランドカラーに厳密に合わせる」「ロゴを正しい位置に配置する」「フォントを指定のものに差し替える」みたいな最終仕上げには、Canva のほうが手数が少ない。

Canva は無料で使えるブラウザのデザインツール。テンプレと素材が豊富で、手動での微調整が直感的。

だから自分は、叩き台は Claude Design・仕上げは Canva、という役割分担で副業案件を回している。

5 ステップ全手順

① Claude Design のキャンバス右上で Export ボタン → Send to Canva を選ぶ

② Canva のログイン画面が開く。初回はアカウント連携の承認 1 回だけ必要（2 回目以降は自動）

③ Canva にスライド or LP が全部読み込まれる。編集可能な状態でインポートされるので、各要素をクリックで直接編集できる

④ ブランドカラー・フォント・ロゴを指定通りに差し替え（クライアント支給のブランドガイドに従う）

⑤ 右上の共有ボタン → ダウンロード → PDF か PNG を選んで書き出し。納品形式に合わせる

ハマりやすい 2 ポイント

ハマり 1 Canva 側でフォントが勝手に変わる

Claude Design で指定した Hiragino Kaku Gothic が、Canva に取り込まれた瞬間に Noto Sans に変わってしまうことがある。これは Canva 側のデフォルトフォントが優先されるため。

対処は、Canva 側で全ページ一括選択 → フォント指定で Noto Sans JP に統一するか、逆にクライアントが OK な手持ちのフォントに揃える。

ハマり 2 画像の位置が微妙にズレる

Claude Design 側で配置した画像の絶対座標が、Canva 側でグリッドに吸着される結果、数 px ズレることがある。

対処は、Canva 側で 1 ページずつ目視確認して、違和感あるページだけ手動で位置調整する。全ページで発生することは稀で、1〜2 ページで済むことが多い。

副業で実際にやった手順の記録

自分が最近受けた提案資料案件で、このフローを使った実録。

① クライアントから「ブランドカラーは濃紺 #1a3a5c、フォントはヒラギノ角ゴ」とヒアリング

② Day2 プロンプト 1（サービス紹介ピッチデック）で叩き台生成。30 秒

③ Send to Canva をクリック。初回承認は済ませてあったので即完了

④ Canva でブランドカラー適用・フォント指定・会社ロゴを差し込み。8 分

⑤ PDF 書き出し・メールで納品。2 分

合計 12 分。クライアントからの反応は即 OK。修正依頼ゼロで納品完了した。
