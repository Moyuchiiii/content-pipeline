なぜ日本語だと壊れるのか

自分は最初、Claude Designに「AIコンタクトセンター AizaCXのLPを作って」とだけ指示して、一発でダーク系のカッコいいLPを出してもらった。でも、プレビュー画面を拡大して日本語部分を見たら、5箇所で文字が壊れてた。

壊れた5パターン

① ヒーローコピーの1文字改行

メインコピー「電話応対の9割を、AIが日本語で。」が「電話応対の9割を、AIが日本語」で1行、次の行に「で。」の2文字だけ落ちてた。デザイン的にキツい。

② 英数字の途中改行

「Enterprise」プランで、「Enter」「prise」に単語が真っ二つに切れてた。固有名詞が途中で切れる状態。

③ CTAボタン内の改行

ボタン内テキスト「導入ロードマップを見る」が「導入ロ」「ードマップを見る」に折り返されてた。ボタンの中で改行されると読めない。

④ 料金カードのバッジがプラン名に重なる

「RECOMMENDED」バッジが真ん中のプラン「Business」に被って「Bu[RECOMMENDED]iness」みたいに表示。

⑤ 句読点が行頭に落ちる

本文で「、」「。」だけが行の先頭に落ちる箇所が複数。日本語の組版ルール（禁則処理）が効いてない状態。

原因

調べたら、Claude Designの内部システムが text-wrap: pretty という英語向けのCSSを自動で入れてた。

text-wrap: prettyは、英語の「最後の行に単語が1個だけ残るのを防ぐ」設計。英語はスペースで単語が切れてるから便利に効く。でも日本語はスペースがないから、文字1個ずつが「単語」扱いされる。結果、「で。」の2文字とか、「Enter」の5文字とかが孤立単語扱いされて、勝手に改行される。

Claude Designの公式仕様を英語で調べたら、システムプロンプトに日本語処理の記述は一切なかった。つまり 日本語のルールは、ユーザー側で追加で教えてあげる必要がある。

そのためのCSSが、次の章の6項目。

6項目の魔法CSS（全文）

これをClaude Designへの指示プロンプトに貼り付けるだけで、日本語の崩れが全部直る。意味がわからなくてもコピペでOK。

/* ===== 1. 日本語全体の禁則処理 ===== */
:lang(ja), .ja {
  word-break: auto-phrase;
  line-break: strict;
  overflow-wrap: break-word;
  hyphens: none;
}

/* ===== 2. BudouX 非対応ブラウザ向けフォールバック ===== */
@supports not (word-break: auto-phrase) {
  :lang(ja), .ja { word-break: keep-all; }
}

/* ===== 3. 見出しの balance 崩れをリセット ===== */
h1, h2, h3, h4, h5, h6 { text-wrap: initial; }

/* ===== 4. BudouX対応ブラウザでのみ balance を復活 ===== */
@supports (word-break: auto-phrase) {
  h1, h2, h3, h4, h5, h6 {
    word-break: auto-phrase;
    text-wrap: balance;
  }
}

/* ===== 5. 本文段落・リスト ===== */
p, li, dd, figcaption {
  text-wrap: pretty;
  word-break: auto-phrase;
  overflow-wrap: break-word;
  hyphens: none;
}

/* ===== 6. ボタン・CTA ===== */
button, .btn, [role="button"], a.cta {
  white-space: nowrap;
  min-width: max-content;
}

/* ===== バッジ・ラベルの重なり防止 ===== */
.card, .pricing-card, [data-has-badge] {
  position: relative;
  padding-top: 36px;
}
.badge, .label, .recommended-badge {
  position: absolute;
  top: 8px;
  right: 8px;
  z-index: 10;
  white-space: nowrap;
}

各項目が何をしてるか

ブロック役割効果
1. 日本語全体文節単位での改行・句読点の禁則処理「で。」が行頭に落ちるのを防ぐ
2. フォールバックFirefox/Safariで auto-phrase が効かない時の保険ブラウザ間の挙動差異を吸収
3. 見出しリセットCJK互換でない balance を一旦止める見出しが1文字改行するのを防ぐ
4. 見出し再適用対応ブラウザだけ綺麗なbalance見出しが両端揃えに
5. 本文最終行の孤立文字を防ぐ「で。」の2文字単独行を防ぐ
6. ボタンボタン内折り返し禁止「導入ロ/ードマップ」を防ぐ
追加: バッジ親要素に余白確保バッジがテキストに被るのを防ぐ

ブラウザサポート

  • word-break: auto-phrase → Chrome 119+（BudouXというGoogle製の機械学習ベースの文節判定）
  • line-break: strict → 全モダンブラウザ
  • text-wrap: pretty → Chrome 117+、Firefox は 2026年4月現在まだ未対応
  • hyphens: none / overflow-wrap: break-word → 全ブラウザ

Firefoxで見たとき auto-phrase は効かないが、ブロック2のフォールバック（keep-all）で「最低限、文字途中では切らない」状態になる。これで実用レベルまで持っていける。

## フォント（必須・省略禁止）
- 日本語: Noto Sans JP (weights: 400, 500, 700)

:lang(ja), .ja {

  line-break: strict;
  overflow-wrap: break-word;
  hyphens: none;
}
@supports not (word-break: auto-phrase) {
  :lang(ja), .ja { word-break: keep-all; }
}
h1, h2, h3, h4, h5, h6 { text-wrap: initial; }
@supports (word-break: auto-phrase) {
  h1, h2, h3, h4, h5, h6 { word-break: auto-phrase; text-wrap: balance; }

p, li { text-wrap: pretty; word-break: auto-phrase; overflow-wrap: break-word; }
button, .btn, a.cta { white-space: nowrap; min-width: max-content; }

実演: AizaCXを30分で作る

自分が作った AizaCX（架空の AIコンタクトセンター SaaS）のLPを、どういう流れで作ったかの実録。

Step 1 Claude Designにアクセス

https://claude.ai/design にブラウザでアクセス。Maxプラン契約済みなら即使える。「New Design」ボタンをクリック。

Step 2 上のプロンプトをコピペして埋める

上のプロンプトの { } 部分をAizaCX用に埋めて、Claude Designのチャット欄に送信。

自分が埋めた例。

  • 対象ユーザー: AI導入を検討してる中小企業の経営者
  • サービス名: AizaCX
  • サービス内容: AIが電話応対の9割を日本語で自動対応するSaaS
  • トーン: 高級感あるダーク系、tealアクセント
  • カラーイメージ: 背景 #0a0e1a、アクセント #14b8a6

Step 3 生成を待つ（約3〜5分）

Claude Designが考えて、1枚ものHTMLを出してくる。生成中は進捗バーが動く。

Step 4 プレビュー確認

画面右にプレビューが出る。デスクトップ表示とモバイル表示を切り替えて確認。

自分のAizaCX初回生成では、6項目CSSを入れたおかげで崩れがゼロ。ヒーローコピー「電話応対の9割を、AIが日本語で。」も綺麗に揃った。

Step 5 修正がある場合

コピー・色・セクション順などを微調整したい時は次のセクション「修正プロンプト集」を使う。

Step 6 エクスポート

画面右上の「Export」メニューから 「Export as standalone HTML」を選ぶ。1ファイルのHTMLがダウンロードされる。

自分のAizaCXは 47KB のHTMLファイル 1個で完成した。

Step 7 デプロイ

ダウンロードしたHTMLをVercelかNetlifyにアップ（後述）。世界中からアクセスできるLPになる。

破綻ゼロチェックリスト（10項目）

Claude Designで作ったLPを公開する前に、以下の10項目を目視でチェックする。

  1. <html lang="ja"> がHTMLタグに付いている
  2. <link> で Noto Sans JP が読み込まれている
  3. <style> 内に6項目CSSが全部含まれている
  4. ヒーローコピーが1文字単独行になっていない
  5. ボタンテキストが途中で折り返していない
  6. 料金カードのバッジがプラン名に重なっていない
  7. 英数字（CRM・SaaS・Enterprise等）が途中で切断されていない
  8. 横スクロールが発生していない（PCの1440px表示で overflow-x なし）
  9. モバイル（375px想定・Chromeの開発者ツールで確認）でも崩れていない
  10. 架空の統計数値（「導入企業数10,000社」等）が入っていない

10番目は自分がハマったやつ。Claude Designはプロンプトで指示しないと、それっぽい架空統計を入れてくることがある。公開前にチェックすること。

デプロイ（Vercel・Netlifyで無料公開）

完成したHTMLを世界に公開する方法。どっちも無料で、5分で終わる。

Vercelの場合

  1. https://vercel.com/signup でアカウント作成（GitHub連携推奨）
  2. 「Add New...」→「Project」
  3. 「Deploy without Git」または、GitHubにリポジトリを作ってそこにHTMLをpush
  4. デプロイ完了→ https://{プロジェクト名}.vercel.app のURLが発行される
  5. 独自ドメインを繋ぎたければ「Domains」メニューから設定

Netlifyの場合（より簡単）

  1. https://netlify.com/signup でアカウント作成
  2. ダッシュボードの「Sites」ページに HTMLファイルをドラッグ&ドロップ
  3. それだけで公開完了。https://{ランダム文字列}.netlify.app のURLが発行される
  4. サイト名を変更したければ「Site settings」→「Change site name」

副業のポートフォリオに載せる用途なら、Netlifyのドラッグ&ドロップが最速。10秒でURLが取れる。

カスタマイズ

完成HTMLを自分で触りたい時のガイド。

色を変える

<style> 内で #14b8a6（tealアクセント色）を Ctrl+H で一括置換するだけ。自分のブランドカラーに差し替え可。おすすめ無料ツール coolors.co で配色案を作って、メイン色1つ+アクセント1つを抜き出す。

フォントを変える

<head> 内のGoogle Fontsリンクと、body { font-family: ... } の部分を書き換える。日本語フォントの選択肢。

  • Noto Sans JP（標準・万能）
  • Zen Kaku Gothic New（モダン・柔らかめ）
  • Shippori Mincho（明朝・高級感）
  • M PLUS 1p（軽快・技術系）

英数字は Inter（定番）、Space Grotesk（モダン）、DM Sans（読みやすい）から選ぶ。

セクションを追加したい

Claude Designの画面に戻って、「現在のLPに『お客様の声』セクションを追加してください。3枚のカードで、会社名・導入前の課題・導入後の効果を含めてください。日本語禁則処理CSSは維持してください」と追加指示するのが一番楽。

モバイル最適化

6項目CSSに加えて、以下をCSSに追加するとモバイルで破綻しにくい。

@media (max-width: 768px) {
  h1 { font-size: 32px; line-height: 1.4; }
  h2 { font-size: 24px; line-height: 1.4; }
  .container { padding: 0 20px; }
  .pricing-card { margin-bottom: 20px; }
}

よくあるトラブルと解決策

.zip版が7MBで重い

Claude Designのエクスポートには「Export as standalone HTML」と「Download as .zip」がある。.zip版は画像・フォント全部同梱で 7MBくらいになる。納品時は standalone HTML を使うと47KBに収まる。

Firefoxで禁則処理が効かない

word-break: auto-phrase がFirefoxではまだ未対応。6項目CSSのブロック2にある @supports not (...) { word-break: keep-all } がフォールバック。これを抜かなければFirefoxでも「文字途中では切らない」状態が最低限保たれる。

料金カードのバッジがプラン名に重なる

親カードに padding-top: 36px が足りてない時に起きる。6項目CSSの最後のブロック（.pricing-card と .badge）を必ず含めること。

「Inter」フォントを避けるべき理由

Claude Design公式のシステムプロンプトで「Inter・Roboto・Arial・Fraunces・system-fonts は避けるべき」と列挙されてる。特に Inter は日本語グリフが含まれないので OS フォントにフォールバックしてデザインが崩れる。日本語を使うなら Noto Sans JP を必ず指定。

プレースホルダーテキストが混ざる

Claude Designは時々「Lorem ipsum」等の意味のないテキストを入れてくる。プロンプトに 「プレースホルダーテキストは使わず、実際のコピーを入れること」 を明記すると回避できる。

架空の統計数値が入る

「導入企業10,000社」みたいな事実と違う数字を入れてくることがある。プロンプトに 「統計・導入実績・顧客数等の架空の数字は絶対に入れないでください」 を追記して対処する。
