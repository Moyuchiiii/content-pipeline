## なぜ日本語だと壊れるのか

自分は最初、Claude Designに「AIコンタクトセンター AizaCXのLPを作って」とだけ指示して、一発でダーク系のカッコいいLPを出してもらった。でも、プレビュー画面を拡大して日本語部分を見たら、**5箇所で文字が壊れてた**。

### 壊れた5パターン

**① ヒーローコピーの1文字改行**

メインコピー「電話応対の9割を、AIが日本語で。」が「電話応対の9割を、AIが日本語」で1行、次の行に「で。」の2文字だけ落ちてた。デザイン的にキツい。

**② 英数字の途中改行**

「Enterprise」プランで、「Enter」「prise」に単語が真っ二つに切れてた。固有名詞が途中で切れる状態。

**③ CTAボタン内の改行**

ボタン内テキスト「導入ロードマップを見る」が「導入ロ」「ードマップを見る」に折り返されてた。ボタンの中で改行されると読めない。

**④ 料金カードのバッジがプラン名に重なる**

「RECOMMENDED」バッジが真ん中のプラン「Business」に被って「Bu[RECOMMENDED]iness」みたいに表示。

**⑤ 句読点が行頭に落ちる**

本文で「、」「。」だけが行の先頭に落ちる箇所が複数。日本語の組版ルール（禁則処理）が効いてない状態。

### 原因

調べたら、Claude Designの内部システムが **`text-wrap: pretty`** という英語向けのCSSを自動で入れてた。

`text-wrap: pretty`は、英語の「最後の行に単語が1個だけ残るのを防ぐ」設計。英語はスペースで単語が切れてるから便利に効く。でも**日本語はスペースがないから、文字1個ずつが「単語」扱いされる**。結果、「で。」の2文字とか、「Enter」の5文字とかが孤立単語扱いされて、勝手に改行される。

Claude Designの公式仕様を英語で調べたら、システムプロンプトに日本語処理の記述は一切なかった。つまり **日本語のルールは、ユーザー側で追加で教えてあげる必要がある**。

そのためのCSSが、次の章の6項目。

## 6項目の魔法CSS（全文）

これをClaude Designへの指示プロンプトに貼り付けるだけで、日本語の崩れが全部直る。意味がわからなくてもコピペでOK。

```css
/* ===== 1. 日本語全体の禁則処理 ===== */
:lang(ja), .ja {
  word-break: auto-phrase;
  line-break: strict;
  overflow-wrap: break-word;
  hyphens: none;
}

/* ===== 2. BudouX 非対応ブラウザ向けフォールバック ===== */
@supports not (word-break: auto-phrase) {
  :lang(ja), .ja { word-break: keep-all; }
}

/* ===== 3. 見出しの balance 崩れをリセット ===== */
h1, h2, h3, h4, h5, h6 { text-wrap: initial; }

/* ===== 4. BudouX対応ブラウザでのみ balance を復活 ===== */
@supports (word-break: auto-phrase) {
  h1, h2, h3, h4, h5, h6 {
    word-break: auto-phrase;
    text-wrap: balance;
  }
}

/* ===== 5. 本文段落・リスト ===== */
p, li, dd, figcaption {
  text-wrap: pretty;
  word-break: auto-phrase;
  overflow-wrap: break-word;
  hyphens: none;
}

/* ===== 6. ボタン・CTA ===== */
button, .btn, [role="button"], a.cta {
  white-space: nowrap;
  min-width: max-content;
}

/* ===== バッジ・ラベルの重なり防止 ===== */
.card, .pricing-card, [data-has-badge] {
  position: relative;
  padding-top: 36px;
}
.badge, .label, .recommended-badge {
  position: absolute;
  top: 8px;
  right: 8px;
  z-index: 10;
  white-space: nowrap;
}
```

### 各項目が何をしてるか

| ブロック | 役割 | 効果 |
|---|---|---|
| 1. 日本語全体 | 文節単位での改行・句読点の禁則処理 | 「で。」が行頭に落ちるのを防ぐ |
| 2. フォールバック | Firefox/Safariで `auto-phrase` が効かない時の保険 | ブラウザ間の挙動差異を吸収 |
| 3. 見出しリセット | CJK互換でない `balance` を一旦止める | 見出しが1文字改行するのを防ぐ |
| 4. 見出し再適用 | 対応ブラウザだけ綺麗なbalance | 見出しが両端揃えに |
| 5. 本文 | 最終行の孤立文字を防ぐ | 「で。」の2文字単独行を防ぐ |
| 6. ボタン | ボタン内折り返し禁止 | 「導入ロ/ードマップ」を防ぐ |
| 追加: バッジ | 親要素に余白確保 | バッジがテキストに被るのを防ぐ |

### ブラウザサポート

- `word-break: auto-phrase` → Chrome 119+（BudouXというGoogle製の機械学習ベースの文節判定）
- `line-break: strict` → 全モダンブラウザ
- `text-wrap: pretty` → Chrome 117+、Firefox は 2026年4月現在まだ未対応
- `hyphens: none` / `overflow-wrap: break-word` → 全ブラウザ

Firefoxで見たとき `auto-phrase` は効かないが、ブロック2のフォールバック（`keep-all`）で「最低限、文字途中では切らない」状態になる。これで実用レベルまで持っていける。

## デプロイ（Vercel・Netlifyで無料公開）

完成したHTMLを世界に公開する方法。どっちも無料で、5分で終わる。

### Vercelの場合

1. https://vercel.com/signup でアカウント作成（GitHub連携推奨）
2. 「Add New...」→「Project」
3. 「Deploy without Git」または、GitHubにリポジトリを作ってそこにHTMLをpush
4. デプロイ完了→ `https://{プロジェクト名}.vercel.app` のURLが発行される
5. 独自ドメインを繋ぎたければ「Domains」メニューから設定

### Netlifyの場合（より簡単）

1. https://netlify.com/signup でアカウント作成
2. ダッシュボードの「Sites」ページに **HTMLファイルをドラッグ&ドロップ**
3. それだけで公開完了。`https://{ランダム文字列}.netlify.app` のURLが発行される
4. サイト名を変更したければ「Site settings」→「Change site name」

副業のポートフォリオに載せる用途なら、**Netlifyのドラッグ&ドロップが最速**。10秒でURLが取れる。

## カスタマイズ

完成HTMLを自分で触りたい時のガイド。

### 色を変える

`<style>` 内で `#14b8a6`（tealアクセント色）を Ctrl+H で一括置換するだけ。自分のブランドカラーに差し替え可。おすすめ無料ツール: [coolors.co](https://coolors.co/) で配色案を作って、メイン色1つ+アクセント1つを抜き出す。

### フォントを変える

`<head>` 内のGoogle Fontsリンクと、`body { font-family: ... }` の部分を書き換える。日本語フォントの選択肢:

- Noto Sans JP（標準・万能）
- Zen Kaku Gothic New（モダン・柔らかめ）
- Shippori Mincho（明朝・高級感）
- M PLUS 1p（軽快・技術系）

英数字は `Inter`（定番）、`Space Grotesk`（モダン）、`DM Sans`（読みやすい）から選ぶ。

### セクションを追加したい

Claude Designの画面に戻って、「現在のLPに『お客様の声』セクションを追加してください。3枚のカードで、会社名・導入前の課題・導入後の効果を含めてください。日本語禁則処理CSSは維持してください」と追加指示するのが一番楽。

### モバイル最適化

6項目CSSに加えて、以下をCSSに追加するとモバイルで破綻しにくい:

```css
@media (max-width: 768px) {
  h1 { font-size: 32px; line-height: 1.4; }
  h2 { font-size: 24px; line-height: 1.4; }
  .container { padding: 0 20px; }
  .pricing-card { margin-bottom: 20px; }
}
```

## よくあるトラブルと解決策

### .zip版が7MBで重い

Claude Designのエクスポートには「Export as standalone HTML」と「Download as .zip」がある。.zip版は画像・フォント全部同梱で 7MBくらいになる。**納品時は standalone HTML を使う**と47KBに収まる。

### Firefoxで禁則処理が効かない

`word-break: auto-phrase` がFirefoxではまだ未対応。6項目CSSのブロック2にある `@supports not (...) { word-break: keep-all }` がフォールバック。これを抜かなければFirefoxでも「文字途中では切らない」状態が最低限保たれる。

### 料金カードのバッジがプラン名に重なる

親カードに `padding-top: 36px` が足りてない時に起きる。6項目CSSの最後のブロック（`.pricing-card` と `.badge`）を必ず含めること。

### 「Inter」フォントを避けるべき理由

Claude Design公式のシステムプロンプトで「Inter・Roboto・Arial・Fraunces・system-fonts は避けるべき」と列挙されてる。特に Inter は日本語グリフが含まれないので OS フォントにフォールバックしてデザインが崩れる。**日本語を使うなら Noto Sans JP を必ず指定**。

### プレースホルダーテキストが混ざる

Claude Designは時々「Lorem ipsum」等の意味のないテキストを入れてくる。プロンプトに **「プレースホルダーテキストは使わず、実際のコピーを入れること」** を明記すると回避できる。

### 架空の統計数値が入る

「導入企業10,000社」みたいな事実と違う数字を入れてくることがある。プロンプトに **「統計・導入実績・顧客数等の架空の数字は絶対に入れないでください」** を追記して対処する。
