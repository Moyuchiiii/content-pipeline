# Xスレッド草稿 — Remote Control副業実録（content-engine版）

note記事: 大学生がClaude Code Remote Controlを副業で1週間使ったら、移動時間も稼ぎになった話

---

## ツイート1（フック — 型6: Before/After型）

PCの前での待機時間、週4〜5時間あった。

Claude Code Remote Control使ったら週ほぼゼロになった。

1週間副業で使ったリアルな記録↓

---

## ツイート2（仕組み — シンプルに）

Claude Codeって自律的に動くんだけど、ファイル操作のたびに「これ承認していいですか？」って聞いてくる。

Remote Controlはその承認をスマホに転送する機能。

PC側で `claude remote-control` を打つとQRコードが出る。スマホで読むだけ。あとはスマホでOKを押すたびに作業が続く。

---

## ツイート3（体験その1 — 月曜・通学中）

通学中に電車でスマホを開いたら承認が3件来てた。

普段なら帰るまで放置してた処理が、その場で全部進んだ。Wi-Fiが切れた瞬間は焦ったけど自動再接続してくれた。

「移動時間に副業が動く」ってこういうことか、と思った。

---

## ツイート4（体験その2 — 図書館）

水曜、図書館でPCはカバンの中。スマホだけ机に出してた。

Claude Codeがエラーを見つけて「直していいですか？」って来たので承認。5分後に完了の報告。

以前なら「図書館行ったから今日は無理か」で終わってた作業が、その日のうちに動いた。

---

## ツイート5（Before/After数字）

1週間の変化:

PCの前にいた時間: 週10h → 3h
承認待ちだけの時間: 週4〜5h → ほぼゼロ
移動中の稼働: ほぼゼロ → 2〜3h

PCの前から動けない時間が7時間消えた。

---

## ツイート6（つまずき — 正直に）

注意点も正直に書く。

・ターミナルを閉じるとセッション切れる（2回やらかした）
・スマホにプッシュ通知は来ない、ブラウザを開いてる前提
・ファイル送受信はPC必須

「承認のリモコン」として使うのが正解で、全部外出先でやろうとすると詰まる。

---

## ツイート7（まとめ + 誘導）

Proプランがあれば無料で使える。セットアップは1分。

「PCの前から動けない問題」持ってる副業勢には刺さると思う。

1週間の詳しい体験記はリプライのnoteに書いた↓
