## 5. 各章の骨子（詳細）

### 第 7 章: なぜ日本語だと壊れるのか
**書きたいこと**:
- Claude Design のシステムプロンプトは `text-wrap: pretty` をデフォルト推奨
- `text-wrap: pretty` は英語の orphan 防止設計 → スペース区切り前提 → 日本語はスペースがないので文字単位で orphan 判定 → 1〜2 文字が孤立行に落ちる
- 5 つの破綻パターン（参考: `search/claude-design-prompt-best-practice/2026-04-19/00-summary.md` の 1 章）
  1. ヒーローコピーの 1 文字改行
  2. 英数字混在の途中改行
  3. CTA ボタン内テキストの途中改行
  4. `line-break: normal` のブラウザ間差異
  5. `position: absolute` バッジのテキスト重なり

**ビジュアル**:
- Before（破綻したスクショ 5 枚）/ After（破綻ゼロのスクショ 5 枚）の対比

### 第 8 章: 6 項目の魔法 CSS
**書きたいこと** — 以下のコードブロックを 1 つずつ解説:
```css
:lang(ja), .ja {
  word-break: auto-phrase;
  line-break: strict;
  overflow-wrap: break-word;
  hyphens: none;
}
@supports not (word-break: auto-phrase) {
  :lang(ja), .ja { word-break: keep-all; }
}
h1, h2, h3, h4, h5, h6 { text-wrap: initial; }
@supports (word-break: auto-phrase) {
  h1, h2, h3, h4, h5, h6 { word-break: auto-phrase; text-wrap: balance; }
}
p, li { text-wrap: pretty; word-break: auto-phrase; overflow-wrap: break-word; }
button, .btn, a.cta { white-space: nowrap; min-width: max-content; }
.pricing-card { position: relative; padding-top: 36px; }
```

各プロパティの役割・なぜ必要か・ブラウザサポート（Chrome 119+、他ブラウザはフォールバック）を解説。

### 第 10 章: AizaCX を作る実演
**構成**:
- Step 1: Claude Design にアクセス（Maxプラン契約者は即使える）
- Step 2: 第 9 章のプロンプトテンプレをコピペ

**スクショ**: 30 枚以上（全ステップを画面キャプチャ）
