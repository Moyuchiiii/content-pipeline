# Deep Research Protocol

note記事のリサーチ精度を標準化するプロトコル。`/note-run` の Phase 3 で参照する。
affaan-m/everything-claude-code の Deep Research パターンを content-pipeline 向けに調整。

---

## Step 1: サブクエスチョンへの分解

トピックを **3〜5 つのサブクエスチョン** に分解する。

例①：速報トピック「Claude Code の新機能リリース」
- Q1: 何が新しくなったのか（機能の概要・公式情報）
- Q2: 実際に使っている人の反応・事例は？
- Q3: 副業・フリーランス業務でどう使えるか
- Q4: 注意点・失敗パターンは？
- Q5: note / X での先行記事のトーンと差別化ポイントは？

例②：実録・ノウハウトピック「AIでクラウドワークス案件を効率化する方法」
- Q1: 実際にどの工程をAIで自動化できるか（応募文・納品物・やり取り等）
- Q2: 非エンジニアでも使えるレベルか（難易度・つまずきポイント）
- Q3: note / X での先行記事に何が書かれているか（競合差別化）
- Q4: Before/After で示せる数字はあるか
- Q5: 「自分もできそう」と思わせるために何を伝えるべきか

---

## Step 2: クエリバリエーション生成

各サブクエスチョンに対して **2〜3 バリエーションのクエリ** を作成し、合計 **15〜30 本** の検索を実行する。

### クエリテンプレート

```
# 一次情報（公式・発表元）
"{キーワード}" site:anthropic.com OR site:github.com
"{キーワード}" announcement OR release OR update 2026

# コミュニティ反応
"{キーワード}" site:note.com
"{キーワード}" site:zenn.dev OR site:qiita.com
"{キーワード}" X.com OR Twitter lang:ja

# 実践・事例
"{キーワード}" 使い方 OR 活用 OR 実装 OR やってみた
"{キーワード}" tutorial OR how to OR workflow

# 批判・懸念・制限
"{キーワード}" 問題 OR 注意 OR 失敗 OR 制限 OR limitation
```

---

## Step 3: ソース収集と信頼度タグ付け

収集した情報には必ず以下のタグを付ける:

| タグ | 基準 | 例 |
|---|---|---|
| ✅ 一次データ | 公式発表、自分の実験結果、インタビュー | anthropic.com, github.com/anthropics |
| 🔶 二次データ | 一次情報を引用している信頼できる記事 | 有名テックブログ、Zenn の詳細記事 |
| ⚠️ 参考情報 | 未検証の口コミ、匿名情報、推測を含む内容 | X のツイート、匿名フォーラム |

**NEVER: ⚠️参考情報のみを根拠に事実として記述すること。**

---

## Step 4: 核心 URL の全文取得

検索結果のうち、**最も信頼度が高い 3〜5 URL** を `WebFetch` で全文取得する。

```
優先順位:
1. ✅ 公式ブログ・リリースノート
2. ✅ GitHub 公式リポジトリ
3. 🔶 スキ数・ブックマーク数が多い note / Zenn 記事
4. 🔶 X でバズったツイートのソース元
```

スニペット（検索結果の抜粋）だけで判断しない。必ず全文を読む。

---

## Step 5: クロス検証

**YOU MUST: 記事に数字・統計・機能説明を書く前に、以下を確認すること。**

- [ ] 同じ情報が複数の独立したソースで確認できるか
- [ ] 情報の日付が古くないか（6 ヶ月以上前は要注意）
- [ ] 公式情報と矛盾していないか
- [ ] 「らしい」「と言われている」という表現のものは ⚠️ タグを付けたか

単一ソースのみの主張は記事内で「〇〇によると」と出典を明示し、断定しない。

---

## Step 6: リサーチサマリーの作成

記事執筆前に以下のサマリーをメモする（`drafts/research/` に保存してもよい）:

```
## リサーチサマリー

### 確定事実（✅）
-

### 信頼度高（🔶）
-

### 参考程度（⚠️）
-

### 情報ギャップ（調査できなかった点）
-

### 差別化ポイント（競合記事にない自分だけの視点）
-
```

---

## 適用基準

| 記事タイプ | 適用レベル |
|---|---|
| 速報（sokuho） | Step 1〜3 のみ（スピード優先。✅ 一次情報確認は必須） |
| 実録（jituroku） | Step 1〜4（自分の体験が主なので Step 5 は軽め） |
| ノウハウ（knowhow） | Step 1〜6 すべて（SEO・ロングテール狙いなので精度最優先） |
