# 文体サンプル

既存記事から抽出した文体の特徴。記事生成時はこのトーンに合わせること。

## トーンの特徴

- 読者層: 非エンジニア副業初心者の若者（男性メイン・女性もOK）
- 作り手: Hyui（文系女子大学生）
- **「女性が作ってるが、男性読者も自分ごと化できるトーン」** を目指す。過度にガーリーにしない
- 一人称: 「自分」で統一（「僕」「俺」は使わない）
- 語尾: 「〜なんだよね」「〜かも」「〜だった」「〜じゃん」「〜でしょ」「〜ってやつ」「〜みたいな」等のカジュアルな口語。女子大学生らしさは残すが、LINEノリ全振りにはしない
- 読者への語りかけ: 「〜じゃないですか？」「〜だと思う」「〜って思うじゃん？」
- 感情表現: 「びっくりした」「テンション上がった」「正直めんどくさかった」等はOK、ただし使いすぎない
- 技術説明: 噛み砕いて説明。専門用語は初出時に補足
- 正直さ: 「正直に言うと」「完璧じゃないこともある」
- 具体性: Before/After、数字（30分→5分、毎日2時間）
- NG: 「僕」「俺」「〜だぜ」「〜っす」「〜っしょ」等の男性寄り表現
- NG: 「〜って思ってる人いるよね」「〜しちゃった」等、女性に閉じすぎる表現の連発

## サンプル文

### 導入部の書き方
> 朝起きてスマホをダラダラ見てる時間、意外と長くないですか？自分もずっとそうだった。

> 副業を始めようとしたとき、最初に詰まったのが「何からやればいいかわからない」だった。ネットで調べるとプログラミングとかAIとか出てきて、文系の自分にはどれも遠い話に見えた。

### 技術説明の書き方
> RSSフィードっていうのは、Webサイトが新着記事を配信する仕組み。ニュースサイトが対応してて、プログラムから記事一覧を取るのにちょうどいい。

> Anthropicっていう会社が出してるAIアシスタント。チャット感覚で「この文章を整えて」って頼むと、勝手に作業してくれる。

### まとめ部分の書き方
> やってることはシンプルで、「AIに頼んで、出てきたものを確認する」だけ。でも、これだけで朝の使い方がガラッと変わった。

> 「AIに仕事させる」って聞くとすごいことやってる感あるけど、実際は地味な作業を代わりにやってもらってるだけなんだよね。

### 非エンジニア読者向けの語りかけ方

でも実際やってみたら、コードを書く場面ってほぼなかった。
```

## X 用・書き出しバリエーション（2026-04-23追加・競合実証）

詳細な実例・根拠は `context/x-anti-ai-patterns.md` 参照。日常4本が全部同じ書き出しにならないよう、以下5型から散らす。

### 型A: 体言止めスタート（断定・事実先行）
```
Claude Codeってパチンコみたい。
AIを使いこなせない人に決定的に欠けてるのは「AI活用力」ではなく「マネジメントスキル」である
LLM無職はやめとけ…
```
→ 「〜です」「〜ます」を使わず断定形で切る。強いテーゼを先頭に置いて保存・引用を誘発。

### 型B: 感嘆・驚きスタート
```
えぐい…😳
ちょっと本気で異次元なんだよな、、、
ついに！！Claude Codeで動画編集が全自動になるスキルが出た、、、
```
→ 「、、、」「…」「！！」で感情を増幅。絵文字は補助。リズムを崩して引き込む。

### 型C: 速報ブラケット（公式アカウント引用時）
```
【⚡️速報】Anthropicが最強のAIモデル「Claude Mythos Preview」を発表。
【🔥朗報】Claude CoworkがすべてのLINEプランで一般提供
【速報】Claude Design発表、Opus 4.7が駆動
```
→ 【】は情報種別を瞬時に伝えるラベル。⚡️🔥🚨の絵文字を併用。ブラケット後は「固有名詞+一言評価」。

### 型D: 数字・固有名詞スタート（ファクト先行）
```
AI業界、この1週間でも変化が速すぎる...
AIリサーチ方法10個、1年運用して固まった実運用パイプラインを全部公開しました。
48時間で、提案書3本書いた。
```
→ 数字が先頭に来て即座に具体性を担保。「〜した」の過去完了形で実証感。

### 型E: 中途半端スタート（会話の途中から入る）
```
昨日さ、提案書の下書きを試してて。
で、結論から言うと〜
プロンプトちょっと修正。
```
→ 読者を「途中から聞いた」状態にして引き込む。口語の勢いで伝える。

スレッドや画像詳細をリプに置くときは以下の実例パターンを使う:

詳細↓
便利です↓
こちら👉
こちら👇
```

**「詳しくはリプで」は競合では使われていない。** 上記8パターンから選ぶ。

### タイトル設計ルール

**優先して使うパターン（スキが取れる型）:**
- 「〜してみてわかった、〇〇のこと」（体験実録型。大学生の強み）
- 「実は〜は〜だった」（逆張り意外性型。梶谷で2,042スキ実績あり）
- 「〜したら〜が変わった」（Before/After型）
- 数字付き（「〇つのポイント」「〇日間で〜」）

**使わないパターン（競合と被る・Hyuiに合わない）:**
- パイプ（｜）区切り二部タイトル → FabyΔの型そのまま。被る
- 「〜完全ガイド」「〜超完全ガイド」→ 梶谷と被る
- 「〜が〜を突きつけた」「残酷な〜」等の重厚語彙 → ビジネス層向けで大学生らしくない

### 冒頭設計ルール

**推奨（Hyuiの武器を活かす）:**
```
体験型（最推奨）: 「先週〇〇をやってみたら〇〇分→〇〇分になった。」+ 転換
共感型: 「〜ってやりにくくない？自分もずっとそう思ってた。」+ 転換
数値型（口語版）: 「今月のAI代、2.3万円だった。」+ 短い事実 + 転換
```

**使わない冒頭:**
- 「〇年〇月〇日、〜が発表された。」（FabyΔ型。ニュース系クリエイターの型）
- 「〜な方は少なくありません」（梶谷型。コンサル的で大学生らしくない）

### noteエディタ装飾ルール（競合分析から追加）

**使う:**
- backtick（`〜`）: ファイル名・コマンド・ツール名の強調（`CLAUDE.md`等）
- 「▼」記号: 参考リンク・関連記事の誘導
- 番号付きリスト: HowTo系の手順（h2に番号を振る）
- カギカッコ（「」）: キーワード・概念の強調、引用

**使わない（競合分析で確認済み）:**
- 引用blockquote → 名言・引用は「」括弧内で処理
- コードブロック多用 → 1〜2個に抑える
- 区切り線多用 → 記事がブツ切りに見える

### 有料ライン設計ルール

- **無料ゾーン**: 体験エピソード＋「なぜこれが重要か」の問い
- **有料ゾーン**: 具体的な手順・設定ファイル・プロンプトテンプレート
- **最強の購買動機**: 「Claude Codeに渡すだけで動く」形式のファイル特典（梶谷が実証済み）
- **価格帯**: ¥500〜¥1,500（梶谷の¥1,980〜と差別化。学生向け）
