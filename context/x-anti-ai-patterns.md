## Step 2: AIくさくないパターン分析

### A. 書き出しのバリエーション（AIくさくない1行目）

実データから抽出した、AIくさくない書き出しパターン。

#### A-1. 体言止めスタート（断定・事実）

競合5アカウント以上で共通して使われている最頻出パターン。

```
例1: 「Claude Codeってパチンコみたい。」（チャエン）
例2: 「AIを使いこなせない人に決定的に欠けてるのは「AI活用力」ではなく「マネジメントスキル」である」（梶谷）
例3: 「LLM無職はやめとけ…」（fladdict）
例4: 「AI時代には、「何かを始めること」「何かを続けること」「何かをやりとげること」は、かなり強力なスキルになる。」（fladdict）
例5: 「AI時代の勝負の仕方は、「生成AIで勝負しないことだ」という話をした。」（fladdict）
```

**共通パターン**: 「〜です」「〜ます」を使わない。断定形で切る。読点なしで短く。

#### A-2. 感嘆・驚き スタート

```
例1: 「えぐい…😳」（hikarun）
例2: 「ちょっと本気で異次元なんだよな、、、」（すぐる）
例3: 「ついに！！Claude Codeで動画編集が全自動になるスキルが出た、、、」（すぐる）
例4: 「すごいことになってた。」（すぐる - 文末だがタイトルテキストとして機能）
```

**共通パターン**: 「、、、」「…」の多用。「！！」で感情を増幅。絵文字は感情の補助として使う。

#### A-3. 速報ブラケット スタート（チャエン・木内・すぐる共通）

```
例1: 「【⚡️速報】Anthropicが最強のAIモデル〜」（チャエン）
例2: 「【速報】Claude Design発表、Opus 4.7が駆動」（チャエン）
例3: 「【🔥朗報】Claude Coworkが〜」（チャエン）
例4: 「【速報】OpenAIの次世代画像生成AI」（すぐる）
```

**特徴**: 【】は情報の種別を瞬時に伝えるラベル。絵文字は⚡️🔥🚨の3種が多用。ブラケット後は「固有名詞+一言評価」。

#### A-4. 数字・固有名詞スタート（ファクト先行型）

```
例1: 「AI業界、この1週間でも変化が速すぎる...」（すぐる）
例2: 「AIリサーチ方法10個、1年運用して固まった実運用パイプラインを全部公開しました。」（すぐる）
例3: 「まず料金だけ並べてみると、差がかなり明確になる。 ㅤ TRAE Lite: $3/月」（すぐる）
```

**特徴**: 数字が先頭に来て即座に具体性を担保する。「〜した」の過去完了形で実証感を出す。

#### A-5. 中途半端スタート（会話の途中から入る感じ）

```
例1: 「Claude Codeってパチンコみたい。 と知り合いの哲学者が言ってて面白かった。」（チャエン）
例2: 「でも正直、ユーザーにとって大事なのは〜」（すぐる）
例3: 「プロンプトちょっと修正。」（fladdict - 別ツイート）
```

**特徴**: 会話が始まっている感。読者を「途中から聞いた」状態にして引き込む。

#### B-1. スレッド1本目の「続く→」書き方

実データから確認できた実際の書き方：

```
「便利です↓」（木内翔大 - 感想+矢印の最短形）
「こちら👉」（かずなり - 右矢印でリンク誘導）
「こちら👇」（かずなり - 下矢印でスレッド誘導）
```

#### B-2. 「詳しくはリプで」系の本物の書き方

```
「スレッドで解説👇」（チャエン）
「以下、要点を共有します↓」（木内翔大）
「以下スレッド続く」（すぐる）
```

画像・詳細をリプに置く明示的な表現は「スレッドで解説」「以下続く」が主流。「詳しくはリプで」という表現は確認できなかった。


**特徴**: 丸数字（①②③）を使う。箇条書きは「・」より①②③が多い。

### C. 問いかけ型のAIくさくない書き方

#### C-1. 「どっち派？」の代わりに使われている実例

x-run の「あなたはどっち派？」に相当する問いかけを、競合アカウントがどう書いているか。

**すぐる**:
```
「でも正直、ユーザーにとって大事なのは
「どのモデルが最強か」より
「どう使い分けるか」だと思ってて、、」
```
→ 問いを問いとして投げない。「〜だと思ってて」で自分の見方を先に出す。同意を誘う構造。

**fladdict**:
```
「AI時代には、〜は、かなり強力なスキルになる。AI時代において大抵の人間は、なんとなくやって、途中でやめてしまう。」
```
→ 「あなたはどう思う？」ではなく「大抵の人間は」という普遍化で読者を引き込む。

**梶谷**:
```
「AIを使いこなせない人に決定的に欠けてるのは「AI活用力」ではなく「マネジメントスキル」である」
```
→ 一文テーゼ形式。問いを投げず断言することで反論・共感・保存を誘発。

**結論**: 「どっち派？」系の直接質問は競合5アカウントで一切確認されなかった。代わりに：
- 「〜だと思ってて」（自分の見方を先に出す）
- 「大抵の人は〜だ」（普遍化）
- 断言テーゼ（反応を問いの代わりにする）

「正直〜」という前置きは確認例なし。「実際に〜した」「試してみたが〜」が代替として機能している。


なぜ効くか: 直接「あなたはどうでしょう？」と問わない。「〜と思う」で自分の見方を先に出すことで読者の共感・反論を自然に引き出す。

#### パターン10: 読者の内心を言語化型（fladdict方式応用）
**例文**:
```
文系でAIとかわたしには関係ないと思ってた時期がある。

AIは理系・エンジニアが使うもので
わたしがやることじゃないという思い込みがあった。

その思い込みが崩れたのは
「使ったら稼げた」という一点だけだった。

難しいかどうかより先に
稼げるかどうかで判断したら良かった。
```

なぜ効くか: 「あなたはこう思いませんか？」と問わずに、過去の自分として読者の思い込みを代弁する。問いかけより強く刺さる。

## 2026-04-24 追記: hyui には「メディア型」の模倣を適用しない

上記 Step 1-4 の実ツイート分析（@masahirochaen・@SuguruKun_ai・@shota7180・@claudecode_lab 等）は **メディア型・CEO型・情報発信者型**のパターン。

hyui は**個人アカウント（文系女子大学生・副業実録・フォロワー数人段階）**のため、これらの型を直接模倣すると「階層ミスマッチ」で AI 臭が出る。

### hyui に合う参考方向性（2026-04-24 等身大調査による）[エージェント調査済]

等身大の大学生・副業実録系アカウントから抽出した方向性:

- **短文OK**（30〜60字の感想も混ぜる）
- **日常ネタ混入**（授業・バイト・コンビニ・天気を副業と同比重で投稿）
- **失敗・迷いの正直な露出**（「今日詰まった」「微妙だった」）
- **読者への質問投げかけ**（「どう思う？」「何日かかった？」）
- **小さな成果を盛らずに書く**（「PV 3。でもスキ1もらえた」）
- **締め言葉は固定しない**（「続ける」「お疲れ様」「また明日」「以上」等を使い分け、締めなしもアリ）
- **タグは週3〜5本程度・日常ネタにはつけない**

### hyui が「やらない」メディア型パターン

- 「〇〇3選」「〇〇10選」形式のまとめツイート
- 「〇〇な人は必見」「〇〇を知らないと損」系の煽り
- 【⚡️速報】【🔥朗報】等の絵文字付きブラケット（絵文字なしの【速報】はたまにOK）
- 成果だけ投稿（プロセス・失敗を隠す）
- 全ツイートにCTA（noteリンク誘導）
