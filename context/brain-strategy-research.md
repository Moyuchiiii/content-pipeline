# Brain 販売戦略大規模リサーチ（2026-04-23）

## Category 1: 100円スタート以外の戦略

### 1. バンドル販売（複数商品セット販売）

Brain プラットフォームにはネイティブのバンドル販売機能は存在しない（2026-04-23 時点）。
ただし以下の疑似バンドル手法が実際に使われている:

- **「特典同梱型」**: メイン商品にテンプレート・GPTs・チェックリストを同梱して実質バンドル化
  - 例: ショウノ「WEBコンテンツ作り方完全マニュアル」にコンテンツ設計テンプレート・添削チケット・マインドマップ活用法を同梱 → 3日で2,000万円売上・1,200部以上販売
- **「note×Brain 組み合わせ誘導」**: note 無料記事 → note 有料記事¥500 → Brain 完全版 という3段階バンドル的導線を設計する手法
- **実装方法**: Brain 商品説明の冒頭に「この商品には〇〇・△△・□□が含まれます」と箇条書きで同梱物を明示。購入者の「これで価格相応」判断を促進

**効果**: ショウノ事例では「テンプレートやオンラインコンサル権、次回割引券など特典を盛りだくさんにして、『これで100円は安い』と感じさせるコツ」が有効と明記されている。

### 2. シリーズ化（第1弾→第2弾→顧客LTV最大化）

実在する成功事例として確認できた構造:

- **迫佑樹（Brain 代表）**: 「ビジネスモデルマスター講座」「裏シンセカイ」など複数商品を段階的にリリース。3日間限定販売で2024年10月ランキング1位を達成
- **支配人ミル**: YouTube 運営系で複数シリーズを展開（「銀のAI攻略エクスプレス」「金のショート攻略大全」）。2024年10月ランキング複数入賞
- **シリーズ設計の原則**:
  - 第1弾: 実録型・低価格（¥1,980〜¥3,980）でレビューと実績を構築
  - 第2弾: 教科書型・中価格（¥2,980〜¥4,980）で体系化
  - 第3弾以降: ツールキット型 or フラッグシップ（¥7,980〜¥30,000）
  - 各シリーズ間でクロスセルリンク（「第1弾を読んだ方へ」の案内）を設置

**LTV向上効果**: 一般的に既存顧客へのアップセル成功率は新規顧客比 60〜70% 高く、シリーズ化はリピート率向上に最も効果的な施策。

### 3. 限定特典（購入者限定コンテンツ・Discord・ZOOM相談）

Brain には「特典機能」があり、購入者限定で以下の特典を設定できる:

- **テンプレート・チェックリスト同梱**: 最も一般的。成約率向上に直結
- **次回割引券**: 次商品購入時の割引をプレゼント → シリーズ化と相乗効果
- **オンラインコンサル権（ZOOM相談）**: ショウノ事例で確認。高額商品（¥10,000+）に適合
- **Discord 招待**: 購入者コミュニティへの招待。継続エンゲージメントと口コミ促進
- **無料アップデート**: 「購入者は新版も自動で受け取れる」明記 → 初期購入インセンティブ
- **アフィリエイター向け販促資料**: 紹介者専用のバナー・テンプレを提供すると紹介数増加

**注意**: 特典が豪華すぎると価格との整合性が崩れ、「内容薄いのでは」という疑念を生む可能性がある。期待値 < 実際の内容 の構図を維持すること。

### 4. 期間限定セール（値下げ・還元率 UP のタイミング運用）

実在する戦略パターン:

- **100円→定価 ¥3,800 の段階値上げ**（ARATAMEDO 事例）: 「3日間のみ100円、その後¥3,800」と明示 → 緊急性を演出。販売数とランキング上昇を同時達成
- **3日間限定ローンチ価格**: 迫佑樹が2024年10月ランキング1位を獲得した手法。短期集中型で爆発的販売数を確保しランキング入りを狙う
- **価格変更の実装**: Brain の「クイック編集」機能を使えば再審査なしに価格のみ変更可能（通常編集では再審査が必要）
- **還元率 UP セール**: 期間限定で還元率を 40%→50% に引き上げてアフィリエイターを活性化させる手法（実在確認できるが具体事例の詳細は非公開）

**景表法注意**: 「限定部数」を頻繁に変更すると景表法リスク。ロードマップ明示（「第1弾の限定期間は〇〇まで」）が最も安全。

### 5. ランキング入りハック（人気タブに入るための具体テクニック）

Brain ランキングのアルゴリズム解析（daimyouji.com 検証より):

- **影響要因**: ①直近売上数（加重平均）②レビュー評価値③販売ペースの持続性
- **好循環の構造**: 短期売上 → ランキング上昇 → レビュー増加 → ランキング維持
- **具体的テクニック**:
  1. **発売初日〜3日間に売上を集中投下**: SNS での告知タイミングを発売直後に集中させる。ランキング7位→5位上昇で流入数が数倍増した実例あり
  2. **複数カテゴリへの出品**: ARATAMEDO は「デザイン」「YouTube」「ビジネス」「マーケティング」など6部門でランキング1位を獲得。複数カテゴリに登録してランキング上位入りの確率を上げる
  3. **アフィリエイター集中動員**: 発売前にアフィリエイターを募集し、発売初日に一斉告知させる
  4. **レビュー購入後7日以内のリマインド**: 購入者に「レビューをお願いします」の DM or メッセージを7日以内に送る

### 6. リピート購入を促す仕組み（無料アップデート・フォロー特典）

- **無料アップデート明記**: 「購入者は今後の追記分も無料で受け取れます」と明示 → 価格正当化と早期購入インセンティブの両立
- **フォロー施策**: Brain のフォロー機能で購入者がフォロワーになると、新商品リリース時に通知が届く
- **次商品の割引券**: 購入時に次回利用可能なクーポンコードを特典に含める（Brain プラットフォームで実装可能か要確認）
- **購入者コミュニティ（Discord）**: 「購入者限定 Discord 招待」は継続エンゲージメントを生み出す。無料の継続価値提供がリピートにつながる

### 7. リリースタイミング（曜日・時間帯の最適値）

X 投稿の最適タイミング（poco-tech.blog 分析):

- **最適曜日**: 月曜日（エンゲージメント 42% で最高）→ 水曜日（28%）→ 金曜日（20%）
- **最適時間帯**: 平日午前9〜11時（Brain・note 告知リンクはこの時間に集中）
- **週末は鈍い**: 土日はリーチが下がる傾向。Brain 発売自体は月〜水に設定が望ましい
- **Brain の公開時刻指定機能**: 公開日時を指定できる。月曜朝9時に合わせて設定し、直前にXで予告投稿する手順が有効

### 8. 告知チャネルの使い分け

実際の成功クリエイターが使っている構成:

| チャネル | 役割 | タイミング |
|------|------|------|
| X（旧Twitter） | メイン告知・拡散 | 発売3日前から毎日投稿 |
| note 無料記事 | 集客・信頼構築 | 発売1週間前に公開 |
| LINE 公式 / メルマガ | クローズドな見込み客への直接告知 | 発売当日に一斉配信 |
| Brain プロフィールページ | クロスセル・ファン育成 | 常時更新 |

- X は「外部リンクはリプライに置く」ことでリーチ低下を防ぐ
- note 末尾の「Brain で完全版あり」案内が最も自然な導線

## Category 3: タイトル・サムネ最適化

### タイトルの型

Brain の売れている商品タイトルをトップページ・ランキングページから実測分析:

| 型 | 例 | 特徴 |
|------|------|------|
| **数字型** | 「AI×音楽で初月10万円」「月15万稼ぐノウハウ」「83万円売れた設計図」 | 数字が最初に来る or 中心にある |
| **Before/After 型** | 「副業ゼロの文系大学生が2ヶ月で月14万円になった」「手取り14万円からAI活用で脱サラ」 | 変化量・期間・到達点を示す |
| **具体名型** | 「Claude Code 完全マスター講座」「Cursor１ポチ note 量産化」 | ツール名＋動詞で完結 |
| **テーゼ型** | 「明鏡」「9,800円は告知ポスト1本では売れない」 | 逆張り・格言型 |
| **リスト型** | 「実際に購入されているnoteだけを集めた勝利の5000リスト」「Claude Codeの制限で困った時の代替AIエージェント3選」 | 具体数量明示 |

### タイトル文字数

トップページ人気タブ商品の実測:
- 「Claude Code完全マスター講座」= 15文字
- 「Cursor１ポチnote量産化プロジェクト」= 18文字
- 「明鏡」= 2文字
- 「AI×音楽で初月10万円収益化ガイド」= 18文字
- 「初めてのBrainで83万円売れた設計図〜【初心者向け】「書くこと以外」の全手順公開」= 42文字
- 「AI使って秒で作成！ベネフィット×マイクロコピー記事設計術」= 28文字

**分析**: タイトル長は 2〜42文字と幅広い。短い商品（「明鏡」）はブランド力が前提。フォロワーが少ない段階では 20〜35文字で「誰が・何を・どれだけ」を明示する型が安全。

### サムネデザイン傾向

確認できた成功商品のサムネ傾向（brain-strategy.md 既存分析と補完）:

- **背景**: ダーク系グラデーション（黒・濃紺・ダーク紫）が圧倒的多数。脳裏に焼き付く視認性
- **文字**: 白 or ゴールドの太字。数字を最大フォントサイズで打ち出す
- **フォント**: 太ゴシック体が主流。手書き風は少数派
- **顔写真**: イケハヤ・支配人ミルなど認知度高い人は顔写真入り。認知度低い段階は不要
- **構図3パターン**:
  1. 中央インパクト型（数字 or タイトルをど真ん中に配置）→ 最も多い
  2. 左人物・右テキスト型（顔出し用）
  3. テキスト密集型（テンプレ・リスト系に多い）

### A/Bテスト事例

Brain ではタイトル・説明文・サムネを再審査で変更できる（ただし修正時は販売部数がリセット）。
具体的な A/B テスト数値は公開情報からは取得できなかったが、判明した原則:

- 「2025年最新版」などの日付付与で成約率向上（「鮮度プレミア」と一致）
- 「完全版」「教科書」「ロードマップ」などの権威ワードが信頼感向上に寄与
- タイトルに「初心者でも」を入れると非エンジニア層への訴求が強まる

## Category 4: アフィリエイト還元率戦略

### 還元率別の実態（実データより）

Brain の還元率設定可能範囲: **10〜50%**（2026-04-23 時点）

| 還元率 | 主な用途 | 実例クリエイター |
|------|------|------|
| 50%（全員） | 初動拡散・認知獲得 | わど（¥30,000）・こうめい（¥3,980）・ジョウ（¥500） |
| 50%（購入者限定） | 高単価×質保証 | イケハヤ（¥29,800）・メラメラ（¥2,980） |
| 40%（全員） | バランス型 | ランキング圏外多め |
| 30%（購入者限定） | 成熟期・利益重視 | めあり（¥24,980） |
| 利用不可 | 外部アフィリ不要な商品 | ヨシ（¥44,800 Threads 教科書） |

### 「全員」vs「購入者限定」の使い分け実態

- **「全員」の優位点**: 誰でも紹介 URL を発行できる → 初動の爆発的な拡散
- **「購入者限定」の優位点**: 「読んでいない人が紹介して売れることはほぼない」（じゅんぺいブログ）→ 購入者限定の方がコンバージョン率が高い
- **高単価帯の傾向**: ¥24,980（めあり）と ¥29,800（イケハヤ）はいずれも「購入者限定」。高単価商品はアフィリエイターの信頼性が必要
- **低〜中価格帯の傾向**: ¥2,980〜¥9,800 は「全員 50%」が多数。初動の認知拡大を優先

### 還元率を時期で変える戦略

実在する手法として確認:

1. **初動（発売〜5件まで）**: 50%（全員）で爆発的拡散を狙う
2. **安定期（6〜20件）**: 50%（購入者限定）に切り替え → アフィリエイター質の向上
3. **成熟期（21件〜）**: 40%（購入者限定）→ 利益率向上

**注意**: Brain 公式から「還元率を下げると既存アフィリエイターが告知をやめる」という現象が報告されている。下げるタイミングは売上が安定してから。

### アフィリエイター向け販促資料の提供

成功クリエイターが実施している施策:

- アフィリエイター向けの「テンプレート告知文」を DM で提供
- サムネ画像・Twitter バナーを提供してコンテンツ統一感を保つ
- 「〇〇と一言添えて紹介してください」という推薦文のひな形提供

## Category 5: レビュー獲得手法

### 購入直後のフォローアップ

確認できた有効手法:

1. **購入後7日以内のレビュー依頼**: Brain には購入者への DM 機能はないが、商品本文の最後尾に「もしよろしければ★をつけていただけると励みになります」と記載するのが最も一般的な手法
2. **特典による誘引**: 「レビューを書いてくれた方に追加特典を送ります」という設計（YUKI 事例で確認）
3. **X での「感想」投稿誘導**: 購入者が X に感想を投稿してくれた際に作者がリツイート・返信 → 購入者が「投稿して良かった」と感じるサイクルを作る

### レビュー投稿率を上げる商品設計

- **期待値 < 実際の内容**: 「こんなに詳しいとは思わなかった」という体験がレビュー投稿を促進
- **即実践できる具体性**: 読んだ当日に試せるテンプレや手順書 → 効果を実感した段階でレビュー投稿に至る
- **コンテンツの完結性**: 「このまま使えば結果が出る」完全パッケージ設計が高評価に直結

### レビュー数が購入転換率に与える影響

- 一般的指標: レビュー4.5以上で成約率 30% アップ（複数ソースで言及）
- ランキング維持にもレビュー数が必要（アルゴリズム上の影響）
- 確認できた最多レビュー: ノブ（67件）・イケハヤ（48件）

### 低評価レビューへの対処

- Brain の低評価は公開されるため、低評価がついた場合は内容改善（クイック編集 or 再審査）で対応
- 「返金対応」の明示により低評価リスクを下げる設計が有効
- 「個人の実体験であり同様の成果を保証するものではありません」の免責文が重要

## Category 6: 販売導線・告知戦略

### 外部誘導の必要性

Brain のランキングは「購入数 × レビュー数」でほぼ決まるため、外部からの流入なしには初動が取れない。Brain 内部での発見は既に認知されているクリエイターにしか期待できない。

### X 告知の最適タイミング・頻度・文言

**タイミング・頻度**:
- 発売3日前: ティーザー投稿（「来週月曜、新しいものを出します」）
- 発売前日: 「明日公開します」予告
- 発売当日: 朝9〜11時に本告知 + リプライにリンク設置
- 発売後3日間: 毎日1〜2本のフォロー投稿（購入者感想のリツイート含む）

**文言のポイント**:
- 「〇〇で悩んでいる人へ」から始める（問題提起型）
- 数字を冒頭に出す（「2ヶ月で月14万達成した方法を公開します」）
- 本文にリンクは貼らない（リーチ 30〜90% 低下） → リプライに置く

### note → Brain の流入設計

実証された最短ルート:

1. note 無料記事で実績・ノウハウのダイジェストを公開（信頼構築）
2. note 有料記事¥500 で入門版を販売（マイクロペイメントでの信頼構築）
3. note 有料記事・無料記事の末尾に「Brain で完全版（コード・テンプレ全部入り）を公開しています → URL」を設置
4. Brain 側でも「この記事の note 版もあります → URL」でクロス誘導

**note → Brain 転換のコツ**: 「Brain 版では note に書けなかった実際のコード・プロンプト・テンプレートが全て入っています」という差分の明示が最重要。

### メルマガ・LINE 公式の活用

- イケハヤ事例: メルマガ・LINE でリストを構築し、新商品ロンチ時に一斉配信
- としぶん事例: LINE アカウントで事前に見込み客を温め、発売日当日に一斉告知 → 初月 34,848円
- Brain 販売初期（フォロワー少ない段階）は LINE 公式よりも X のコンスタントな発信の方が現実的

### ロンチ期間の設計（告知開始→発売→クロージング）

確認できたベストプラクティス:

| フェーズ | 期間 | 施策 |

| ティーザー | 発売7日前 | X でチラ見せ投稿（「準備中です」） |
| 告知強化 | 発売3日前〜前日 | 毎日告知。note で関連無料記事を公開 |
| ローンチ | 発売当日（月朝9時） | X 本告知（リプライにリンク）+ note 末尾案内更新 |
