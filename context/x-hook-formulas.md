# X フック公式集

`/x-run` でツイートを生成する際に参照するフック型の定義集。
obra/superpowers-skills の engagement パターンを X 運用向けに整理。

---

## フックとは

ツイートの **1 行目**。これが全て。スクロールを止められなければ読まれない。
フックは「型」を意識して書く。毎回感覚で書かない。

**NEVER: 1 行目をあいさつや自己紹介から始めること。**
**YOU MUST: ツイート生成前に、以下の型から 1 つ宣言してから本文を書くこと。**

---

## フック型 10 種

### 型 1: 数字型
具体的な数字で価値を瞬時に示す。

```
テンプレート: [X時間/X円/X倍] [何が変わったか]
例: 「1日2時間かかってたリサーチが、Claude Code の /note-run で15分になった」
```

**使いどころ:** Before/After が明確な実録・実績ツイート

---

### 型 2: 逆張り型
「普通そう思うよね」という期待を裏切る。

```
テンプレート: [一般論を否定] [実は〇〇だった]
例: 「AIに記事を書かせると読者に伝わるって言うけど、自分の場合は逆だった」
```

**使いどころ:** ノウハウ・実録。バズりやすいが根拠が必要。

---

### 型 3: 問題提起型
読者が抱えている悩みを代弁する。

```
テンプレート: [ターゲットの行動/状況] ってなんか [問題点] じゃない？
例: 「毎回 ChatGPT か Claude か迷いながら作業してる人、それ全部 Claude で解決できるかも」
```

**使いどころ:** ノウハウ系。共感から入る。

---

### 型 4: 驚き・発見型
自分が驚いたことをそのまま伝える。

```
テンプレート: [予想外の発見] を知った / 気づいた
例: 「副業でAI使うと稼げるって言うけど、実際に何をAIにやらせてるのか初めて理解した」
```

**使いどころ:** 速報・新機能紹介。一人称の感情が乗る。

---

### 型 5: リスト予告型
「このスレッドに〇〇が詰まってる」と伝えて読み進めさせる。

```
テンプレート: [テーマ] について [X個] まとめた
例: 「副業で月10万超えるまでにやったこと、5つにまとめた（スレッド）」
```

**使いどころ:** ノウハウスレッドの 1 ツイート目。保存率が上がる。

---

### 型 6: Before/After 型
変化を一行に圧縮する。

```
テンプレート: [以前の状態] → [今の状態]
例: 「Upwork の提案文、毎回 30 分かけて書いてた → 今は Claude で 3 分」
```

**使いどころ:** 実録・実績報告。結果が見えてクリック率が上がる。

---

### 型 7: 限定・希少性型
「これを知ってる人は少ない」という特別感を出す。

```
テンプレート: [知られていない事実] を [どこで発見したか]
例: 「エンジニアじゃなくてもAIで副業できるって、まだ知らない人が多すぎる」
```

**使いどころ:** キュレーション・調査系ツイート。拡散されやすい。

---

### 型 8: 問いかけ型
リプライを促す。エンゲージメント上げ目的。

```
テンプレート: [対象者の状況] って [はい/いいえ] ？
例: 「副業始めたいけど何から手をつければいいかわからない人、まだいる？」
```

**使いどころ:** 週 1〜2 回。リプライ数を増やしてアルゴリズムに乗せる。

---

### 型 9: 実況型
今やってることをそのまま流す。親近感ツイート。

```
テンプレート: [今やってること] + [リアルな感想]
例: 「案件の納品物を Claude Code に丸投げして自分はコーヒー飲んでる。もう戻れない。」
```

**使いどころ:** 週 3〜4 回の実況ツイート。気軽に投稿できる。

---

### 型 10: 対比型
2 つを比べて違いを際立たせる。

```
テンプレート: [A] と [B] の違い、[切り口]
例: 「クラウドワークスとココナラ、副業初心者が最初に使うべきはどっち」
```

**使いどころ:** ノウハウ・初心者向けコンテンツ。「わかりやすい」と保存される。

---

## フック生成フロー（/x-run 内での使い方）

```
1. ツイートのテーマ・ネタを確認する
2. 上記 10 型の中から「このネタに最も合う型」を 1 つ選ぶ
3. 「型: ○○型」と宣言してから本文を書く
4. 必要に応じて 2〜3 バリエーションを出し、最良を選ぶ
```

## フック評価チェックリスト

生成したフックを出力する前に確認:

- [ ] 1 行目だけで「続きを読みたい」と思えるか
- [ ] AI っぽい言い回し（「〜でしょう」「重要です」）が入っていないか
- [ ] 具体性がある（抽象的なワードだけでない）か
- [ ] 型が宣言どおりに機能しているか
