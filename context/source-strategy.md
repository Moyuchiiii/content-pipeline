# source-run スキャン戦略

`/source-run` の Phase 1（広域スキャン）と Phase 2（プラットフォーム適合度判定）の詳細戦略。

---

## スキャン目的の整理

| 系統 | ターゲットプラットフォーム | スキャン対象 |
|---|---|---|
| A: 副業トレンド | note 主軸・X 副軸 | クラウドワークス・AI副業・在宅ワーク系 |
| B: Claude/Anthropic 新機能 | 全プラットフォーム共通 | リリースノート・公式ブログ・公式 X |
| C: Brain 市場 | Brain 主軸 | brain-market.com ランキング・新着 |
| D: X 公式アカウント | X 主軸（引用RT） | AI 公式アカウント 19 個 |
| **E: X 検索・トレンド** | **X 主軸 + note/Brain 副軸** | **公式外を含むホットトピック・GPT-5.5 等のリーク話題も拾う** |

---

## 系統別スキャン頻度の目安

source-run は手動起動だが、以下が目安:

- **毎日推奨**: 系統 D（X 公式・速報性命）+ **系統 E（X 検索・ホット話題はリアルタイム性命）**
- **2〜3 日に 1 回**: 系統 A・B（副業トレンドは日々動かない）
- **週 1 回**: 系統 C（Brain 市場は変化遅い）

## 系統 E の検索キーワード戦略（2026-04-24 追加）

### キーワードの選び方

公式アカウントで取れる発信は系統 D に任せ、**系統 E は「第三者経由でホットになる話題」を主に拾う**。

**毎回実行する固定キーワード:**

| カテゴリ | キーワード | 狙い |
|---|---|---|
| OpenAI 次世代 | `"GPT-5"` `"GPT-5.5"` `"GPT-6"` `"OpenAI 新モデル"` | 公式発表前のリーク・メディア解説 |
| Claude 速報 | `"Claude" 新機能` `"Claude" リリース` `"Claude Code" 速報` | 公式発表後の解説・実検証ツイート |
| Anthropic | `"Anthropic" 発表` `"Anthropic" 新製品` | リリース後の話題拡散 |
| 競合大手 | `"Gemini" 新機能` `"Grok" リリース` `"Llama" 新版` | Claude との比較ネタ |
| メディア生成 | `"Sora"` `"Veo"` `"Nano Banana"` | 動画/画像生成のバズ |
| AI エージェント | `"AIエージェント" バズ` `"computer use"` | エージェント系の流行 |
| 副業文脈 | `"AI 副業" 月10万` `"Claude 副業" 案件` | 副業発信者のバズ |

### 動的に追加するキーワード

トレンドタブ（系統 E-2）で取得した急上昇ワードを、固定キーワードに追加して再検索。

例: トレンドに「Claude Mythos」が出ていたら → 「`"Claude Mythos"`」を E-1 で追加検索。

### バズ判定基準

- **直近 24 時間以内** の投稿のみ抽出
- **いいね 1,000 超** OR **リポスト 200 超**
- 第三者発信（公式アカウント以外）優先

### Xネタ帳投入時のニュアンス自動判定

| 元ツイートの性質 | 書きたいニュアンス（自動付与） |
|---|---|
| 速報・解説系 | `速報所感・副業視点で乗っかる` |
| 個人の成功体験 | `共感・自分も同じ体験` |
| 批判・論争系 | `逆張り or 中立で意見出す` |
| 数字・実績紹介 | `データ追加・自分の実績比較` |
| ツール紹介 | `使ってみた検証・副業視点` |

ユーザーが手動で投入する場合（`/source-run add`）は、ニュアンスをユーザーが明示。

→ 全部一度にやるか、`/source-run x` 等で系統絞って軽量実行

## プラットフォーム適合度判定の詳細

### note 適合度（noteAdapt）— 6 項目から平均算出

| 項目 | 5点 | 3点 | 1点 |
|---|---|---|---|
| 非エンジニア適合度 (HARDFLOOR) | 完全に副業初心者向け | 半々 | エンジニア専用 |
| 市場の大きさ | 数十万人規模 | 数万人 | 数千人以下 |
| 悩みの深さ | 「金払ってでも解決」 | 「無料なら読む」 | 興味本位 |
| 緊急度 | 24時間以内に書くべき | 1週間以内 | いつでも |
| 差別化可能性 | hyui しか書けない | 似た記事あり | 競合多数 |
| 過去実績相性（PDCA） | 過去スキ多めテーマと近い | 中立 | 過去反応薄テーマ |

**ハードフロア**: 非エンジニア適合度が **4 点未満** なら noteAdapt = 0 にして強制不採用。

---

### X 適合度（xAdapt）— 5 項目から平均算出

| 項目 | 5点 | 3点 | 1点 |
|---|---|---|---|
| 短文化可能性 | 100 字以内でフック立つ | 200 字必要 | 説明に長文必要 |
| リアクション性 | 引用RT・リプが活発見込 | 静かな反応 | 反応見込めず |
| バズ可能性 | 数字・驚き・速報含 | 普通 | 地味 |
| 副業視点で語れるか | hyui の体験と接続 | 部分的 | 接続困難 |
| 速報性 | 24時間以内が命 | 鮮度はある | いつでも |

---

### Brain 適合度（brainAdapt）— 5 項目から平均算出

| 項目 | 5点 | 3点 | 1点 |
|---|---|---|---|
| ノウハウ体系化 | テンプレ/プロンプト/チェックリスト化容易 | 部分的 | 体系化困難 |
| 販売単価妥当性 | ¥2,980〜¥4,980 で買う価値 | ¥1,980 相当 | 単価出ない |
| ターゲット層適合 | 副業学習者にドンピシャ | 一部刺さる | ズレる |
| 既存 note 起点 | 既存 note 記事から拡張可 | 部分的 | ゼロから書く |
| 競合差別化 | Brain 内に競合少 | 競合あり差別化可 | 競合飽和 |

## ネタタイプ自動判定（投入時）

### noteネタ帳の `記事タイプ候補` 自動判定

```
- Claude/Anthropic 新機能リリース 48時間以内 → 速報
- 副業/案件/AI活用の体験談ネタ → 実録
- AI 副業の方法・手順・効率化ネタ → ノウハウ
```

### Xネタ帳の `ネタタイプ` 自動判定

```
- 公式アカウント引用候補（系統 D） → 引用RT候補
- ニュース速報（系統 B） → 速報所感
- 副業実績数字 → 数字報告
- AI ツールの Before/After 体験 → Before/After
- 副業の失敗談・愚痴・日常 → 日常
- 読者への質問・相談ネタ → 問いかけ
- ユーザー手動投入（リアクションしたい等） → リアクション
- 上記以外 → フック案 or その他
```

### Brainネタ帳の `商材タイプ` 自動判定

```
- 「○○の手順全部」「使い方マニュアル」 → 実録マニュアル
- 「○○のテンプレ集」 → テンプレ集
- 「プロンプト一覧」 → プロンプト集
- 「○○の方法・ノウハウ」 → ノウハウ集
- 上記以外 → その他
```

## 優先度自動判定

### `優先度` の付け方

```
- 速報性が 24時間以内 + 適合度 4.0 以上 → 高
- 適合度 3.5〜4.0 → 中
- 適合度 3.0〜3.5 → 低
```

### Xネタ帳の `推奨時間帯` 自動判定

```
- 副業の朝活・Before/After → 朝
- AI ニュース速報 → 昼
- 副業実績・数字報告・体験談 → 夜前半
- 問いかけ・1日の振り返り → 夜後半
- 雑談・日常ネタ → 任意
```

### Brainネタ帳の `想定価格帯` 自動判定

```
- 「テンプレ・プロンプト集」（軽い商材） → 1980
- 「ノウハウ集・実録マニュアル」（標準） → 2980
- 「複数テンプレ + プロンプト + 実録の総合パッケージ」（重い） → 4980
```

## スキャン優先順位（時間がない時）

時間制約がある場合の優先順位:

1. **系統 D（X 公式）** — 速報性命・1日遅れると鮮度切れ
2. **系統 B（Claude 新機能）** — 速報記事ネタの源泉
3. **系統 A（副業トレンド）** — 実録・ノウハウ記事の源泉
4. **系統 C（Brain 市場）** — 変化遅い・週 1 回で十分

## クリーンアップロジック

### 30 日経過自動見送り

source-run 起動時、各ネタ帳の `未使用` レコードを `発見日` で走査:
- 30 日経過 → ステータスを `見送り` に更新
- 見送り理由メモに「30 日経過自動見送り（source-run）」追記

### 高優先ネタの催促

`未使用` かつ `優先度: 高` のネタが 7 日以上残っていたら、Phase 5 報告に「⚠️ 高優先ネタ {タイトル} が 7 日未消化」と明記。

## ユーザー手動投入の受付

### 受付トリガー

ユーザーが Claude Code に以下のような自然言語で投げる:

- 「このツイートにリアクションしたい: {URL}」
- 「このネタで note 書きたい: {テーマ}」
- 「Brain 商品のアイデア出てきた: {コンセプト}」

### 自動振り分けロジック

```
URL が x.com を含む → Xネタ帳・リアクション
「note」「記事」を含む → noteネタ帳
「Brain」「商品」「販売」を含む → Brainネタ帳
判定不能 → ユーザーに確認
```

### 投入時のメタデータ

- 発案元: `ユーザー手動`
- ステータス: `未使用`
- 発見日: 投入日
- 優先度: ユーザーが「急ぎ」と言ったら `高`、それ以外 `中`
- メモ: ユーザーの自然言語をそのまま転記
