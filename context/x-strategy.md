# X 運用戦略

## ツイートタイプ別設計

### 1. 速報ツイート（Claude/Anthropic 新機能リリース時）

**目的**: トレンドに乗る・リーチ拡大
**頻度**: Claude 新機能リリース時（不定期）
**フォーマット**:
- 1行目: 「〇〇が出た」「〇〇リリースされた」（事実ファースト）
- 2行目: 「自分的に一番刺さったのは〜」（個人の視点）
- 3行目: 詳しく書いた → **note リンクはリプライに置く**（本文直埋めはリーチが30〜90%落ちる）
**ハッシュタグ**: #ClaudeCode か #Claude のどちらか1つだけ

### 2. 実況ツイート（日常・作業中）

**目的**: 親近感・継続フォロー動機
**頻度**: 週 3〜4 本
**フォーマット**:
- 今やってること + 結果（Before/After）
- 「〇〇してたら〇〇になった」形式
- 1ツイートで完結
- 画像（スクショ・結果画面）を必ず添付する
**ハッシュタグ**: なし or #ClaudeCode のみ

### 3. ノウハウスレッド（テクニック・Tips）

**目的**: 保存・リポスト・フォロワー獲得
**頻度**: 週 1〜2 本
**フォーマット**:
- 1ツイート目: フック（副業の困りごと・Before/After起点にする）
  - 例: 「副業始めたいけど何から手をつければいいかわからない人、多すぎる」
  - 例: 「クラウドワークスで全然受注できてた時期に、やってたことを公開する」
  - 例: 「AIで副業って言うけど、実際どうやって稼いでるの？全部書く」
  - ❌NG例: 「Claude Code でこれやってる人少なすぎる」（ツールファーストになってる）
- 2〜7ツイート目: 具体的な手順・Tips（1ツイート1アイデア）
- 最終ツイート: まとめ + note 誘導 or フォロー訴求
- **各ツイートに自分でリプライして繋げる**（アルゴリズム上最大75点ボーナス）
**ハッシュタグ**: スレッドの最後のみ 1〜2 個

### 4. 実績報告ツイート（スキ数・PV・案件）

**目的**: 信頼構築・実績の見える化
**頻度**: 月 1〜2 本
**フォーマット**:
- 数字 + 一言コメント（「〇〇記事がスキ16になってた」「〇〇円の案件受注した」）
- 自慢ではなく「こうやったらこうなった」報告スタイル
- 実績を見せる画像（数字のスクショ等）を添付
**ハッシュタグ**: なし

### 5. note 誘導ツイート（記事公開時）

**目的**: note→X→note の循環を作る
**頻度**: note 記事公開のたび
**フォーマット**:
- 記事の一番おいしいところを 1〜2 行で
- 「続きは note に書いた」+ **リンクはリプライに** （本文フックのみ）
- フックが先、リンクは後

### 6. 問いかけ型（コメント誘発型） ← New

**目的**: リプライ数を増やしてエンゲージメントを伸ばす（リプライは13.5点）
**頻度**: 週 1〜2 本
**フォーマット**:
```
[問い：「〇〇ってどっち使ってる？」]
[自分の意見と理由3点]
[「あなたはどっち？リプで教えて」]
```
**ポイント**: Claude Code vs 他ツールの比較、使い方の好みなど「答えやすい問い」を設定する

### 7. 図解カード型 ← New

**目的**: ブックマーク率・保存率を高める（保存はアルゴリズム上位シグナル）
**頻度**: 週 1 本
**フォーマット**:
```
[本文テキスト1〜2行]
[画像：比較表・フローチャート・まとめ図解（2〜4枚）]
[「保存推奨」または「詳しくはスレッドで↓」]
```
**ポイント**: Canva で作った図解・コードの解説図・ワークフロー図が効果的。画像でアルゴリズムスコア約2倍。

## アルゴリズム対策（2026年版）

### 絶対やること

| 施策 | 理由 |
|------|------|
| 外部リンクは**本文ではなくリプライに**貼る | 本文直埋めでリーチが30〜90%落ちる |
| 投稿後**30〜60分は張り付いて初動エンゲージを確保** | 最初の10〜20件のエンゲージが閾値。これを超えると一気に拡散される |
| **画像または動画を必ず添付**する | 画像でスコア約2倍、動画で2〜10倍 |
| スレッドは**自分でリプライして繋げる** | アルゴリズム上最大75点のボーナス（最大の単一シグナル） |
| **火〜木の朝9〜11時・夜19〜21時**に投稿を集中させる | エンゲージメントが最も高い時間帯 |
| テーマを**AI副業 × 非エンジニア × 大学生の1〜2ジャンル**に絞る | 一貫性がアルゴリズム評価に直結 |

### やってはいけないこと

| 行動 | ペナルティ |
|------|------------|
| 外部リンクを本文に直埋め | リーチ30〜90%低下 |
| 7日以上投稿しない | マイナス評価 |
| 「いいねして」等の露骨なエンゲージメントバイト | スパム判定 |
| ボット・購入エンゲージメント | アカウント停止リスク |
| ヘイト・通報を誘発するコンテンツ | -1,500点相当のペナルティ |

### 初動エンゲージメント手順（投稿直後30分）

1. 投稿後すぐ **自分でリプライ**してスレッドを繋げる（リンクもここに置く）
2. 過去に絡んでいたアカウントのタイムラインを確認 → 自然なリプライ
3. 投稿内容に関連するアカウントの最新ツイートに引用リポストかリプライ
4. 10件のエンゲージが集まったか確認 → 超えたら引き続き反応する

## Typefully Auto-Retweet 運用ルール（2026-04-28 確定）

### 設定方針

**Default は OFF。投稿時に必要なものだけ個別ON。**

[WebSearch済 Typefully公式 support.typefully.com] Auto-Retweet機能は指定時間後に自動リポストし、24時間後に自動でRT解除する。Default ONにすると日常実況・短文・問いかけ型まで全部リポストされて過剰になる（フォロワーから「同じ投稿何回も見せるの」感・プロフィール画面の整理が乱れる）。告知系のみ個別設定する運用にする。

### Auto-RT 判定基準

**ON にすべき投稿（告知・拡散価値高）:**
- note記事公開告知（無料・有料）
- メイン商品の販促ツイート
- 実績報告（数字・売上・PV・スキ数）
- 渾身のノウハウスレッド・図解カード型
- スレッド投稿の1ツイート目（伸ばしたいフック）

**OFF にすべき投稿（日常・短文）:**
- 日常実況（コーヒー・課題・気づき系）
- 問いかけ型（リプ誘発が目的・1度のタイミングで完結）
- 引用RT（元投稿が拡散の主役）
- バズ宣伝リプ（既にバズに乗る形）
- 短文の感情吐露

### Auto-RT 設定値（採用構成）

告知系のみ ON 時に3段階で発火する:

| タイミング | 目的 |
|---|---|
| 6時間後リポスト | エンゲージメントゴールデンタイム再活用（[Read済 Book.md] らっこ本書推奨と一致） |
| 25時間後リポスト | 違うタイムゾーン・翌日朝のフォロワーへ |
| 7日後10:30リポスト | 古い投稿の再活性 |

[WebSearch済] 各リポストはTypefullyが24時間後に自動解除するため、プロフィール画面が乱れない。

### 運用観察ポイント

- 2週間運用してフォロワー反応を観察
- 「同じ投稿ばかり」感が出たら25時間後RTを削除して2段階に減らす

## note × X 連携

### 基本設計（X→note→X の循環）

```
X 日常発信（実況・問いかけ・ノウハウ）
    ↓
プロフィール固定ポストにベスト note 記事を設置
    ↓
X から note へ流入（無料記事）
    ↓
無料記事末尾に「続き（テンプレ・実践編）は有料記事で」のCTA
    ↓
有料記事売上 + note 内回遊
    ↓
読者が「感想は #moyuchi_cc でシェアして」→ X に戻ってくる
    ↓
新規フォロワー獲得
```

### 具体的な連携パターン

1. **記事要約ツイート → note 誘導**: note 記事の核心を200字に圧縮してX投稿 → リプライに note URL
2. **固定ポストにベスト記事**: プロフィールページに来た人が自然に note へ流れる設計
3. **X 反応が高いネタを note 記事に昇華**: 週次でいいね・保存が多いツイートを確認 → note で長文化
4. **無料→有料への導線**: 無料 note 記事末尾に有料記事への誘導 CTA を設置（売上が2〜3倍変わる）

### 日常Tips : note誘導 = 7 : 3

誘導ばかりだとフォロワーが離れる。日常 Tips・実況で価値を提供しながら、3割ほど note へ誘導する。

## エンゲージメント戦略

- 副業・AI副業界隈のツイートにリプライ（目立つよりも「価値ある返信」を意識）
- 他の副業系・AI副業系アカウントの投稿を引用ポスト（15倍のスコア重み）して自分の視点を加える
- リプライで「それどうやりましたか？」が来たら詳細 note に誘導
- **週1回**、自分の過去ツイートで反応が良かったものをスレッドで深掘り

### ④ いいね返しルール（毎日）

1日 **30〜40件** を目安に、自分のツイートにいいね・リプライしてくれたアカウントをいいね返しする。

- 目的: エンゲージメントの相互強化・フォロワーとの関係構築
- **自動化は規約違反。必ず手動で行う**
- 返すタイミング: 投稿直後の30分 + 寝る前の2回で分けると楽

### ① バズったときの宣伝リプライ（半自動）

バズ検知（いいね100超 or インプレッション1万超）はユーザーが行う。
バズったツイートのリプライ欄に note 記事への誘導文を入れることで、拡散を収益に転換する。

**フロー:**
1. ユーザーがバズを検知 → `/x-run buzz {ツイートURL}` で Claude に文生成を依頼
2. Claude が「自然なリプライ宣伝文」を生成（宣伝感を出さない）
3. ユーザーがリプライを投稿（**投稿はユーザーが行う。Claude は押さない**）

**文体のルール:**
- 「実はこれに関連した記事まとめてます」「詳しく書いたやつがあって」のトーン
- 「ぜひ〜」「参考にしてください」は使わない
- note URL はこのリプライ本文に直接貼ってOK（バズツイートのリプライはアルゴリズムの影響を受けにくい）

## やらないこと

- ハッシュタグ大量使用（スパムに見える）
- 「フォローお願いします」的な直接的な訴求
- 宣伝色が強い文言（「〇〇を使ってみて！」等）
- 他ツール（ChatGPT 等）の悪口
- 嘘の数字・盛った実績
- **外部リンクの本文直埋め**（これだけでリーチが激減する）

## ③ 投稿頻度目標

**毎日最低1本を絶対ノルマとする。**
7日以上投稿しないとアルゴリズムにマイナス評価される。量より質ではなく、**毎日出しながら質を上げていく**スタンスに切り替える。

| フォロワー段階 | 最低ノルマ | 理想目標 |
|---|---|---|
| 〜50人（現在） | **毎日1本** | 毎日2本 |
| 50〜200人 | 毎日1本 | 毎日2〜3本 |
| 200人〜 | 毎日2本 | 毎日3〜5本 |

ネタがない日は実況ツイート or 問いかけ型で最低1本出す。0投稿の日を作らない。

## PDCA

- 週次: エンゲージメント率（いいね/インプレッション）+ ブックマーク数を確認
- 月次: フォロワー増加数・note への流入数・有料記事売上を確認
- `context/x-performance.md` に記録して傾向を分析
- 反応が良かったタイプを翌月の投稿比率に反映
