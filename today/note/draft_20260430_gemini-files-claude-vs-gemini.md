提案資料、まだ Claude にも Gemini にも丸投げしてませんか？

4月29日の朝、Sundar Pichai がさらっと爆弾を投げてきた。

「Gemini で Docs もスライドも PDF も Excel も、全部チャットから直接生成できるようになった」って内容のポスト。

1.3万いいね、109万閲覧。タイムラインに何度も流れてきて、わたしも仕事の手を止めて見にいった。

Claude派のわたしとしては、正直「またか」って思いそうになった。でも今回ばかりは、無視するとたぶん副業の効率が落ちるやつだった。

だから今日、Claude と Gemini の使い分けを書き直した。同じ場所で副業やってる君のためにも、整理して残しておく。

## 4月29日に何が起きたか、3行で

Google 公式（blog.google）の発表ベースで言うと、こう。

> Gemini チャット内で出力できるようになった形式
> Google Docs / Sheets / Slides / .pdf / .docx / .xlsx / .csv / LaTeX / TXT / RTF / Markdown

これが Gemini アプリの全ユーザーにグローバルで開放された。Pro でも無料でも使える。

つまり、Gemini に「企画書まとめて」って言うと、Word ファイルがそのまま落ちてくる。スライドって言えばスライドで降ってくる。

「Claude で本文書いて、Google Docs に手でコピペして体裁整える」っていう、わたしが副業でずっとやってた工程がまるごと消える話なんだよね。

## Claude派なのに、これを無視できなかった理由

正直に言う。わたしは普段の副業案件、ほぼ Claude で書いてる。

提案文も、CW のメッセージも、企画の骨格も、全部 Claude。文章の質と思考の整理力が、自分の中ではダントツ Claude だから。

でも副業案件って、最後は必ず「ファイル」で渡すじゃないですか。

Word の提案資料、Excel の集計シート、PowerPoint のスライド。クライアントが Workspace で動いてたら Google ファイル指定。

そこで毎回、Claude の出力を Google Docs に貼って、見出しスタイル直して、表をエクスポートして……っていう「資料化」の手作業が30分くらい乗ってた。

Gemini のファイル直生成は、ここを丸ごと潰してくる。だから Claude派でも触らないと損だった。

## 副業の現場で、Claude と Gemini を実際にどう使い分けたか

4月29日の夜、CW で受注したばかりの提案資料案件で、両方並走させてみた。

クライアントは中小のEC企業。「商品の販促企画書を A4 3枚分」っていう案件。

結論から言うと、両方使うのが一番早かった。

> Claude が強かったところ
> 企画の骨格を考える。「こういう順番で話すと刺さる」「ここに具体例を入れる」みたいな、文章の構造設計。本文の言葉選びと締めの一文。

> Gemini が強かったところ
> Claude が出した本文を「Google Docs の体裁に直して」って渡したときの仕上がり。見出しスタイル、箇条書き、太字の整理。そのまま納品できる粒度のファイルが落ちてくる。

所要時間で言うと、こうなった。

> Before（全部 Claude → 手で資料化）
> 本文 15分 → 資料化の手作業 30分 → 合計 45分

> After（Claude で本文 → Gemini で資料化）
> 本文 15分 → Gemini にコピペで資料化 5分 → 合計 20分

1案件で25分浮く。月10案件回してたら、まるっと半日返ってくる計算。

これ、副業初心者の君には特に効くはず。「Claude vs Gemini どっちか選ばなきゃ」って迷ってた時間ごと、いらなくなる。

## でも、Claude を解約しなくていい理由

「じゃあ Gemini だけでよくない？」って思った人、わかる。わたしも一瞬そう思った。

でも実際に Gemini だけで本文を書かせたら、文章の質がまだ Claude には届いてなかった。

具体的に言うと、提案資料の「結論を先に出す」「読者が何を得るか冒頭で示す」みたいな、文章設計の精度。Claude のほうが圧倒的に自然な日本語で組んでくる。

逆に Claude にファイル化を任せると、Markdown 出力までで止まる。Word・Excel への変換は人間がやる。

つまり、両者の役割は違う。

> 本文を考える人
> Claude

> ファイルを作る人
> Gemini

副業の納品物って「文章」と「ファイル」の2層でできてる。それぞれに強いツールを並べたほうが、全体の効率が上がる。

## 同じ場所で詰まってる君へ、3つだけ置いていく

1. Claude 解約しなくていい。役割が違うから両方持つほうが速い。

2. Workspace 系の案件（Google Docs / Sheets / Slides 指定）受けてるなら、Gemini を一度触ってみて。資料化の30分が消える体験は、書くより使うほうが早い。

3. 同じプロンプトを Claude と Gemini に並走させる癖をつける。「販促企画書を3枚で」みたいな指示で両方に投げて、本文は Claude、ファイル化は Gemini に分担させる。3分やれば違いが見える。

わたしも今日まで、Claude 一本でやれると思ってた。でも Gemini が「ファイルを直接作る」側に振り切ってきたことで、副業の作業フローを書き直す合図が出た。

4月29日のリリースは、Claude派にとっても朗報だった。って話。
