Claude Designで日本語のLP作ったら、文字がぐちゃぐちゃになりませんでしたか？

先週、副業の次の武器にLP制作を入れようと思って、Claude Designを触ってみた。

Claude DesignはAnthropicが4/17に出した新しいAIツールで、チャットで「LP作って」と言うだけで1枚もののサイトが完成する。自分みたいな非エンジニアでもブラウザ1つでLPが作れるって聞いて、これは副業のポートフォリオに使えると思って即試した。

英語のLPは普通に綺麗に出る。ただ、日本語でLP作らせたら文字が壊れまくって1週間詰んだ。

結論から言うと、原因はClaude Designのデフォルトが英語向けに作られてて、日本語だと勝手に変な改行が入るから。これは6行のCSS（デザインの設定ファイルみたいなやつ）をプロンプトに足すだけで直る。

この記事では、崩壊した5パターン→原因→直し方→完成までの全工程を書く。無料ゾーンで「どこが壊れるか」「なぜ壊れるか」まで、有料ゾーンで「コピペで直る呪文6行」「日本語LP生成プロンプトテンプレ」「完成したAizaCXのスペック」を出す。

何を作ろうとしたか

作ろうとしたのは、AizaCXっていう架空のAIコンタクトセンターのLP。

なんで架空かっていうと、今後CWやココナラで「LP作ります」って出品するときのサンプルが欲しかったから。副業初心者が営業するとき、ポートフォリオが1本もないと発注側は怖くて任せられない。で、実案件が来る前に自分で1本作って「これが作れます」を見せる必要があった。

題材は「電話応対の9割をAIが日本語で対応するSaaS」っていう想定。ダーク系の高級感あるLPにしたかった。こういうの、Figmaとか使ったら数日かかる。Claude Designなら数分で出るって聞いてたからワクワクしてた。

一発目の出力は、ぱっと見かっこよかった

プロンプトに「AIコンタクトセンター AizaCXのLPを作って。ダーク系でteal（青緑）のアクセント」って送ったら、30秒くらいで1枚もののLPが出てきた。

ヒーローセクション、機能紹介、料金プラン、CTAボタン。全部入ってた。色合いも思った通りダーク×tealで、「これ本当にチャットだけで出たの？」ってテンション上がった。

でも、プレビュー画面を拡大して日本語の部分をよく見たら、5箇所で文字が壊れてた。

壊れた5パターン

① ヒーローコピーの1文字改行

ヒーローセクションの大きいコピー「電話応対の9割を、AIが日本語で。」が「電話応対の9割を、AIが日本語」で1行、次の行に「で。」だけポツンと落ちてた。

「で。」の2文字だけで1行。デザイン的にも読みにくいし、ダサい。

② 英数字の途中改行

料金プランに「Enterprise」プランがあったんだけど、画面幅によって「Enter」で1行、「prise」で次の行みたいに単語が真っ二つに切れてた。

固有名詞が途中で切れるの、けっこうキツい。「Claude」が「Cla/ude」になるのと同じ。

③ CTAボタン内の改行

ボタンの中のテキスト「導入ロードマップを見る」が「導入ロ」で改行されて「ードマップを見る」になってた。ボタンの中で改行するなって感じ。

④ 料金カードのバッジがプラン名に重なる

「RECOMMENDED」バッジを付けた真ん中のプランで、バッジがプラン名の「Business」に被って「Bu[RECOMMENDED]iness」みたいな表示になってた。パッと見読めない。

⑤ 「、」「。」が行頭に落ちる

本文の段落で、句読点だけが行の先頭に落ちる箇所が複数あった。これは日本語の組版ルール（禁則処理ってやつ）を無視してる状態。

日本語LP、5箇所ぜんぶ壊れてた

1週間詰まった話

最初は「プロンプトの書き方が悪いのかな」と思って、「日本語の組版ルールを守って」「文字が途中で切れないように」って追加で指示を入れた。

直らなかった。同じ箇所で同じ崩れ方をする。

次に、出てきたHTMLをダウンロードして自分でCSS（デザインの設定ファイル）を書き換えようとした。でも自分はエンジニアじゃないから、CSSのどこを触ればいいか全然わからない。AIに聞いても「font-sizeを調整してください」みたいな返答で、根本的な解決にならなかった。

5日くらいずっと「Claude Designは日本語に対応してないんだ、英語のLPしか作れないのか」って諦めかけてた。

原因にたどり着いた

詰まりきって、Claude Designの仕様を英語で調べまくったら、Claude Designの内部システムが「text-wrap: pretty」っていう英語向けのCSSを自動で使ってることがわかった。

text-wrap: prettyは、英語の「最後の行に単語が1個だけ残るのを防ぐ」設計。英語はスペースで単語が切れてるから、これが便利に効く。

でも日本語はスペースがないから、文字1個ずつが「単語」扱いされてしまう。結果、「で。」の2文字とか、「Enter」の5文字とかが「孤立した単語」と判定されて、勝手に改行が挟まれる。

つまり、Claude Designは日本語の仕様を知らないまま英語のルールで組んでる。日本語のルールを追加で教えてあげれば直る。

で、日本語のルールを教えるためのCSSが、たった6行くらいのブロックだった。

コピペで直る、日本語LPの呪文

まずこれをClaude Designのプロンプトに貼る。CSSの中身はわからなくていい。コピペするだけで日本語が壊れなくなる。

CSSに以下のブロックを必ず含めること。

:lang(ja), .ja {
  word-break: auto-phrase;
  line-break: strict;
  overflow-wrap: break-word;
  hyphens: none;
}
@supports not (word-break: auto-phrase) {
  :lang(ja), .ja { word-break: keep-all; }
}
h1, h2, h3, h4, h5, h6 { text-wrap: initial; }
@supports (word-break: auto-phrase) {
  h1, h2, h3, h4, h5, h6 { word-break: auto-phrase; text-wrap: balance; }
}
p, li { text-wrap: pretty; word-break: auto-phrase; overflow-wrap: break-word; }
button, .btn, a.cta { white-space: nowrap; min-width: max-content; }
.pricing-card { position: relative; padding-top: 36px; }

またHTMLタグには lang="ja" を必ず付けてください。
全ての日本語テキストブロックには class="ja" を付けてください。
Google FontsからNoto Sans JP を読み込んでください。

それぞれのCSSが何をしてるか、1行ずつ説明する。意味がわからなくてもコピペで動くからスキップしてOK。

word-break: auto-phrase
日本語を「文節」単位で改行する。「電話応対の9割」が「電話応対」「の」「9割」で切れるイメージ。Googleが作った機械学習ベースの仕組み
line-break: strict
句読点が行の先頭に落ちないようにする禁則処理
overflow-wrap: break-word
長い英単語（Enterpriseとか）が画面をはみ出ないように、必要な時だけ折り返す
white-space: nowrap (ボタン用)
ボタンの中の文字を絶対に途中で切らない
padding-top: 36px (料金カード用)
バッジがプラン名に被らないように、カードの上に余白を確保する

これを入れるだけで、さっきの5パターンが全部一発で直る。自分で試したら、ヒーローコピーもEnterpriseもボタンもバッジも、ぜんぶ綺麗に揃った。

つまずいた3点

① .zip版は7MBでクライアントに送れない

Claude Designのエクスポートには「Export as standalone HTML」と別に「Download as .zip」がある。.zip版は画像・フォント全部同梱で7MBくらいになる。

これをそのままnoteのサムネやメール添付に使おうとすると重すぎる。standalone HTMLの方を使うと47KBに収まる。自分は最初この違いがわからなくて、7MBをクライアントに送ろうとしてた（送る前に気づいた）。

② ChromeだとOK、Firefoxで崩れる

6行CSSを入れても、Firefoxだけ一部の禁則処理が効かないケースがあった。原因は「word-break: auto-phrase」がFirefoxでまだ未対応だから。

対策は上のCSSに入ってる@supports not (word-break: auto-phrase) { word-break: keep-all; }の部分。これがFirefox向けのフォールバック（代替処理）になる。これを抜くと半分の人に崩れて見える状態になる。省略禁止。

③ 料金プランのバッジでpadding-top忘れると被る

料金カードに「RECOMMENDED」バッジを絶対配置で付けるとき、カード側にpadding-topを36px以上入れないとバッジがプラン名に被る。上の6行CSSの最後の行（.pricing-card）がそれ。

Claude Designは言われないとpadding-topを足さないから、プロンプトに「バッジがプラン名に被らないように」と書いても無視されることがある。CSSに直接書いた方が確実。

副業でどう使うか

① CW・ココナラのポートフォリオにする

AizaCXみたいな架空サービスのLPを3〜5本作ってポートフォリオにまとめる。CWやココナラのプロフィールに載せとけば、LP制作案件の応募で「作れます」を実物で見せられる。

自分はまだLPでは案件取れてない。ただ、プロフィール文だけの応募から「これが作れます」ってサンプルURL添付できる応募に変えるだけで、発注者からの見え方がだいぶ変わる。LP作れる非エンジニアはまだ少ないから、今のうちにポートフォリオ側だけ先に作っておくのが効く。

② 提案資料の叩き台に使う

CWで提案資料案件が来たとき、いきなりCanvaで作るよりClaude Designで1枚ものの叩き台を30分で出す→ Canvaで微調整、の方が速い。

同じ流れで、自己紹介スライドをLP形式で量産してCW応募のときに添付する使い方もある。テキスト応募だけの人より印象に残る。
