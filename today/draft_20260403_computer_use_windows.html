PC操作の自動化、Macユーザーだけのものだと思っていませんか？


3月24日にMac版が出て、Windowsユーザーの自分は10日間ひたすら待ってた。

今日4月3日、ついに来た。ClaudeのComputer Use、Windows版。

「Claudeがパソコンを動かす」っていう機能。ブラウザを開いて、ファイルをクリックして、キーボードを打って—そういうPC上の操作を全部Claudeに任せられるやつ。Mac版が出たとき記事7・11で紹介しながら、内心「早くWindowsにも来てくれ」とずっと思ってた。

今日やっとスタートラインに並べた。

設定がゼロだった

Claude CoworkのWindowsアプリを開いたら、設定画面に「コンピュータ制御を許可する」のトグルがあった。それをオンにするだけ。終わり。

APIキーも、Dockerも、ターミナルも不要。30秒で使える状態になった。Mac版が出た時点では「Windowsはどうせ複雑な設定がいる」と覚悟してたので、拍子抜けした。

CW新着案件チェック

「クラウドワークスを開いて、今日の新着案件のなかからWebアプリUI開発の案件を探してリストにして」と頼んだ。

Claudeがブラウザを立ち上げてCWにアクセスして、カテゴリを絞り込んで、4件をMarkdownリストで返してくれた。Mac版のときと同じ体験がWindowsでも動いた。

自分でやると15〜20分かかる作業が6分で終わった。これだけで「10日間待った甲斐があった」と思った。

Excelレポートへの入力

先月分の納品実績をExcelのフォーマットに入力する作業をやってもらった。ここで気づいたのが、WindowsではネイティブのExcel for Windowsをそのまま操作してくれること。Macだと Numbers やブラウザ版になりがちなところを、.xlsxファイルをそのまま開いて入力してくれた。

30分くらいかかってた入力作業が8分になった。

deny rulesはWindowsパスで書く

Mac版の設定と1か所だけ違うのが、deny rulesのファイルパスの書き方。

MacはUnixパス（/Users/moyuchi/Documents/）だが、WindowsはC:/Users/moyuchi/Documents/になる。settings.jsonに書くときはフォワードスラッシュで統一するのが一番簡単だった。

// settings.json（Windows版 deny rules）
{
  "permissions": {
    "deny": [
      "Bash(rm *)",
      "Bash(del *)",
      "Bash(rd *)",
      "Write(C:/Users/moyuchi/Documents/clients/*)"
    ]
  }
}

クライアント関連のフォルダだけは書き込み禁止にしておくのが安全。delとrdはWindowsのコマンドで、Macにはない削除系コマンド。この2行はWindows版では必ず追加した方がいい。

Auto Modeを使っている場合はここが二重の安全網になる。deny rulesがあればAuto Mode中に意図しないファイル削除が走らない。

Claude CodeのPowerShell対応との組み合わせ

最近のClaude Codeのアップデートで、WindowsでPowerShellが正式にサポートされた。今まではWSLを別途用意しないとBashが動かなかった部分が、PowerShellで代替できるようになった。

自分が今日試したのは、Computer UseとClaude Codeが分業する2段フロー。

Claudeがブラウザで案件データ取得 → Claude CodeがPowerShellで整形 → Excelに出力

Mac版では自分のPCで試せなかったフローを、今日初めてWindowsで完結させられた。

動かなかった2パターン

失敗パターン1
CWのメッセージへの返信。メッセージを読むところまでは動いたが、返信の入力フォームでカーソルが正確に当たらず、1〜2回に1回は失敗した。Mac版の記事11でも書いたが「チャット系アプリの入力フォームは苦手」という傾向は変わっていない。
失敗パターン2
複数モニターをまたぐ操作。副業PCに外部ディスプレイを繋いでいるが、Claudeが認識するのはメインモニターのみ。サブモニターにあるアプリの操作は今のところ無理。

「1つのアプリの中で完結する作業」に向いているのはMac版から変わっていない。
