logout</code> → <code>claude login</code> で再認証。それでも駄目ならネット接続を確認。</p><h3>② Auto Mode が反応しない</h3><p>原因: ターミナルアプリが <code>Shift + Tab</code> をショートカットとして横取りしている。</p><p>解決: ターミナル設定でキーバインディングを確認。iTerm2 / Windows Terminal の設定で重複していないか。</p><h3>③ deny rules が効かない</h3><p>原因: settings.json の構文エラー・ファイル配置場所の間違い。</p><p>解決: ファイルを <code>~/.claude/settings.json</code>（ユーザー全体）か <code>.claude/settings.json</code>（プロジェクトローカル）に正しく置く。JSON 構文を JSONLint で検証。</p><h3>④ コードが提案されるけど実行されない</h3><p>原因: Claude Code が「これを実行していいか」確認待ちになっている。</p><p>解決: 「実行してください」を明示的に追加するか、Auto Mode を有効化。</p><h3>⑤ 同じエラーが繰り返し出る</h3><p>原因: Claude Code が局所的な修正だけで根本原因に踏み込んでいない。</p><p>解決: <code>このエラーの根本原因を 3 つ以上の視点で分析してから、最も確からしい修正を提案してください</code> と投げ直す。</p><h3>⑥ コード生成が途中で止まる</h3><p><strong>原因: 出力長制限・コンテキスト長超過。</strong></p><p>解決: 「ファイルを 3 つに分けて、1 つずつ提案してください」と分割指示。</p><h3>⑦ 設定ファイルが反映されない</h3><p><strong>原因: Claude Code を再起動していない。</strong></p><p>解決: 設定変更後は必ず Claude Code を一度終了して再起動。</p><h3>⑧ Remote Control が PC 側に繋がらない</h3><p>原因: PC 側の Claude Code が起動していない・スリープ中。</p><p>解決: PC 側で Claude Code を起動した状態にしておく。スリープ防止設定を入れる。</p><h3>⑨ Python のライブラリが見つからない</h3><p>原因: 仮想環境（venv）が有効化されていない・別の Python バージョンを参照している。</p><p>解決: <code>which python</code> / <code>which pip</code> で参照先を確認。venv を再有効化。</p><h3>⑩ クライアントの環境で動かない</h3><p><strong>原因: 自分の環境に依存した設定・パスがハードコード。</strong></p><p>解決: 設定はすべて環境変数化。パスは相対パスで書く。クライアントに README.md の動作確認手順を試してもらう。</p><h3>⑪ メッセージ送信時に文字化け</h3><p><strong>原因: 文字コード非対応の特殊記号（エンダッシュ等）。</strong></p><p>解決: 半角の通常記号に置換してから送信する。長文を CW の入力欄に直書きしない。</p><h3>⑫
