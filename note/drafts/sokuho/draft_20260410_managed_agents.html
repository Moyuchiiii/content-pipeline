AIエージェントを作ろうとして、インフラの設定で詰まったことはありませんか？

4月8日、AnthropicがClaude Managed Agentsを公開ベータとして発表した。

一言でいうと「エージェントを動かすためのインフラをAnthropicが全部用意してくれる」サービスで、自分でサーバーを立てたりコンテナを管理したりしなくていい。

出た当日に触ってみたので、副業で実際に使えるかを正直に書いておく。

Claude Managed Agentsって何者

従来のClaude APIでエージェントを自作しようとすると、「エージェントループ」を自分で実装する必要があった。ツールを呼び出して、結果を受け取って、次の判断をして……という繰り返しの制御ロジックを全部書く感じ。

Managed Agentsはこれを全部引き受けてくれる。定義するのは4つだけ。

Agent
モデル・システムプロンプト・使用ツール・MCPサーバーの設定
Environment
コンテナの設定（インストール済みパッケージ・ネットワークアクセス）
Session
実際にエージェントを動かすインスタンス（1タスク1セッション）
Events
自分のアプリとエージェントの間でやり取りするメッセージ

これを定義すれば、あとはAnthropicのクラウド上でエージェントが自律実行される。Bash・ファイル操作・ウェブ検索・MCPサーバーへの接続がデフォルトで使える状態になっている。

実際に触ってみた

Claude APIキーがあれば今すぐ使える。全APIアカウントで有効になっているので、申請不要。リクエストにベータヘッダー managed-agents-2026-04-01 を追加するだけ。

試したのは「CW案件の仕様書を渡したら、質問リストを自動生成して結果をファイルに書き出す」という簡単なもの。

以前は同じことをClaude Codeで手動でやっていて、セッション管理・エラーハンドリングの設計に1日近くかかった。Managed Agentsだと設定を書いてSession起動まで30〜40分で動いた。

地味に助かったのは「長時間実行しても切断後に進捗が保持される」仕組み。PCを閉じてもセッションが生きてるので、翌朝結果だけ確認すればいい。

副業で使えそうな場面

定期レポートの自動生成

毎月末に「今月の作業ログをまとめてレポートにする」タスクがあるなら向いてる。エージェントがファイルを読んで、集計して、ドキュメントを書き出すまで自律実行してくれる。自分が確認するのは最終アウトプットだけ。

案件ヒアリングの前処理

クライアントから届いた仕様書を渡して「質問事項・リスク・見積もり参考情報を整理して」と指示するだけで、ヒアリング準備の8割が終わる感覚。CW案件は仕様書の精度にムラがあるので、この前処理があると助かる。

今のところの制限

マルチエージェント連携（複数エージェントの並列実行）とメモリ機能はResearch Previewで、別途アクセスリクエストが必要。現時点では使えなかった。

料金はモデル使用料に加えてアクティブ時間あたり$0.08程度の追加コストがかかるとされている。副業規模なら月数百〜数千円の追加になりそう。

あと、Claude CodeやClaude Coworkとは別サービスなので、既存のワークフローに組み込むには設計が要る。「Claude Codeの延長線」という感覚で触ると少し戸惑うかもしれない。

結論

「エージェントは作れるけどインフラで詰まる」という問題を持ってる人には刺さる。APIで自律エージェントを動かしたい人向けの話で、Claude Codeをターミナルで使っているだけの副業スタイルには今すぐ恩恵は薄い。

ただ「AIに長時間タスクを放置したい」「定期実行エージェントを低コストで作りたい」という用途には確実に答えてくれる。Remote ControlやAuto Modeで実現していたことを、もっとシンプルな設定で動かせる可能性があるので、もう少し使い込んでみる。
