副業を始めようと思いながら、「で、具体的に何をすれば稼げるの？」がずっとわからないまま時間が過ぎてる、ってことない？

自分もそうだった。

Claude Codeで副業ができるって聞いてた。YouTubeを見ても、noteを読んでも、出てくるのはフワッとした成功談ばかりで「自分に再現できるのか？」がわからなかった。最初の2週間、クラウドワークスに登録してプロフィールを書いて案件に応募し続けた。全部で0件受注だった。

そこから動き方を変えて、1ヶ月目に55,000円、2ヶ月目に140,000円になった。

この記事では、その2ヶ月でやったことを「実績の話」ではなく「手順の話」として全部書く。副業未経験で「Claude Codeで何かやりたい」と思ってる人に向けて、最初の一歩から2ヶ月目の月14万まで、順番に書いた。

この記事で手に入るもの（無料ゾーン）

  1. 最初の2週間で0件だった理由と、そこから変えたこと
  2. Claude Codeで稼ぐために最初に覚えること3つ
  3. 1ヶ月目55,000円の内訳と、何が決め手だったか
  4. クラウドワークスで初案件を取るまでの全記録（失敗談含む）

全体の流れ（3フェーズで考える）

副業ゼロから月14万になるまでを3つのフェーズに分けて整理する。この地図を先に頭に入れておくと、今自分がどこにいるかがわかって詰まりにくくなる。

フェーズ1（最初の2週間）
CWアカウント開設→応募→0件→発想転換→初受注
フェーズ2（1ヶ月目）
案件量産×Claude Code3核心（CLAUDE.md・Auto Mode・/loop）→55,000円
フェーズ3（2ヶ月目）
単価アップ×継続収益×ococナラ参入→140,000円

ほとんどの人が「フェーズ1で詰まる」か「フェーズ1を通過したあとフェーズ2で詰まる」。この記事はその2つの詰まりポイントを全部言語化してある。

最初の2週間で0件だった理由

副業を本気でやると決めた日、最初にしたのはクラウドワークスのアカウントを作ることだった。プロフィールを書いて、案件を眺めて、「これなら自分でもできそう」と思った案件に応募し続けた。

2週間で0件受注。

原因はシンプルで、応募文が「やります、よろしくお願いします」レベルだったから。クライアント側からしたら誰でも書けるような文章で、何十人かいる応募者の中から選ばれる理由がなかった。

一方でClaude Codeは、最初の1週間で環境構築だけして実際には使えていなかった。「使い方がわからない」じゃなくて「何に使えばいいかわからない」という状態。

動き方が変わったのは「Claude Codeで応募文を作る」というアイデアが浮かんだ瞬間だった。案件詳細を全文Claudeに読ませて、「このクライアントが一番気にしてることは何か、自分の経歴でどう役立てられるか整理して、応募文を書いて」と投げた。

次の週、初受注した。

Claude Codeを「コードを書くツール」として見てたのが間違いで、最初の段階では「案件獲得の相棒」として使うのが正解だった。

Claude Codeで稼ぐ上で、最初に覚えること

全部覚えようとすると絶対詰まる。最初の1ヶ月で稼ぐために必要だったのは3つだけだった。

CLAUDE.md（記憶を永続化する）

Claude Codeはセッションをまたいで記憶を持てない。毎回「今やってる案件はこういう内容で、クライアントはこういう人で...」と説明するのは時間の無駄。CLAUDE.mdというファイルにプロジェクトの情報を書いておくと、セッション開始時に自動で読み込んでくれる。

プロジェクトのフォルダに1ファイル作るだけ。これだけで1案件あたりの準備時間が30分→5分になった。

Auto Mode（承認の手間をなくす）

Claude Codeはデフォルトだとファイルを作るたびに「実行していいですか？」と確認してくる。1案件で何十回もEnterを押すことになる。Auto Modeをオンにするとこの確認フローが自動になる。自分は「何をやるか」の方向性だけ決めれば、細かい実行はClaudeが進めてくれる。1案件あたりの作業時間が8〜10時間から2.5時間になったのはこれが一番大きかった。

/loop（放置しながら案件を進める）

/loopは指定した条件でコマンドを繰り返し実行してくれる機能。「テストが通るまで修正し続けて」「エラーが出たら修正して」を夜中の間に放置で動かせる。授業中・移動中でも処理が進んでいる感覚は、精神的にも大きかった。

初月55,000円の内訳

1ヶ月目に受注した案件は2本。

案件A
業務自動化（ExcelデータのCSV変換＋レポート生成スクリプト）
報酬 30,000円 / 実作業時間 約8時間
案件B
フォーム処理の自動化（Google Forms→Spreadsheet連携）
報酬 25,000円 / 実作業時間 約6時間

どちらもClaude Codeなしでは受注できなかった案件だった。要件定義から実装・テストまで全工程をClaudeと一緒に進めた。エラーの原因がわからなくて詰まる場面が何度もあったが、「このエラーを読んでデバッグして」と投げると8割は解決した。残り2割は自分で調べた。

「8割Claude・2割自分」という分担が、最初の1ヶ月で見えてきたバランスだった。

ここから先で手に入るもの

  1. 実際に受注した案件3ケースの詳細（Chrome拡張機能・コーチング月額継続・複合ツール案件の判断）
  2. すぐ使えるテンプレート集（応募文3タイプ全文＋CLAUDE.md設計2タイプ）
  3. ococナラ×CWのデュアル戦略（Claude関連ブルーオーシャンの実態）
  4. 2ヶ月目140,000円の全内訳と転換点3つ
  5. 月10万を超えた後にやること（指名依頼・継続収益の積み上げ方）

ケース1 EC事業者からのChrome拡張機能案件

ECサイト運営者から「Amazonの商品ページを開いたまま、ワンボタンで自社ECに商品登録できるChrome拡張機能を作ってほしい」という依頼が来た。

最初に見たとき「Chrome拡張機能か、どう進めるんだろ」と思った。でも最初にやったのは「クライアントからもらった仕様書の全文をCLAUDE.mdに貼り付けること」だった。

# EC自動化ツール CLAUDE.md

## 案件概要
Amazonの商品ページを開いた状態でボタンを押すと、以下を自動実行する
Chrome拡張機能（Manifest V3）を作成する。

処理フロー:
Amazon商品ページ → DOM抽出（商品名・説明・画像URL）
  → OpenAI APIで商品説明を加工
  → カラーミーショップAPI（OAuth2.0認証）で商品登録

- カラーミーショップAPI（OAuth2.0 Authorization Code Grant）

## 開発ルール
- コードには日本語コメントを入れる
- エラーハンドリングは必ずつける
- gitコミットは自動実行しない（自分で確認してからコミット）

これをCLAUDE.mdに入れてAuto Modeで動かした。基本的な骨組みが2日で完成した。自分でゼロから書いたら2週間かかっていた類の内容だった。

この案件からわかったことが3つある。

仕様書がある案件はCLAUDE.mdへの貼り付けで8割解決する
「読んで実装して」と投げるだけで方向性が決まる。自分で設計書を作る必要がない。
外部API連携の案件は単価が高い割にClaudeが得意なゾーン
OAuth2.0・REST API・WebExtensionのような「ドキュメントが整っている技術」はClaudeがほぼ書ける。単価2〜4万円の自動化案件と比べて単価が高い。
初回の見積もりは「仕様書をClaude Codeに読ませてから出す」と精度が上がる
「この仕様で実装するとしたら何時間かかりそうか」と聞くと、工数の見積もりが現実的になる。

ケース2 コーチング型で月額継続収益を作る

CWで「Claude Codeを仕事に使えるようになりたい、伴走型で教えてほしい」という案件が来た。クライアントはセミナー販売・講座運営をしているマーケターで、ChatGPTは使っているがClaude Codeは触ったことがないという人だった。

固定11,000円で受注して、打ち合わせ（Zoomヒアリング1回）→Claude Code初期設定サポート（Zoom1回）→月額継続サポートという流れで進んだ。

継続サポートは「テキスト＋画面収録でやりとり」する形にして、月10,000円で合意した。

ここで気づいたのが「コーチング型案件は一度受注したら毎月収益が積み上がる」ということだった。開発案件は1本終わるたびに次の案件を取り直す必要がある。でも月額継続は、先月の信頼関係の上に今月の仕事が始まる。

この案件の収益モデル（実際の数字）
初回固定 11,000円
月額継続 10,000円/月（スタンダードプランで継続中）
3ヶ月継続した場合の累計 41,000円

継続収益を作りたいなら「開発代行」より「コーチング・導入サポート」を狙った方が早い。Claude Codeの使い方をわかっている人間は、2026年時点でまだ少ない。これが最大の武器になる。

ケース3 受けるべき案件と断るべき案件の判断基準

「eBayと国内フリマサイトの価格差を自動監視してLINE通知するWebツール」の案件が来た。仕様が3フェーズ構成で、相場DB構築→リアルタイム利ザヤ検出エンジン→セラーリサーチ自動還元という内容だった。

見積もりを出す前に「この仕様書の全文をClaude Codeに読ませて、実装工数を見積もらせた」。出てきた数字は220〜280時間（フルタイム換算で約6週間）だった。

この案件は「受注しない」と判断した。

理由が3つあった。フェーズ間が密に連動する複合システムなので1フェーズのバグが全体に波及しやすいこと、複数の外部APIのレート制限・仕様変更に依存するため保守コストが読めないこと、そして単発報酬での受注は仕様変更が増えると赤字になること。

この経験から「複合システムで単価が高い案件は、月額保守込みの契約以外では基本受けない」というルールを持つようにした。

受けるべき案件
・単機能スクリプト系（CSV処理・自動化・フォーム連携等）
・Webアプリのフロントエンド改修・LP制作系
・Claude Code導入・設定サポート系（月額継続に移行しやすい）
・仕様書が明確で、変更リスクが低い案件
断るべき（または月額保守込みで交渉する）
・3フェーズ以上の複合システム開発
・複数の外部APIに依存する構成（仕様変更リスク大）
・「仕様は相談しながら決めていきたい」系（要件が固まる前の着手は危険）
・報酬が仕様の複雑さに見合わない案件

「工数の見積もりをClaudeに出させる」だけで、引き受けるべきかどうかの判断が格段に早くなった。

ококоナラ×クラウドワークスのデュアル戦略

クラウドワークスだけやっている人が見落としている市場がある。

ококоナラのClaude関連サービスは2026年3月時点で119件しかない。ChatGPT関連が1,423件あるのと比べると1/12のブルーオーシャンだった。

Claude Codeを毎日使っている自分にとって、これは圧倒的な先行者利益だった。

自分が出品したサービスは「Claude Code導入・初期設定サポート」で、3プランに分けた。

ライトプラン 2,000円
インストール確認＋初期設定＋基本的な使い方説明（テキストで納品）
スタンダードプラン 5,000円 （メイン）
ライト全内容＋CLAUDE.md構築（業務に合わせてカスタマイズ）＋ビデオ通話30分レクチャー
プレミアムプラン 10,000円
スタンダード全内容＋MCPサーバー接続設定3つまで＋運用ガイド納品＋納品後1週間チャットサポート

CWが「開発代行」中心なのに対して、ококоナラは「サポート・コーチング」型のサービスに向いている。この2つを組み合わせると収益の種類が増える。

CWは「開発案件→完了→次の案件探し」の繰り返し。ококоナラのコーチング系は「購入→継続→追加購入」の積み上げ型。どちらか片方より、両方やる方が収益が安定する。

2ヶ月目に月収が2.5倍になった3つの転換点

55,000円が140,000円になった。3つの変化があった。

転換点1 Auto Modeのdeny rulesを絞る

1ヶ月目はAuto Modeを使っていたが、「安全のため」という理由でdeny rules（実行を禁止する操作リスト）を広く設定していた。結果として詰まることが多くて、Auto Modeの効果が半分以下になっていた。

deny rulesを「gitコミット・外部APIへのリクエスト」だけに絞ったら、1案件あたりの作業時間がさらに短くなった。設定ファイルの数行を変えるだけで、月に10時間以上の差が出た。

# settings.json の deny rules（絞ったあとの設定）
{
  "permissions": {
    "deny": [
      "Bash(git commit*)",
      "Bash(git push*)",
      "WebFetch(*)"
    ]
  }
}

これ以外は全部許可した。最初は怖かったが、実際にやらかした事故はゼロだった。

転換点2 WebアプリUI案件にステップアップ

業務自動化案件は単価2〜4万円が相場。WebアプリUI案件は1本4〜7万円のレンジが多い。同じ作業時間で単価が1.5〜2倍になる。

「LP改修＋管理画面」案件（65,000円）を2ヶ月目に受注できたのが転換点だった。Claude Code＋Auto Modeで設計書から20分で基本UIが完成したから、クライアントへのデモが速かった。「デモが速い」ことで信頼が上がって、その後の修正対応もスムーズになった。

転換点3 Remote Controlで移動中も案件を進める

Claude Code Remote Controlを使うと、スマホからPCのClaude Codeに指示を送れる。

授業の移動中・電車の中で「この部分を修正して」とスマホから送ると、帰宅したら完成している。週10時間かかっていたPC前の待機時間が3時間になった。これで複数案件を並行して進められるようになった。

2ヶ月目140,000円の全内訳

案件C（LP改修＋管理画面）
報酬 65,000円 / 実作業時間 約15時間
案件D（指名依頼・タスク管理ツールUI）
報酬 45,000円 / 実作業時間 約10時間
案件E（月額継続・Claude Code導入サポート1ヶ月目）
報酬 10,000円 / 作業時間 約4時間（Zoomセッション2回）
案件F（1ヶ月目の既存クライアントからの追加修正）
報酬 20,000円 / 実作業時間 約5時間

合計 140,000円。前月比＋85,000円。

注目してほしいのは案件Fで、1ヶ月目の既存クライアントからの追加依頼だった。新規案件を取り続けるより、既存クライアントへの追加対応の方が単価交渉がしやすく、応募コストがゼロ。「既存クライアントに追加提案する習慣」が2ヶ月目の底上げに大きく効いた。

月10万を超えた後にやること

10万を超えてから、やることが変わった。「案件をこなす」から「どの案件を取るか選ぶ」になる。

受注できる量より多く取ろうとした時期もあったが、クオリティが落ちてクライアントの評価が下がった。プラットフォームの評価は一度下がると取り戻すのが難しい。

安定させるために意識したのは「プロフィールの評価を積み上げる」こと。評価が10件を超えると指名依頼が来るようになって、指名依頼は単価が高く応募コストもない。2ヶ月目の140,000円の内訳には、指名依頼1本45,000円が含まれていた。

それ以降は「月収を増やす」ではなく「同じ収益をより少ない時間で」という方向に意識が変わった。ococナラの月額継続が積み上がると、CWで毎月必死に稼ぐ必要がなくなってくる。

副業ゼロから2ヶ月でやったことを、全部書いた。

難しいことは何もない。「Claude Codeをどう使うか」より「どんな案件を取って、どう動くか」の方が最初は100倍大事だった。最初の1ヶ月は「初受注を取る」だけに集中すればいい。テンプレートを使って、応募文を1本ずつ改善する。それだけでいい。
