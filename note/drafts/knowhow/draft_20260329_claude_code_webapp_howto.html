「Webアプリって自分で作れますか？」

Claude Codeの話をするたびに、これを聞かれる。

結論から言うと、作れる。しかも自分が想像してたより全然速く。今回は「Claude Codeでゼロからどうやってアプリを完成させるか」を、実際にやってみた経験とリサーチをまとめて書く。

プログラミング経験ゼロの人でも3日で動くものができる。この記事ではその全手順を書く。

実際に作られたWebアプリの事例

「本当に作れるの？」という疑問から始めると思うので、まず実例を見てほしい。自分がリサーチした中で特に印象に残った3つ。

音楽スクールの予約管理システム（非エンジニア・3日間）

noteにあった事例で、音楽スクールを運営している非エンジニアの人がClaude Codeで予約管理システムを3日間で作った話。コード経験ゼロでも動くものができたと書いてあった。

生徒が自分でレッスンを予約・キャンセルできるフォーム
講師側の予約一覧画面
自動確認メール送信機能

3日でここまで作れるのは、Claude Codeがコードを全部書いてくれるから。自分がやることは「何を作りたいか」を日本語で説明するだけ。

社内匿名Q&Aアプリ（Zenn掲載事例）

Zennで見た事例で、URLを開くだけで参加できる社内用の匿名Q&Aアプリ。CLAUDE.mdとHooksという機能を組み合わせて「爆速開発した」と書かれていた。認証不要・匿名投稿・いいね機能付き。

この事例が面白かったのは、「シンプルな機能に絞った」という点。認証システムを省いたことで、URLを共有するだけで誰でも使えるようになった。

在庫管理システム（開発3時間・年間240時間削減）

これが一番インパクトがあった事例。開発時間3時間で、年間240時間の業務が削減できた。3時間で240時間分の仕事が消えた計算になる。

機能は在庫数の記録・アラート・レポート出力。シンプルだけど、毎日使うものだから削減効果が積み上がった。

共通してるのは「一つの問題を解決するシンプルなアプリ」という点。最初から完璧なものを作ろうとしてない。

なぜClaude CodeだとWebアプリ開発が速いのか

他のAIツールと比べて、Claude CodeがWebアプリ開発で速い理由が3つある。自分が実際に使ってみて「これが効いてる」と感じたやつを説明する。

CLAUDE.mdでプロジェクトの記憶を持てる

プロジェクトのルートにCLAUDE.mdというファイルを置くと、Claude Codeがそれを毎回読み込む。「このプロジェクトはTypeScriptを使う」という情報を一度書いておけば、セッションが変わっても設定が引き継がれる。

CLAUDE.mdなしの場合
セッションのたびに「このアプリの構成は…」から説明し直す
CLAUDE.mdありの場合
「ユーザー認証機能を追加して」と言えばコンテキスト込みで動いてくれる

これだけで開発速度が体感で3倍は変わる。後で説明するテンプレートをコピペするだけでいい。

コードベース全体を理解して動く

Claude Codeはコードベース全体を読んでから動く。「このファイルとこのファイルが連携してる」という理解の上で修正してくれるから、既存のコードを壊しにくい。

チャットAIだと「ここのコードを直して」と言っても他の部分を壊すことがある。Claude Codeはそれが少ない。自分が使ってみた印象では「手を入れてない部分に影響が出る」ケースが明らかに減った。

コマンドまで自動で実行してくれる

「npm installして起動して確認して」まで自動でやってくれる。自分でコマンドを打つ必要がない。これが地味に大きくて、環境構築で詰まることが減る。正直、開発中に一番時間をとられてたのがコマンドまわりだったので、ここが自動化されるのは助かる。

Day 1：CLAUDE.mdを設計する

最初に一番大事なことを言う。Webアプリ開発でClaude Codeを使い始めるとき、最初にやるべきはコードを書くことじゃなくCLAUDE.mdを設計すること。

CLAUDE.mdがないまま開発を始めると、後から修正するたびに文脈がずれていく。最初にここに時間をかけた方が結果的に速い。経験上、CLAUDE.mdをサボって始めると後半の修正コストが2倍以上になる。

CLAUDE.mdの基本テンプレートはこれ（コピペして自分のプロジェクトに合わせてください）

# プロジェクト概要
このWebアプリは[目的を1行で]のためのツールです。

対象ユーザー: [誰が使うか]
主な用途: [何をするためのアプリか]

## 技術スタック
- フロントエンド: React + TypeScript
- バックエンド: Node.js + Express
- データベース: SQLite（開発時）/ PostgreSQL（本番）
- スタイリング: Tailwind CSS
- デプロイ: Vercel

## 主要機能
1. [機能1: 例）ユーザーが予約を入れられる]
2. [機能2: 例）管理者が一覧を確認できる]
3. [機能3: 例）確認メールが自動送信される]

## 開発ルール
- コンポーネントはsrc/components/に作成
- APIエンドポイントはsrc/api/に作成
- 日本語でコメントを書く
- 変更後は必ずnpm testを実行
- エラーが出たら必ずエラー内容を報告してから修正する

## ファイル構成
src/
  components/  # UIコンポーネント
  api/         # APIルート
  types/       # TypeScript型定義
  utils/       # ユーティリティ関数

## 現在の開発状況（毎日更新する）
- 完成済み: （作業が進んだら記入）
- 作業中: プロジェクト初期化
- 次のタスク: （決まったら記入）

「現在の開発状況」のセクションを毎日更新するのがポイント。翌日のセッション開始がスムーズになる。

Day 1：最初の指示プロンプト

CLAUDE.mdを作ったら、最初の指示を送る。「全部一気に作って」と言ってしまうのが一番よくある失敗。自分が実際に使っているプロンプトはこれ

CLAUDE.mdを読んで、このWebアプリの基本構造を作ってください。

今回のゴール: 「npm start でサーバーが起動すること」だけです。

手順:
1. プロジェクトの初期化
2. 必要なパッケージのインストール
3. 起動確認

機能の実装はまだ不要です。まずは起動できる状態にしてください。
終わったら、起動コマンドを教えてください。

小さく分割して確認しながら進む。これをやらないと「大量のコードが出てきたけど動かない」という状態になりやすい。

Day 2：機能を一つずつ追加する

アプリが起動できたら、機能を一つずつ追加していく。指示の出し方はこのフォーマットで統一するといい

[機能名]を追加してください。

条件:
- [要件1: 例）入力フォームは名前・日付・時間の3項目]
- [要件2: 例）送信ボタンを押すとDBに保存される]
- [要件3: 例）保存後に確認画面に遷移する]

追加したら、動作確認の手順も教えてください。

機能の追加順序はこれを参考にしてほしい

① データの保存・取得
バックエンドのAPIから作る。ここが動けばあとは繋げるだけ
② 一覧表示
フロントエンドの基本画面。保存したデータが見えるようにする
③ 追加・削除機能
CRUD操作。①②が動いてから着手する
④ バリデーション・デザイン
最後でいい。③まで動けばアプリとして使える

Day 3：デプロイ

Vercelを使うのが一番楽。GitHubリポジトリを作ってプッシュすれば、VercelのダッシュボードからImportするだけで自動デプロイされる。

Claude Codeへの指示はこれ

このアプリをVercelにデプロイしたいです。

1. 必要な設定ファイルを作ってください（vercel.jsonなど）
2. .gitignoreを確認して不要なファイルが含まれてないか確認してください
3. デプロイ前に確認すべきことがあれば教えてください

実際につまずいたポイント3つ

1. 「何を作るか」が曖昧なまま始めた

最初に失敗したのはここ。「予約システムを作って」と言ったら、Claude Codeが「どんな予約システムですか？」と逆に質問してきた。

開発を始める前に「誰が使うか」「何ができればいいか」「絶対に必要な機能」を5行で書き出す。この5分を省いて始めると、後で2時間ロスする。

2. エラーが出てパニックになった

開発中にエラーが出ることは普通にある。「エラーが出ました、直してください」だけ言うより、エラーメッセージを全文貼る方が解決が早い

以下のエラーが出ました。

[エラーメッセージ全文をここにペースト]

エラーが出る操作: [例）「保存ボタンを押したとき」]
環境: [例）localhost:3000で確認中]

エラーメッセージを全文貼るだけで、解決時間が体感で半分になる。

3. セッションをまたぐとコンテキストがずれた

1日作業して翌日再開すると、前日のやりとりがリセットされる。対処法は、CLAUDE.mdの「現在の開発状況」セクションを毎日更新すること。作業終わりに2分でいいのでここを書き直す

## 現在の開発状況（2026-03-29更新）
- 完成済み: ユーザー認証、一覧表示画面
- 作業中: 予約フォームのバリデーション（エラー表示が出ない問題あり）
- 次のタスク: メール通知機能の追加

翌朝「続きから」とだけ言えば前日の状態を把握して動いてくれる。

クラウドワークスで案件化する方法

Webアプリが作れるようになると、クラウドワークスでの案件の幅が広がる。自分が実際に受注している案件で多いのはこういう種類

業務の一部をデジタル化したい（紙の台帳をWebアプリに変換したい）
社内ツールを作ってほしい（予約管理・在庫管理・進捗管理など）
既存のWebサイトに機能を追加してほしい

応募文の型はこれを使っている

[クライアントの課題]を解決するWebアプリを開発できます。

実績:
・[類似案件の実績（例: 予約管理システム開発、3日間、¥45,000）]
・使用技術: React / Node.js / Vercel

開発の進め方:
1. 要件確認（1日）: どんな機能が必要かをヒアリング
2. 基本機能の実装（2〜3日）: 動くものを早めに見せます
3. 修正対応（1〜2日）: フィードバックをもとに調整

合計5〜7日で納品可能です。
まずは要件のヒアリングから始めさせてください。

「動くものを早めに見せます」という一文を入れると、クライアントの不安が下がってレスが早くなる傾向がある。
