## 「予約管理、どうにかならないの」と言われた日

高校の同期が個人サロンを始めた。ネイルとまつ毛の店で、予約はぜんぶLINEで受けてると言っていた。

「予約フォームをサイトに置きたいんだけど、どこかに頼んだら30万って言われた。高くない？」

30万。自分もその数字を聞いてちょっと引いた。外注相場的には妥当な金額だけど、開業直後の個人サロンには重い。

「Claude Codeで作れるかも」と思ったのがそのときで、実際にやってみたら3日で完成した。費用はClaude Proの月額3,000円程度だけ。

この記事では、自分が実際にやった工程を全部書く。プロンプトも、詰まったポイントも全公開。

---

## 何を作るのかを先に整理する

まず「どんなシステムを作るか」を決める。今回の構成はこれだ。

**フロントエンド**
HTML + CSS + JavaScript（1ファイルに全部まとめる）

**バックエンド**
Google Apps Script（GAS）— Googleが無料で提供するサーバーレス環境

**データ保存場所**
Google スプレッドシート — 予約データがここに溜まっていく

なぜこの組み合わせにしたかというと、サーバー代が0円で、オーナーがExcelみたいな感覚でデータを見られるから。

作った機能はシンプルに3つ。
- 予約フォーム（名前・メール・希望日時・メニュー選択）
- 管理者ページ（予約一覧・キャンセルボタン付き）
- スプレッドシートへの自動記録

---

## 「CLAUDE.md」を先に書く。これが全て

いきなりClaude Codeに「予約管理システムを作って」と投げても、まともなものは出てこない。

CLAUDE.mdというファイルを先に作ることが、Claude Codeをうまく使う上で一番重要なステップだ。Claude Codeはプロジェクトフォルダにあるこのファイルを起動時に自動で読み込む。

今回書いたCLAUDE.md:

```
# 予約管理システム

## 概要
個人サロン向けの予約管理Webアプリ。

## 技術スタック
- フロントエンド: HTML + CSS + JavaScript（1ファイルで完結させる）
- バックエンド: Google Apps Script（GAS）
- データ保存: Google スプレッドシート

## 機能
### 予約フォーム（お客さん用）
- 名前・メール・希望日・希望時間・メニューを入力して送信
- 送信後に「予約を受け付けました」の確認メッセージを表示
- GASのエンドポイントにPOSTで送る

### 管理ページ（オーナー用）
- URLに ?admin=true を付けると管理ページを表示
- 予約一覧をテーブルで表示
- 各予約にキャンセルボタンを付ける
- ステータスを変更できる

## ルール
- シンプルに作る。余計な機能は追加しない
- コメントは日本語で書く
- エラーが起きたときはコンソールに詳細を出力する
```

これを書くのに30分かかったが、この30分が後の全工程を速くする。

## [有料] Day 1 — フォームを動かすまで

最初のプロンプト:
```
CLAUDE.mdを読んで、予約フォームのHTMLファイルを作ってください。
まずフォームだけ。GASとの連携は後でやります。
index.htmlというファイルで出力してください。
```

「全部作って」と言わないのがコツ。フォーム → GAS連携 → 管理者ページと積み上げる。
