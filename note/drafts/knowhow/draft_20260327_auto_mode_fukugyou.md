# 副業の"承認待ち"が1案件あたり30分→5分になった話｜Claude Code Auto Mode 導入記

## 副業タスク中に、毎回承認ポップアップが来る問題

CWの案件を進めていると、こういう流れになることがある。

Claudeがコードを書く → **承認しますか？** → OK → テスト実行 → **承認しますか？** → OK → ログファイル保存 → **承認しますか？** → OK

1つのタスクを仕上げるだけで、5〜10回は確認が来る。作業に集中してても、スマホをいじってても、その度に手を止めてPCの前に戻る必要があった。

地味にきつい。「承認地獄」って表現がぴったりな状態だった。

Auto Modeを使ったら、その承認待ちが1案件あたり**30分→5分**になった。今日はその話をする。

---

## Auto Modeとは何か（30秒で理解できる説明）

Auto Modeは、Claude Codeが実行する各操作を**AI分類器がリアルタイムで判定**して、安全な操作は自動実行・危険な操作はブロックするモード。

これまでは3つのモードがあった。

> ① **Default**: 全操作に対して承認を求める（デフォルト）
> ② **--dangerously-skip-permissions**: 全承認をスキップ（危険、隔離環境向け）
> ③ **Auto Mode（新）**: AIが安全/危険を分類して自動判定

Auto Modeは①と②の中間。低リスク操作は自動、高リスク操作だけ確認を求める。

「全部OKにすればいいじゃん」と思うかもしれない。でも②を使うと、たとえばClaudeが間違えて`rm -rf`を実行しようとしても止められない。Auto Modeならそういう操作を事前にブロックしてくれる。

---

## 副業での"判断基準"が大事な理由

Auto Modeで自動実行されるのは「安全な操作」だが、この判断はAIがやる。完璧じゃない。

副業で使うとき、自分が特に意識したのは「どのタスクをAuto Modeに任せていいか」の整理だった。

実際に整理してみると、こんな感じに分かれた。

> **Auto Modeに任せていいタスク（低リスク）:**
> - プロジェクトファイルの読み込み、内容確認
> - コードの編集・リファクタリング
> - テストの実行（ローカル）
> - ログファイルの生成・保存
> - README等のドキュメント更新

> **自分が確認すべきタスク（高リスク）:**
> - 外部APIへのリクエスト
> - ファイルの大量削除
> - 環境変数・秘密鍵が含まれるファイルへのアクセス
> - クライアントへの成果物の送信

この判断基準を最初に整理しておくと、Auto Modeでどこまで放置できるかがはっきりする。

## Before / After

> **Before（Auto Modeなし）:**
> CWの案件1件（自動化ツール開発、約3時間の作業）で承認ポップアップが平均12回。
> 1回ごとに「画面に戻る→確認→再開」で平均2〜3分かかる。
> 合計: **約30分が承認対応で消える**

↓ Auto Mode ONにして1週間後

> **After（Auto Modeあり）:**
> 同じ規模の案件で、確認が必要だったのは外部APIテストと最終ファイル送信の**2回だけ**。
> 合計: **約5分**

週3〜4件こなしていると、月換算で4〜5時間の差になる。

その時間を別の案件の応募文作成や次の学習に使えるようになった。

## Auto Modeの有効化手順

まず前提: Auto ModeはClaude Codeの**Teamプランのみ**が対象（2026年3月時点ではリサーチプレビュー）。Proプランはまだ使えない。

Teamプランに入っているなら、有効化は1コマンドで終わる。

```
claude --enable-auto-mode
```

ターミナルでこれを実行したあと、セッション内で**Shift+Tab**を押すとモードが切り替わる。Default → Auto → Plan の順番で切り替わっていく。

VS Codeの拡張機能で使っている場合は、Settings → Claude Code → Auto ModeをオンにすればOK。

## 必ず設定すべきdeny rules

Auto Modeを使う前に、これだけは設定してほしい。

deny rulesは「このコマンドは絶対に実行させない」というブロックリストを自分で定義できる機能。Auto Modeの分類器は完璧じゃないので、手動でのダブルロックを入れておく。

`.claude/settings.json` に以下を追加する。

```json
{
  "deny": [
    "Bash(rm:-rf*)",
    "Bash(curl:*)",
    "WebFetch(*)",
    "Bash(cat:*.env*)"
  ]
}
```

> **各deny ruleの意味:**
> - `rm -rf *`: ファイルの大量削除をブロック
> - `curl`: 外部へのリクエストをブロック（意図しないAPI呼び出し防止）
> - `WebFetch`: 外部URLへのアクセスをブロック
> - `cat *.env*`: 環境変数ファイルの閲覧をブロック

この設定でAuto Modeの分類器が見逃した操作をdeny rulesが止めてくれる。二重の安全網になる。

## CLAUDE.mdに書くAuto Mode向けの設定テンプレート

副業案件用のCLAUDE.mdに、Auto Mode前提の指示を追加しておくと効果が増す。

```
# 作業ルール

## Auto Mode使用時の動作指定
- ファイル読み込み・編集・テスト実行は承認不要で進める
- 外部APIテストの前に必ずdry runで動作確認する
- ファイル削除が必要な場合は必ず確認を取る
- 秘密鍵・.envファイルには触れない

## 作業完了の定義
- テストが全件グリーン
- READMEを最新状態に更新済み
- コードにコメントを追加済み
```

「外部APIテストの前にdry run」という一文がポイント。これを入れておくと、Claudeが外部リクエストを送る前に必ずローカルでの動作確認を挟んでくれる。

## 副業タスク別 ワークフロー設計 3パターン

### パターン1: コード開発・リファクタリング案件

CWで一番多い「既存コードの改修・機能追加」系の案件。

> Auto Modeで自動化できる部分:
> ① 既存コードの読み込み・理解
> ② コードの修正・追記
> ③ ユニットテストの実行
> ④ ログ・エラーメッセージの確認
> ⑤ ドキュメントの更新

> 自分が確認する部分:
> - 外部サービスへの接続テスト
> - クライアントへの納品前の最終確認

これだけで案件作業の80%以上をClaude一人に任せられる。

### パターン2: データ整理・ドキュメント作成案件

CSVの整形、スプレッドシートの集計、レポート作成など。

> Auto Modeで自動化できる部分:
> ① CSVファイルの読み込み・パース
> ② データの集計・フォーマット変換
> ③ レポートファイルの生成
> ④ 複数ファイルのマージ処理

> 自分が確認する部分:
> - クライアントの個人情報が含まれるデータの扱い
> - 完成レポートの内容チェック（数字の確認）

個人情報や機密情報が含まれる場合は、deny rulesに加えて「特定フォルダ以外は触らない」とCLAUDE.mdに書いておく。

### パターン3: リサーチ・情報収集案件

競合調査・市場調査・要件定義のリサーチなど。

このパターンはやや注意が必要。Auto Modeをオンにしていても、外部URLへのアクセスはdeny rulesでブロックしているので、WebFetchが使えない。

リサーチ案件では自分がURLを渡して、Claude Codeが処理するフローにする。

> Auto Modeで自動化できる部分:
> ① 自分が渡したテキストデータの分析・整理
> ② 競合情報の比較表作成
> ③ レポートのMarkdown/HTML化
> ④ キーポイントの抽出

## つまずいたこと3つ

### ① Teamプラン限定で最初から使えなかった

Auto ModeはProプランでは使えない（2026年3月時点）。最初にこれを知らなくて、`claude --enable-auto-mode`を実行しても反応がなくてハマった。

副業でClaude Codeをがっつり使うなら、Teamプランへの切り替えを検討するタイミングかもしれない。Proとの差額は月数千円だが、Auto Modeだけでも元が取れる可能性がある。

### ② 分類器が「Borderline」と判断してくる操作がある

完全に安全なはずの操作でも、Auto Modeが「確認しますか？」と聞いてくることがある。CLAUDE.mdで作業ルールを明確にしておくと、この頻度が下がった。

### ③ gitチェックポイントを忘れがち

Auto Modeで一気に作業が進むので、いつの間にかコードが大量に変更されている。Auto Modeを使う前に必ずgit commitをしておく習慣をつけた。

```
git add . && git commit -m "before auto-mode session"
```

この一行を習慣にするだけで、万が一のときに戻れる。

## まとめ

Auto Modeを副業に使うときのポイントをまとめると、こういう話になる。

> ① Auto Modeを使う前に「任せるタスク」と「確認するタスク」を整理する
> ② deny rulesで二重のブロックを設定する
> ③ CLAUDE.mdで「外部APIはdry runから」等のルールを明示する
> ④ セッション前に必ずgit commit

これだけやれば、案件の承認待ちが劇的に減る。

副業の作業時間は有限で、承認ポップアップに使える集中力なんてない。その時間を他に使えるようになったのが、一番の変化だった。
