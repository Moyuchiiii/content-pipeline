# Claude Codeの/team-onboardingを1人副業で使ったら、自分のCLAUDE.mdに抜けてた3つが見えた話

自分がClaude Codeで何をどう使ってるか、ちゃんと説明できますか？

副業で毎日触ってるのに、いざ「どんな設定で、どのコマンドを、どういう順で使ってるか」を聞かれると、意外と答えられない。自分がそうだった。

CLAUDE.mdもdeny rulesもskillも、全部「その時の気分」で足していった結果、頭の中にしか記録がない。新PCに移ったら再現に30分。弟に「Claude Codeで副業始めたい」と言われた時は、説明だけで1時間半かかった。

4/10に出た`/team-onboarding`を、自分に向けて使ってみた。副業は1人プレイだから本来の用途とは違うけど、「自分のワークフローを自分に説明し直す」ツールとして使える気がしたから試した。

結果、**CLAUDE.mdに書き忘れてた3つの抜け**が出てきた。

## /team-onboardingが何をやるのか

Claude Code v2.1.101（2026-04-10リリース）で追加された新コマンド。ローカルのClaude Code使用履歴を分析して、チームメンバー用のオンボーディングガイドを自動で生成する。

- 入力: ローカルのセッション履歴・CLAUDE.md・memory・設定
- 出力: 使い方の棚卸しドキュメント（よく使うコマンド・skill・hook・deny rulesの傾向）

本来はチームで共有するツールだけど、**出力ドキュメントを自分で読むと「自分が無意識にやってること」が文字になって出てくる**。この効果が大きい。

## 実際に動かしてみた

`/team-onboarding` を打つだけ。30秒で約1,800字のガイドが出た。中身は4ブロック。

1. よく使うコマンドとskill（頻度順）
2. CLAUDE.mdの要約
3. 典型的なセッションの流れ
4. 新規メンバーが最初にやるべきこと

## 見えた3つの抜け

### 抜け1: deny rulesの「理由」が書いてない

`rm -rf` や `git push --force` がdenyされてるのに、なぜdenyなのか書いてなかった。弟に説明した時、理由を聞かれて5分使った。

### 抜け2: skillの組み合わせパターンが暗黙知

`/collect-stats` → `/note-run` → `/content-engine` → `/x-run` がワンパターンなのに未記述。onboarding出力がセッション履歴から順序を拾ってきた。

### 抜け3: 案件タイプ別の分岐が記録されていない

CW案件とococナラ案件で使うskillが違う。頭の中にしかなかった。

## Before/After

- セットアップ再現: 30分 → 30秒
- 人に教える時: 1時間半 → 5分の質問対応のみ
- CLAUDE.md: 900字 → 2,100字

## つまずき

- セッション履歴30未満だと出力が薄い
- 固有名詞が混ざるのでサニタイズしてから共有
- チームプラン限定ではない

## 1人プレイでも使う価値がある3つの理由

1. CLAUDE.md改善の材料になる（linter感覚）
2. 新PC移植が速くなる
3. 将来「教える」素材になる

## 次にやること

月1で再生成して差分をCLAUDE.mdに反映するフローを試す。
