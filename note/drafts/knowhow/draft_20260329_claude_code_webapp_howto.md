## 実際に作られたWebアプリの事例

「本当に作れるの？」という疑問から始めると思うので、まず実例を見てほしい。自分がリサーチした中で特に印象に残った3つ。

### 音楽スクールの予約管理システム（非エンジニア・3日間）

noteにあった事例で、音楽スクールを運営している非エンジニアの人がClaude Codeで予約管理システムを3日間で作った話。コード経験ゼロでも動くものができたと書いてあった。

> 作ったもの: 生徒が自分でレッスンを予約・キャンセルできるフォーム
> 作ったもの: 講師側の予約一覧画面
> 作ったもの: 自動確認メール送信機能

3日でここまで作れるのは、Claude Codeがコードを全部書いてくれるから。自分がやることは「何を作りたいか」を日本語で説明するだけ。

### 社内匿名Q&Aアプリ（Zenn掲載事例）

Zennで見た事例で、URLを開くだけで参加できる社内用の匿名Q&Aアプリ。CLAUDE.mdとHooksという機能を組み合わせて「爆速開発した」と書かれていた。認証不要・匿名投稿・いいね機能付き。

この事例が面白かったのは、「シンプルな機能に絞った」という点。認証システムを省いたことで、URLを共有するだけで誰でも使えるようになった。

### 在庫管理システム（開発3時間・年間240時間削減）

これが一番インパクトがあった事例。**開発時間3時間で、年間240時間の業務が削減できた**。3時間で240時間分の仕事が消えた計算になる。

機能は在庫数の記録・アラート・レポート出力。シンプルだけど、毎日使うものだから削減効果が積み上がった。

共通してるのは「一つの問題を解決するシンプルなアプリ」という点。最初から完璧なものを作ろうとしてない。

## なぜClaude CodeだとWebアプリ開発が速いのか

他のAIツールと比べて、Claude CodeがWebアプリ開発で速い理由が3つある。

### CLAUDE.mdでプロジェクトの記憶を持てる

プロジェクトのルートに**CLAUDE.md**というファイルを置くと、Claude Codeがそれを毎回読み込む。「このプロジェクトはTypeScriptを使う」という情報を一度書いておけば、セッションが変わっても設定が引き継がれる。

- CLAUDE.mdなし: セッションのたびに「このアプリの構成は…」から説明し直す
- CLAUDE.mdあり: 「ユーザー認証機能を追加して」と言えばコンテキスト込みで動いてくれる

### コードベース全体を理解して動く

Claude Codeはコードベース全体を読んでから動く。「このファイルとこのファイルが連携してる」という理解の上で修正してくれるから、既存のコードを壊しにくい。

### コマンドまで自動で実行してくれる

「npm installして起動して確認して」まで自動でやってくれる。自分でコマンドを打つ必要がない。

## 実際の作り方：3日で完成させる全手順

### Day 1：CLAUDE.mdを設計する

Webアプリ開発でClaude Codeを使い始めるとき、最初にやるべきはコードを書くことじゃなく**CLAUDE.mdを設計すること**。

CLAUDE.mdの基本テンプレート:

```
# プロジェクト概要
このWebアプリは[目的を1行で]のためのツールです。

対象ユーザー: [誰が使うか]
主な用途: [何をするためのアプリか]

## 技術スタック
- フロントエンド: React + TypeScript
- バックエンド: Node.js + Express
- データベース: SQLite（開発時）/ PostgreSQL（本番）
- スタイリング: Tailwind CSS
- デプロイ: Vercel

## 主要機能
1. [機能1]
2. [機能2]
3. [機能3]

## 開発ルール
- コンポーネントはsrc/components/に作成
- APIエンドポイントはsrc/api/に作成
- 日本語でコメントを書く
- 変更後は必ずnpm testを実行
- エラーが出たら必ずエラー内容を報告してから修正する

## ファイル構成
src/
  components/  # UIコンポーネント
  api/         # APIルート
  types/       # TypeScript型定義
  utils/       # ユーティリティ関数

## 現在の開発状況（毎日更新する）
- 完成済み:
- 作業中: プロジェクト初期化
- 次のタスク:
```

### Day 1：最初の指示プロンプト

```
CLAUDE.mdを読んで、このWebアプリの基本構造を作ってください。

今回のゴール: 「npm start でサーバーが起動すること」だけです。

手順:
1. プロジェクトの初期化
2. 必要なパッケージのインストール
3. 起動確認

機能の実装はまだ不要です。まずは起動できる状態にしてください。
終わったら、起動コマンドを教えてください。
```

### Day 2：機能を一つずつ追加する

```
[機能名]を追加してください。

条件:
- [要件1]
- [要件2]
- [要件3]

追加したら、動作確認の手順も教えてください。
```

機能の追加順序:
1. データの保存・取得（バックエンドのAPI）
2. 一覧表示（フロントエンドの基本画面）
3. 追加・削除機能（CRUD操作）
4. バリデーション（入力チェック）
5. デザイン調整（Tailwindで整える）

### Day 3：デプロイ

```
このアプリをVercelにデプロイしたいです。

1. 必要な設定ファイルを作ってください（vercel.jsonなど）
2. .gitignoreを確認して不要なファイルが含まれてないか確認してください
3. デプロイ前に確認すべきことがあれば教えてください
```

## 実際につまずいたポイント3つ

### 1. 「何を作るか」が曖昧なまま始めた

CLAUDE.mdを作る前に「誰が使うか」「何ができればいいか」「絶対に必要な機能」を5行で書き出す。この5分を省いて始めると、後で2時間ロスする。

### 2. エラーが出てパニックになった

```
以下のエラーが出ました。

[エラーメッセージ全文をここにペースト]

エラーが出る操作: [例）「保存ボタンを押したとき」]
環境: [例）localhost:3000で確認中]
```

### 3. セッションをまたぐとコンテキストがずれた

CLAUDE.mdの「現在の開発状況」セクションを毎日更新するだけで解決。

```
## 現在の開発状況（2026-03-29更新）
- 完成済み: ユーザー認証、一覧表示画面
- 作業中: 予約フォームのバリデーション
- 次のタスク: メール通知機能の追加
```
