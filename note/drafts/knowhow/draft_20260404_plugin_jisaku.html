PC買い替えのたびに、Claude Codeの設定を最初からやり直してませんか？

昨日、公式プラグインを3本入れたら週の副業作業時間が5時間から2時間になった（別記事に書いた）。でも正直、まだ問題が残ってた。

CW案件専用のスキル、納品前に走らせるシェルスクリプト、PR作成前の確認フック。これらは自分のPCの .claude/ フォルダに散らばっていて、新しいPCを使うときに毎回コピーしてた。どこに何があったか探す時間も含めると、1台あたり30分ぐらいかかってた。

そこで「プラグインって、自分でも作れないのか」と思って調べてみた。作れた。しかも /plugin install 1コマンドでどのPCにも入る。

プラグインに詰め込めるもの

Claude Codeのプラグインは、ディレクトリ1つで完結する。以下のフォルダを置くと、インストール時に自動で展開される。

skills/
カスタムスキル。/plugin-name:skill-name の形で呼び出せる
hooks/
フック設定（hooks.json）。ファイル保存後のlint実行、ツール使用前の確認など
bin/
実行ファイル。プラグイン有効中はBash toolのPATHに追加される（v2.1.91から）
settings.json
プラグイン有効時に適用されるデフォルト設定

これをgitリポジトリに入れておけば、どのPCでも /plugin install ./cw-plugin で全部入る。

最低限の構造から作る

まずディレクトリを作る。

mkdir cw-plugin
mkdir cw-plugin/.claude-plugin

次にマニフェストファイルを作る。

{
  "name": "cw-plugin",
  "description": "CW案件用ワークフロープラグイン",
  "version": "1.0.0",
  "author": {
    "name": "moyuchi"
  }
}

保存先は cw-plugin/.claude-plugin/plugin.json。name フィールドがスキルのプレフィックスになる。 cw-plugin:check のような形で呼び出せるようになる。

スキルを追加する

案件受け取り時に毎回確認する項目をスキルにする。

mkdir -p cw-plugin/skills/check

cw-plugin/skills/check/SKILL.md を作る。

---
description: CW案件受け取り時の確認チェックリスト
---

受け取った案件を確認してください。

- 納期と作業時間のバランス
- 使用技術スタックの実現可能性
- 曖昧な要件が残っていないか
- テスト環境の有無

問題がある項目はリストアップし、クライアントへの確認文も一緒に生成してください。

ローカルでテストする

--plugin-dir フラグで、インストールせずにそのまま読み込める。

claude --plugin-dir ./cw-plugin

起動したら /cw-plugin:check を叩いてみる。スキルが動いたらOK。変更後は /reload-plugins を叩くと再読み込みできる。Claude Codeを再起動しなくていい。

ここまでで、スキルだけのプラグインが動く。これだけでも毎回 .claude/commands/ にファイルを作る手間がなくなる。

ここから有料エリア

CW案件専用プラグインの全コードを公開します

  1. bin/ スクリプト（v2.1.91新機能）で納品前バリデーションを自動化
  2. hooks.json でファイル保存のたびに自動フォーマット
  3. GitHubにpushして別PCに10秒でインストールする方法
  4. つまずきポイント3つ（権限エラー、name制限、hooks書き方）

bin/ スクリプトを追加する（v2.1.91の新機能）

v2.1.91から、プラグインに bin/ ディレクトリを作ると、その中の実行ファイルがBash toolのPATHに追加される。つまりClaudeが「コマンドとして」呼び出せるようになる。

CW案件の納品前バリデーション用スクリプトを作る。

mkdir cw-plugin/bin

cw-plugin/bin/cw-validate を作る。

#!/bin/bash
# CW案件 納品前バリデーション
echo "=== テスト実行 ==="
npm test --silent 2>&1 | tail -5

echo ""
echo "=== lint チェック ==="
npm run lint --silent 2>&1 | grep -E "error" | head -10

echo ""
echo "検証完了: $(date '+%Y-%m-%d %H:%M')"

実行権限を付ける。ここを忘れると command not found になる。

chmod +x cw-plugin/bin/cw-validate

これで Claude が cw-validate を Bash tool から直接呼べるようになる。SKILL.md に「納品前は cw-validate を実行してください」と書いておけば、Claudeが自動で走らせてくれる。

hooks.json を追加する

hooks.json を追加すると、特定のイベントに反応して処理を走らせられる。TypeScript・JavaScriptファイルを保存するたびに自動フォーマットするhookを作る。

mkdir cw-plugin/hooks

cw-plugin/hooks/hooks.json を作る。

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{
          "type": "command",
          "command": "jq -r '.tool_input.file_path // empty' | grep -E '\\.(ts|tsx|js|jsx)$' | xargs -I{} npx prettier --write {} 2>/dev/null || true"
        }]
      }
    ]
  }
}

Write または Edit ツールが使われるたびに発火する。TypeScript/JSファイルだった場合だけPrettierを走らせる。CW案件でコードを書いている最中、Claudeが何かファイルを編集するたびにフォーマットが整う。

最終的なプラグイン構造

cw-plugin/
├── .claude-plugin/
│   └── plugin.json
├── skills/
│   └── check/
│       └── SKILL.md
├── hooks/
│   └── hooks.json
└── bin/
    └── cw-validate

GitHubにpushして別PCにインストールする

GitHubにリポジトリを作ってpushする。

cd cw-plugin
git init
git add .
git commit -m "init cw-plugin"
git remote add origin https://github.com/yourname/cw-plugin.git
git push -u origin main

別PCやサブPCで使うときは、クローンしてからローカルインストールするだけ。

git clone https://github.com/yourname/cw-plugin.git

Claude Codeを起動してインストールする。

/plugin install ./cw-plugin

これだけ。skills・hooks・bin/ が全部入る。

新しいPCで副業を再開するとき。CLAUDE.mdを書き直す（10分）→プラグインをインストールし直す（5分）→hooksを settings.json に書く（10分）→スクリプトをDropboxからコピー（5分）＝30分

つまずきポイント3つ

bin/ のファイルに chmod +x が必要

これを忘れると Claude が cw-validate を呼んだときに Permission denied か command not found が出る。gitにpushする前に実行権限をつけておく。

chmod +x bin/cw-validate

plugin.json の name は英数字とハイフンのみ

日本語を入れるとスキルが呼び出せなくなる。"name": "cw-plugin" のように英数字だけにする。

/reload-plugins を忘れると変更が反映されない

公式プラグインでも同じだけど、ローカル開発中は特によく引っかかる。SKILL.mdを編集した後や hooks.json を変えた後は必ず /reload-plugins を叩く。
