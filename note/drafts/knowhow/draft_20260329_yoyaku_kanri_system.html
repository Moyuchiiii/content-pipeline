「予約管理、どうにかならないの」と言われた日

高校の同期が個人サロンを始めた。ネイルとまつ毛の店で、予約はぜんぶLINEで受けてると言っていた。

「予約フォームをサイトに置きたいんだけど、どこかに頼んだら30万って言われた。高くない？」

30万。自分もその数字を聞いてちょっと引いた。外注相場的には妥当な金額だけど、開業直後の個人サロンには重い。

「Claude Codeで作れるかも」と思ったのがそのときで、実際にやってみたら3日で完成した。費用はClaude Proの月額3,000円程度だけ。

この記事では、自分が実際にやった工程を全部書く。プロンプトも、詰まったポイントも全公開。

何を作るのかを先に整理する

まず「どんなシステムを作るか」を決める。今回の構成はこれだ。

フロントエンド
HTML + CSS + JavaScript（1ファイルに全部まとめる）
バックエンド
Google Apps Script（GAS）
Googleが無料で提供するサーバーレス環境
データ保存場所
Google スプレッドシート
予約データがここに溜まっていく

なぜこの組み合わせにしたかというと、サーバー代が0円で、オーナーがExcelみたいな感覚でデータを見られるから。「データはどこにあるの」という質問に「Googleスプレッドシートです」と答えられるのが非エンジニアには一番伝わりやすい。

作った機能はシンプルに3つ。

予約フォーム（名前・メール・希望日時・メニュー選択を入力して送信）
管理者ページ（予約一覧・キャンセルボタン付き）
スプレッドシートへの自動記録

これだけあれば個人サロン規模なら十分動く。

「CLAUDE.md」を先に書く。これが全て

いきなりClaude Codeに「予約管理システムを作って」と投げても、まともなものは出てこない。自分も最初の案件でやって失敗した経験がある。

CLAUDE.mdというファイルを先に作ることが、Claude Codeをうまく使う上で一番重要なステップだ。Claude Codeはプロジェクトフォルダにあるこのファイルを起動時に自動で読み込む。「何を作るか」「どんなルールで作るか」をここに書いておけば、以降の指示がブレない。

今回書いたCLAUDE.mdはこれだ。

# 予約管理システム

## 概要
個人サロン向けの予約管理Webアプリ。

## 技術スタック
- フロントエンド: HTML + CSS + JavaScript（1ファイルで完結させる）
- バックエンド: Google Apps Script（GAS）
- データ保存: Google スプレッドシート

## 機能
### 予約フォーム（お客さん用）
- 名前・メール・希望日・希望時間・メニューを入力して送信
- 送信後に「予約を受け付けました」の確認メッセージを表示
- GASのエンドポイントにPOSTで送る

### 管理ページ（オーナー用）
- URLに ?admin=true を付けると管理ページを表示
- 予約一覧をテーブルで表示
- 各予約にキャンセルボタンを付ける
- ステータスを変更できる

## ルール
- シンプルに作る。余計な機能は追加しない
- コメントは日本語で書く
- エラーが起きたときはコンソールに詳細を出力する

これを書くのに30分かかったが、この30分が後の全工程を速くする。CLAUDE.mdがしっかりしていれば、Claude Codeがブレずに動き続けてくれる。

ここから先は有料コンテンツです（¥500）

この先で手に入るもの

  1. Day 1〜3 で実際に使ったプロンプト全文（コピペで使える）
  2. GASスクリプトの作成手順と設定方法（画面キャプチャ付き説明）
  3. 自分が詰まった2つのバグと解決法
  4. この仕組みをCW副業案件に展開した話

Day 1 — フォームを動かすまで

CLAUDE.mdを作ったら、Claude Codeを起動してプロジェクトフォルダを開く。

最初に送ったプロンプトはこれ。

CLAUDE.mdを読んで、予約フォームのHTMLファイルを作ってください。
まずフォームだけ。GASとの連携は後でやります。
index.htmlというファイルで出力してください。

ここで「全部作って」と言わないのがコツだ。一気に全機能を指示すると、Claude Codeがコードを作りきれずに中断したり、設計がブレたりする。フォーム → GAS連携 → 管理者ページと、1つずつ積み上げていく。

返ってきたのは、予約フォームのHTMLだった。デザインもそこそこで、名前・メール・日付・時間・メニューのフォームが入っていた。ブラウザで開くと動く。

日付のカレンダーが過去の日付を選べる状態になっていたので、こう追加指示した。

日付の入力欄で、今日より前の日付を選べないようにしてください。
今日以降だけ選べるよう、min属性で制限してください。

2秒で直った。この調子で細かい修正を追加プロンプトで入れていく。Day 1 の作業時間は約2時間。

Day 2 — GASと繋げる

GAS（Google Apps Script）を使うには、まずGoogleスプレッドシートを一つ作る。

Google Sheetsを新規作成して開いたら、URLの中にあるシートIDをメモしておく。

シートIDの確認
URLの中の「/d/」と「/edit」の間にある文字列がシートID
docs.google.com/spreadsheets/d/【ここがシートID】/edit

シートIDを控えたら、Claude Codeに次のプロンプトを送る。

以下の条件でGoogle Apps ScriptのコードをCode.gsというファイルに書いてください。

シートID: 【自分のシートIDをここに入れる】

doPost(e)関数の仕様:
- POSTリクエストを受け取って、スプレッドシートの最終行にデータを追加
- 受け取るデータ: name, email, date, time, menu
- 追加する列: 受付日時, 名前, メール, 希望日, 希望時間, メニュー, ステータス
- ステータスの初期値は「受付済み」
- 処理が成功したらHTTPステータス200でJSONを返す

doGet(e)関数の仕様:
- スプレッドシートの全データをJSON形式で返す

その他:
- CORSヘッダーを設定する（どのドメインからでもアクセスできるように）
- 文字コードはUTF-8
- エラーの場合はスタックトレースをJSONに含めて返す

コードが出てきたら、スプレッドシートのメニューから「拡張機能」→「Apps Script」を開いてペーストして保存する。

次にデプロイ。「デプロイ」→「新しいデプロイ」→「種類」で「ウェブアプリ」を選ぶ。

デプロイ時の設定（重要）
「次のユーザーとして実行」→「自分（自分のGoogleアカウント）」
「アクセスできるユーザー」→「全員」

デプロイするとURLが発行される。これをHTMLのフォーム送信先として設定する。

index.htmlのフォームの送信先を以下のGAS URLに変更してください。

GAS URL: 【発行されたURL】

送信が成功したら「予約を受け付けました」と表示してフォームをリセット。
エラーの場合は「送信に失敗しました。もう一度お試しください」を表示してください。

詰まり1 — CORSエラー

フォームを送信したら、ブラウザのコンソールに「CORS error」が出た。GASにアクセスできていない状態。

原因はデプロイ設定の「次のユーザーとして実行」を間違えていたこと。「ウェブアプリにアクセスしているユーザー」にしていたのを「自分（自分のGoogleアカウント）」に変えて再デプロイしたら解決した。

デプロイ設定を変えたら必ず「新しいバージョンとしてデプロイ」を選ぶこと。「既存のバージョンを更新」では設定が反映されないことがある。

詰まり2 — 日本語の文字化け

予約データがスプレッドシートに入ったが、名前とメニューが文字化けしていた。GASのdoPost関数でUTF-8デコード処理が抜けていた。

エラーメッセージをそのままClaude Codeに貼って聞いた。

スプレッドシートに保存された名前とメニューが文字化けしています。
GASのdoPost関数にUTF-8のデコード処理を追加してください。
現在のコードは以下です:
【Code.gsの中身をここに貼る】

コードが更新されて文字化けが解消した。「コードをここに貼る」という作業が重要で、Claude Codeはコンテキストがあるほど精度が上がる。

Day 3 — 管理者ページとデプロイ

最終日は管理者ページを追加して、Webに公開して完了。

index.htmlに管理者ページを追加してください。

表示条件: URLに ?admin=true が付いている場合だけ表示する

管理者ページの仕様:
- GASのdoGetからデータを取得して一覧表示
- テーブルの列: 受付日時 / 名前 / メール / 希望日時 / メニュー / ステータス
- 各行に「キャンセル」ボタンを付ける
- キャンセルボタンを押すとステータスを「キャンセル済み」に変更してテーブルを更新
- 受付済みは白背景、キャンセル済みはグレー背景で表示

管理者ページが完成したら、Netlifyで公開する。

Netlifyでの公開手順
1. netlify.com でアカウント作成（無料）
2. 「Add new site」→「Deploy manually」
3. index.html が入ったフォルダをドラッグ＆ドロップ
4. 数秒でURLが発行される

「https://〜.netlify.app」というURLでアクセスできるようになる。オーナーに送ったら「これで十分！」と即日使い始めてくれた。管理者ページのURLは「〜.netlify.app?admin=true」で、オーナーにだけ共有する。

Day 3 の作業時間は約1.5時間。3日間の合計で7時間かかった。
