先週の木曜日、午後2時から3時半まで統計学の講義があった。

その間、クライアントから「仕様を少し変えたい」というメッセージが届いた。

以前なら「帰ってからやります」とだけ返信して、その間は何もできなかった。でも今回は違って、スマホのTelegramを開いて、Claude Codeに向かってタスクを投げた。

「〇〇のAPIのエンドポイントを/v1から/v2に変更して、型定義も合わせて直して」

90分後に講義が終わって確認したら、コードの修正が完了していた。

これが Claude Code Channels を使い始めてから、副業の働き方が変わった瞬間の話。

Claude Code Channels ってなに？

3月20日にAnthropicがリリースした新機能で、TelegramやDiscordなどのチャットアプリから、PC上で動いているClaude Codeに指示を出せるようになった。

仕組みとしては、MCPサーバーがTelegramのメッセージをClaude Codeセッションに転送してくれるイメージ。PCのローカルファイル、git、MCPツール、全部使える状態でスマホから操作できる。

Dispatch（スマホからPC操作を遠隔操作）に近いけど、Channelsはより「チャットで非同期にタスクを投げる」感じが強い。「帰るまでにこれやっといて」って感覚で使える。

なぜ試してみたか

自分がクラウドワークスで受けてる業務自動化の案件は、だいたいこんな流れ。

  1. クライアントから要件が来る
  2. Claude Codeで実装
  3. テスト・確認
  4. 納品

問題は2のステップで、どうしてもPCの前に座る必要があった。大学の講義中や移動中は「案件が止まる時間」だったわけで、正直もったいないと思ってた。

Channels を見たとき、「これ使えばその時間を潰せるかも」と思ってすぐ試した。

セットアップした話

まず最初にぶつかった壁が、Bunが必要ってこと。

公式ドキュメントを読むと「Bun runtime required」と書いてある。最初Node.jsで動かそうとしたら何も起きなくて、10分くらい悩んだ。

curl -fsSL https://bun.sh/install | bash

これでBunを入れたら動いた。

あとはBotFatherでTelegramのボットを作って（/newbot で作れる）、Claude Codeを起動するとき --channels フラグをつけてプラグインを読み込む。

ペアリングはTelegramでボットにメッセージを送ると6文字のコードが返ってくるので、Claude Code側で入力すると接続完了。

ここで絶対忘れてはいけないのがセキュリティ設定。

デフォルトだと、誰でもそのボットに話しかけてペアリングコードを取得できてしまう。設定後すぐに policy allowlist に切り替えないと、自分以外の人がClaude Codeを操作できる状態になる。

自分は確認するまでの3分くらいそのままにしてしまったので、冷や汗かいた。

実際に副業でどう使ったか

クラウドワークスで受けていた「Pythonでスクレイピングツールを作る」案件で試した。

案件の作業は全部Claude Codeでやってたので、Channelsを繋ぐだけで外からも操作できる状態になった。

大学に行く前にClaudeを起動して --channels をつけておく。あとはTelegramから指示を投げるだけ。

Before: 外出中は案件作業がゼロ。1日の有効稼働時間: 帰宅後の夜3〜4時間のみ
After: 移動中・講義中もタスクを投げられる。1日の有効稼働時間: 朝から夜まで断続的に6〜8時間分

ある日の実績:

朝8時（通学中）: 「昨日のスクレイピングロジックにエラーハンドリングを追加して」→ 大学着いたら完了
昼12時（ランチ中）: 「テストコードを追加して全部通してみて」→ 食事終わったら結果がTelegramに届く
午後2〜4時（講義中）: 「クライアントの追加要件、ファイルに追記しておいて」→ 講義後に確認

つまずいたこと

PCのスリープ問題

外出中にPCがスリープに入ると、Claudeのセッションが切れてTelegramからの指示が受け付けられなくなる。しばらく気づかなかった。

暫定対策として、Caffeineというアプリでスリープを無効化して外出するようにした。これで今のところ安定してる。

指示が曖昧だと困る

Channelsで投げるタスクは、その場でClaudeと対話しながら修正できないので、指示を最初からちゃんと書く必要がある。

「いい感じにして」みたいな指示はNG。「〇〇ファイルの△△関数を〜の仕様に変えて、テストも追加して」くらい具体的に書かないと、全然違う方向に進んでたり、Telegramに「どうすればいいですか？」と返ってきたりする。

非同期で操作する分、指示の精度がより重要になる。これが一番の学びだった。

ここまで無料。この先では、副業案件での具体的な使い方3パターン（コマンド全文付き）と、長時間タスクを安全に回すための設定を全部書いた。

この先で手に入るもの

  1. セットアップコマンド全文（Telegram接続〜セキュリティ設定まで）
  2. 副業案件での実践ユースケース3選（クラウドワークス向け）
  3. 長時間タスクを安全に回す設定方法
  4. 「Telegramへの指示文テンプレート」コピペOKで使える版

セットアップコマンド全文

順番通りにやれば30分かからずに動く。

Bunのインストール（まだ入れてない場合）

# Mac/Linux
curl -fsSL https://bun.sh/install | bash

# インストール後、ターミナルを再起動して確認
bun --version

Telegramボットの作成

Telegramで @BotFather を開いて:

  1. /newbot を送る
  2. ボット名を入力（表示名、例: "MyClaude Bot"）
  3. ユーザー名を入力（_bot で終わる必要あり、例: "my_claude_work_bot"）
  4. トークンが返ってくる（123456789:ABC... の形式）→ コピーしておく

プラグインの設定と起動

# [YOUR_BOT_TOKEN] を実際のトークンに置き換える
TELEGRAM_BOT_TOKEN=[YOUR_BOT_TOKEN] claude --channels plugin:telegram@claude-plugins-official

ペアリングとセキュリティ設定

  1. Telegramで自分のボットにメッセージを送る（内容はなんでも）
  2. ボットから6文字のペアリングコードが届く
  3. Claude Code側で以下を実行:
/telegram:access pair [6文字のコード]

# allowlistモードに切り替え（必須）
/telegram:access policy allowlist

# 自分のTelegram IDをallowlistに追加
# IDは @userinfobot に「/start」で確認できる
/telegram:access add [TelegramユーザーID]

これで完成。Telegramからメッセージを送ってClaude Codeが返答したら成功。

副業案件での実践ユースケース3選

1. 「帰るまでにこれやっといて」型

外出前にタスクの全量を把握して、優先度順に投げる。自分がよく使う指示のテンプレ:

今日の外出中（14:00〜18:00）のタスクです。
完了したらTelegramに報告してください。

【タスク1・優先度高】
src/scraper.pyのfetch_data()関数にリトライロジックを追加。
最大3回、1秒インターバルで。エラー内容もログに記録すること。

【タスク2・タスク1完了後】
tests/test_scraper.pyにリトライロジックのテストを追加。
ネットワークエラーのモックを使って、3回リトライ後に失敗するケースを必ずカバーすること。

全部まとめて1通で送ると、Claude Codeが順番にこなしてTelegramで進捗を報告してくれる。

2. 「結果だけ教えて」型

テストの実行やスクレイピングの試しrunなど、長時間かかる処理を回して結果だけ受け取る。

tests/ディレクトリ配下のテストを全て実行して。
完了したら結果のサマリーをTelegramに送って:
- 合計テスト数
- 成功/失敗数
- 失敗したテストのファイル名とエラーメッセージ

30〜40分かかるテストスイートも、移動中に投げれば到着した頃には結果が来てる。

3. 「クライアントの追加要件をすぐ反映」型

クライアントから追加要件が来たとき、PCを開かずにその場で対応を開始できる。

クライアントから以下の追加要件が来ました:
「検索結果を最大50件ではなく100件取得したい。パフォーマンスへの影響も確認して」

以下を対応してください:
1. src/config.pyのMAX_RESULTSを100に変更
2. 100件取得した場合の実行時間を計測（既存の10件と比較）
3. 結果をdocs/performance_test.mdに記録

完了したらTelegramで結果を報告してください。

「確認してから折り返します」の回数が大幅に減った。

長時間タスクを安全に回す設定

PCのスリープ対策

Macの場合、Caffeineというフリーアプリを使ってスリープを無効化するのが手軽。電力消費の問題があるので、Channelsを使う日はACアダプタ接続が前提。

Claudeへの事前指示テンプレ（これをコピペして使う）

このタスクは外出中（XX時間程度）に実行します。
途中でエラーが発生したら、解決策を試みた上でTelegramに報告してください。
重大なエラーでどうにもならない場合は作業を停止して状況を報告してください。
自己判断でリスクのある変更（ファイルの削除・データベースの変更等）はしないでください。

Claudeに「自律的に動いていい範囲」を最初に明示しておくと、帰宅後に予想外の変更が起きていた、みたいな事故が起きにくくなる。

まとめ

Claude Code Channels を使い始めてから、副業の「稼働できる時間」が体感で1.5〜2倍になった。

以前は夜の3〜4時間だけだったのが、移動中・昼休み・講義の合間も含めて、一日6〜8時間分の処理を流せるようになった。

自分がやることは変わらない。指示を書いてTelegramに投げるだけ。あとはClaude Codeがやってくれる。

セットアップは30分もかからない。Channels、試してみてほしい。

次回は、Computer Useを実際のクラウドワークス案件（画面操作が必要なタスク）に使ってみた体験記を書く予定。Channelsで作業を自動投入 + Computer Useで画面操作まで自動化すると、自分がやることがさらに減るのでその話を。
