# ClaudeのComputer UseにCW案件を探させたら、2時間が40分になった話

## 先週の記事で書いた「案件リサーチ3時間」の話

先週書いた「週10時間が3時間になった話」の中で、副業の内訳をこう書いた。

- クライアントへのメッセージ確認・返信: 約2時間
- 進捗レポート作成: 約3時間
- **案件リサーチ・情報収集: 約3時間**
- ファイル整理・納品準備: 約2時間

あの記事ではレポート作成（35分→8分）やファイル整理（90分→20分）の話が中心になって、案件リサーチの詳しい手順は書けなかった。

今回はそこに絞る。Computer UseにCWの案件探しを丸ごと任せてみた話。

---

## CW案件探しの「地味な面倒さ」

副業でCWを使ってる人なら、この流れ、わかるんじゃないかと思う。

CWを開く → 検索条件を入れる（業種・予算・期間）→ 一覧をスクロールする → 気になるものをクリックして開く → 「これは合わない」で閉じる → 次を開く。

これを繰り返して、1週間で「応募できそう」な案件を3〜5件見つける。所要時間は週2時間くらいだった。

長いとは言えないけど、毎週同じ作業をやってる感がずっとあった。条件も毎回同じ。探し方も毎回同じ。なのに手動でやってた。

Computer Useに任せたのは、前回「情報収集+まとめはComputer Useが得意」とわかったから。CWの画面操作も同じ原理でいける気がした。

---

## Computer Useに頼んだ手順（全部書く）

### ステップ1: 案件スキャン

まずClaude Desktopを開いて、こんな指示を出した。

> 「クラウドワークスにアクセスして、PythonまたはExcel自動化のカテゴリで新着案件を探してください。予算3万円以上・期間1ヶ月以内のものを優先して。見つけた案件を、案件名・予算・応募数・クライアント評価の一覧にしてください」

Computer Useがブラウザを起動してCWを開いて、検索して、1件ずつ読んで、表にまとめてくれた。

自分がやると20〜30分かかる作業が7〜8分で完了した。

### ステップ2: 案件の絞り込み

一覧が出てきたら、次にClaudeに評価させた。

> 「この案件一覧を見て、自分のスキルに合いそうな順に並べ替えてください。自分はPythonでのWebスクレイピング・業務自動化スクリプト作成・データ整理が得意で、CWでの副業歴は3ヶ月です」

スキルセットを渡すと、優先順位と「なぜこの案件が合いそうか」の理由まで出してくれる。自分だと流し読みで見落としてた条件（「エクセルマクロ経験者優遇」とか「週10時間以上稼働できる方」とか）もちゃんと拾ってくる。

### ステップ3: 応募文の生成

気になる案件が2〜3件に絞れたら、Claude Codeで応募文を作った。

> 「この案件の要件を読んで、自分が受注できる提案文を書いて。クライアントの本当の悩みを推測して、どう解決するか具体的に。300文字以内で、謙虚すぎず、でも押しつけがましくもない文体で」

案件のURLか、案件内容のテキストを貼ると、2〜3分で提案文が出てくる。

### ステップ4: 確認 + 送信（ここは自分がやる）

送信ボタンは自分で押した。

Computer Useで入力フォームへの入力まではやってもらえるけど、「最終確認してから送る」というワンクッションは自分でやるようにした。内容をざっと読んで問題なければ送信、という流れ。

## 実際の結果

**Before:** 案件探し2時間 + 応募文作成0.5時間 = 計2.5時間/週（3〜5件）

↓ Computer Use + Claude Code 導入後

**After:** スキャン+絞り込み30分 + 応募文確認・修正10分 = 計40分/週（同じ3〜5件）

2.5時間→40分。削減率84%。

件数は変わってないけど、見落としが減った感覚がある。毎回「見逃してる案件があるかも」という焦りがあったけど、Claudeがスキャンした後は「今週の候補はこれで全部」と思えるようになった。

---

## つまずいたこと2パターン

### CWのUIが変わると動かなくなる

Computer UseはCWの画面構造を読みながら操作してるので、CW側でUIが変わると誤操作が起きることがある。検索フィルターのレイアウトが変わった時に「条件が見つかりません」というエラーになったことが1回あった。

対処は「もう一度試して」か「手動で絞り込んだ一覧ページのURLをClaudeに読ませる」。完全に自動というよりは、うまく動かない時に人間が補助するイメージ。

### 応募文が長すぎる問題

Claude Codeに「300文字以内」と指定しても、たまに500文字近い応募文が出てくる。後から「200文字に圧縮して」と指示すれば修正できるけど、1手間増える。

今は「300文字以内、でも案件の具体的な要件には必ず触れること」という指示の方がバランスがいい。

## 全体の組み合わせはこうなってる

副業のCWワークフローを図式化するとこんな感じ。

**案件スキャン（Computer Use）**
CWを自動で見回って案件一覧を取得。7〜10分。

**案件の絞り込み・評価（Claude本体）**
スキルとのマッチ度で並べ替え。3〜5分。

**応募文作成（Claude Code）**
案件ごとにカスタマイズした提案文を生成。2〜3分/件。

**確認 + 送信（自分）**
内容を一読してから送信ボタンを押す。2〜3分/件。

Computer Use・Claude Code・自分、という役割分担がうまく噛み合ってる感じがしてる。

---

## やってみて分かったこと

最初は「CWの画面をAIに操作させて本当に使えるの？」という疑問があった。

実際は、CWのUIがシンプルな分、Computer Useが操作しやすかった。複雑なアプリよりも、定型的な検索+一覧閲覧の方が向いてる。

「全部自動」じゃなくて「9割任せて、最後だけ確認する」というスタンスで使うのが今のところ一番うまくいってる。

副業でCWを使ってる人には試してみる価値がある。

次回は、今回使ったプロンプト（案件スキャン用・応募文生成用）の全文とカスタマイズの仕方を書く予定。
