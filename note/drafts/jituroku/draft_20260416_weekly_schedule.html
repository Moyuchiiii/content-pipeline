副業やりたいけど、授業もあるしバイトもあるし、時間なくない？

自分もずっとそう思ってた。

副業を始めた最初の月、気づいたら週20時間以上作業してた。授業の合間にカフェでPC開いて、夜も寝る前にギリギリまで。それで月5.5万円。時給に換算すると688円。近所のコンビニバイトより安い。

「これバイトした方が早くない？」って本気で思った。

でも今は、週12時間の作業で月14万円になった。時給は約2,900円。何が変わったかというと、AIに「自分がやらなくていい作業」を全部渡すようになった。

この記事では、月14万を超えた月のリアルな1週間を全部見せる。授業がある日もない日も、どう時間を使って、AIに何を任せてるか。

月曜日 — 授業4コマの日

一番忙しい曜日。でもスキマ時間だけで副業2.5時間分が回る。

7:00 起床。布団の中でスマホを開く。前の晩にRemote Control（スマホからPCのAIアシスタントを遠隔操作できる機能）で「CWの新着案件をチェックして、自分のスキルに合うやつをリストアップして」と投げてた。

スマホに5件のリストが届いてる。布団の中でざっと見て、2件にチェック。

7:30〜8:30 通学中。電車の中でブラウザ版Claude（スマホでも使えるチャット型AIアシスタント）を開いて、さっきの2件の案件詳細をコピペ。「この案件に応募する提案文を書いて。自分のプロフィールはこれ」と投げる。

駅に着くまでに2本の提案文ドラフトが出来上がってる。自分の言葉に少し直して、CWから応募。通学時間が「提案文タイム」に変わった。

8:30〜12:00 授業。副業のことは一切考えない。ここは切り替えが大事。

12:00〜12:15 昼休み。進行中の案件でクライアントから修正依頼が来てた。ブラウザ版Claudeに「この修正依頼の対応方針を整理して」と投げる。5分で方針が返ってくるので、クライアントに「確認しました、本日中に対応します」と返信。

12:30〜14:30 授業。

15:00〜17:00 大学近くのカフェ。ここが今日のメイン作業時間。Claude Code（PCで使うAIアシスタント。指示を出すとコードを書いたり作業を自動化してくれるツール）を開いて、さっきの修正依頼を処理する。

「クライアントの修正依頼はこれ。対応して」とClaude Codeに投げて、出てきたものを確認。OKならそのまま納品ファイルにまとめる。自分がやるのは「指示出し」と「最終確認」だけ。

2時間で修正対応＋次の案件の下準備まで終わった。

月曜の副業時間 → 約2.5時間（通学30分＋昼休み15分＋カフェ2時間）

AIツールの使い分け

自分が使ってるのは主に3つ。場所と「どれくらい時間があるか」で分けてる。

ブラウザ版Claude（claude.ai）
スマホでもPCでも使える。通学中・昼休みの「10分以内で終わる作業」に。提案文作成、修正方針の整理、案件の分析。チャット感覚で指示するだけ
Claude Code
PCで使うAIアシスタント。カフェや自宅で「30分以上の集中作業」に。スクリプト作成、Webアプリ実装、テスト、ドキュメント生成。重い作業はこっち
Remote Control
スマホからPCのClaude Codeを操作する機能。「寝てる間・移動中にやっておいてほしい作業」に。前の晩に指示を出して、朝結果を確認するだけ

判断基準はシンプルで、「今スマホしかない？→ブラウザ版Claude」「PCある＋30分以上ある？→Claude Code」「今は無理だけど明日までにやっておきたい？→Remote Control」。これだけ。

スキマ時間×AI 3つのコツ

「投げる」と「確認する」を完全に分ける

スキマ時間（通学中・昼休み）は「AIに指示を投げる」だけ。結果の確認と修正は、まとまった時間（カフェの2時間）でやる。

これを意識し始めてから、15分のスキマ時間が本当に使えるようになった。以前は「15分じゃ何もできない」と思ってたけど、「投げるだけ」なら5分で終わる。残り10分は次の授業の準備に使える。

前の晩に「明日やること」を仕込む

寝る前の5分でRemote Controlに翌日のタスクを投げておく。朝起きたら結果が届いてる状態。

これだけで朝の30分が浮く。その30分で通学中に提案文を書ける。1日のスタートが変わった。

全休の日に「重い作業」を全部集める

授業がある日はスキマ時間だけ。コードを書いたりテストしたりの重い作業は、全休の水曜にまとめる。

最初はこの切り分けができなくて、授業日に無理やり重い作業をしてた。授業に集中できないし、作業も中途半端になるし。切り分けるようになって、両方の質が上がった。

つまずいた3つの話

授業中にClaude開いて、先生に見つかった

3コマ目の授業中に、どうしても気になる案件があってスマホでClaude開いたら、先生に「何見てるの？」と聞かれた。

本当に恥ずかしかった。以来、授業中は絶対にやらないと決めた。冷静に考えると、授業中に中途半端にやるより昼休みに集中してやる方がトータルで速い。

疲れてる日にAIの出力をそのまま納品した

水曜の夜、作業で疲れて「もう確認めんどくさい」とAIの出力をそのまま納品したことがある。案の定、クライアントから「ここ違います」「ここも直してください」と3回修正が入った。

最初に15分確認してたら済んだものが、修正対応で2時間かかった。疲れてる日は無理しないで寝る。翌朝確認した方が100倍マシ。

浮いた時間を「自由時間」にして収入が変わらなかった

AIを導入した最初の2週間、作業時間は確かに減った。でも収入は変わらなかった。

理由はシンプルで、浮いた時間をそのまま「自由時間」にしてたから。AIで時間を浮かせた→遊んだ→収入同じ。当たり前。

浮いた時間で「案件を増やす」「単価の高い案件に応募する」をやって、初めて収入が上がった。AIは時間を作ってくれるけど、その時間をどう使うかは自分で決めないといけない。これが一番大事な学びだった。

「時間がない」は、切り分け方を知らなかっただけだった

副業で大事なのは「何時間作業するか」じゃなくて「どの作業を自分でやるか」だった。

AIに任せられる作業を全部渡す。自分がやるのは「判断」と「確認」だけ。この切り分けで週20時間が12時間になって、収入は2.5倍になった。

授業がある大学生でも、スキマ時間にAIに「投げる」だけなら5分でできる。エンジニアじゃなくても、コードが書けなくても、AIに「これやって」と日本語で指示するだけ。自分がやってるのは本当にそれだけ。
