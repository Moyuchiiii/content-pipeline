今月（3月）の売上をCWのダッシュボードで確認したとき、合計が14万円を超えてた。

2ヶ月前の2月上旬、副業収入は0円だった。Claude Codeは名前だけ聞いたことある、くらいのツールで、実際に動かしたことはなかった。

2ヶ月でどう変わったかを、ここでまとめておく。なるべく正直に。

最初の1ヶ月 — ほぼ試行錯誤だった

2月中旬にClaude Codeを使い始めて、最初の2週間は正直なんも稼げなかった。

「Claude Codeすごい」という情報を見て触り始めたけど、何を使って何を作ればクラウドワークス（CW）で案件が取れるかが全然わかってなかった。ターミナルでコマンドを叩いてClaudeが動いてる様子を眺めてた、みたいな期間。

最初の受注ができたのが2月末頃で、Pythonでスプレッドシートのデータを整形して別フォーマットに変換する業務自動化スクリプトの案件だった。工数4〜5時間、報酬5,000円くらい。時給換算で1,000円ちょいで、数字は全然良くなかった。

でも「受注できた」が積み重なって、2月トータルは5万5千円になった。案件単価1〜2万円くらいのスクリプト系を3件受注した計算になる。

この1ヶ月目でわかったのが、「受注できるかどうか」と「効率よく稼げるかどうか」は全然別の話だということ。受注の流れはわかってきたけど、1案件あたりの作業時間が多すぎて、収益的にはまだきつかった。

3月で変わった3つのこと

Auto Modeで「コードを全部読む」をやめたら作業時間が3分の1になった

3月に入って最初に変えたのが、Claude Codeとの向き合い方だった。

最初は「Claudeが書いたコードを全部理解してから進める」スタイルでやってた。コードを読んで、ロジックを確認して、テストして、という流れ。情報系なのでコードは読めるけど、それをやってると1案件に8〜10時間かかってた。

Auto Modeを使い始めてから変わった。Auto Modeというのは、人間の承認を待たずに複数のステップを連続で実行できる機能。ファイルの読み取り→分析→コード生成→保存、という一連の流れを自分が介入せずに進める。「コードを一行一行確認する」作業が大幅に減った。

Auto Mode導入前
要件確認 → Claudeに指示 → コード確認 → 手修正 → テスト → 再確認... を繰り返して平均8〜10時間
Auto Mode導入後
要件確認 → Claudeに渡す → 完成物を確認 → テスト → 納品で平均2〜2.5時間

同じ時間で抱えられる案件数が月2〜3件から4〜5件に増えた。「Claude Codeを使う」から「Claude Codeに任せる」に変わった感覚があった。

業務自動化からWebアプリUIにステップアップして単価が上がった

3月下旬に、それまでとは種類の違う案件を取った。

LP改修＋管理画面UIの開発案件で、金額は6万5千円。自分はWebフロントエンドをほぼ触ったことがなかったので応募するか迷ったけど、「Claude Codeがいるから大丈夫」という謎の確信があって応募した。

設計書をClaude Codeに食わせたら20分くらいで基本的なUI構造が出てきて、それをベースに修正指示を出し続けるだけで実装が進んだ。3日くらいで完成した。

その後もう一件、タスク管理ツールのUI案件（4万5千円）も受注して、この2案件だけで11万円。他の案件と合わせて3月トータルが14万円を超えた。

1ヶ月目から2.5倍になったのは、単価の高い種類の仕事に広げられたことが一番大きかった。

Remote Controlで「PCの前にいる時間」と「稼ぎの時間」を切り離した

もう一つ大きかったのが、Remote Controlを使い始めたこと。スマホからClaude Codeに指示が出せるようになって、大学の移動中に「この案件のコンポーネント、こういう仕様で修正しておいて」と投げれば、帰ったときには終わってる状態になった。

1週間で「PCの前に座って作業している時間」が10時間から3時間に減った。残りの7時間はClaudeが自分の代わりに動いてる。

「自分の時間を切り売りする副業」から「時間を切り離せる仕事の仕方」に変わった感覚があった。これが体感として一番大きかった変化だったかもしれない。

正直に書いておくと失敗もある

順調に見えるかもしれないけど、最初の2週間はほぼ何もできなかった。

Claude Codeを起動して雑な指示しか出せなくて、出てきたコードがなぜそうなってるか理解できなくて、手作業でやった方が早いと感じてた。「Claude Codeに慣れた」と思えるまでに3週間近くかかった。今でも複雑な要件の案件だと設計書レベルで詳細に書かないとズレたものが出てくる。

あと、Auto Modeで全部任せると「自分のスキルが上がってるかどうか」が見えにくくなる問題もある。どこを自分でやってどこをClaudeに任せるか、今もちょっと模索中。

2ヶ月の数字まとめ

1ヶ月目（2月）
売上 ¥55,000 / 受注案件 3件 / 業務自動化スクリプト中心
2ヶ月目（3月）
売上 ¥140,000 / 受注案件 6件 / 業務自動化4件 + WebアプリUI2件

1ヶ月目から2ヶ月目で2.5倍になった要因は2つだと思ってる。WebアプリUI案件という単価の高い種類の仕事に広げられたことと、Auto Modeで1案件あたりの作業時間を3分の1に削減できたこと。

今は案件4件を並行して抱えてて、1日あたりの稼働は2〜3時間くらい。2ヶ月前には全く想像してなかった状態になった。

これから始める人へ

最初の3週間は作業時間に対してリターンが全然合わない。それは多分みんなそうで、Claude Codeの使い方を覚えるのに時間がかかるから。

諦めそうになったのは1回あったけど、最初の受注が取れてから一気に変わった。金額は小さくていいから、まず1件受けてみることが一番の近道だと思う。
