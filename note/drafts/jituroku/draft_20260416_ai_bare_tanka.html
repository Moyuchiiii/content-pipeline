AI副業、まだ"AIに書かせたまま"納品していませんか？

AI副業の文字単価が崩壊してる、ってよく見かけるようになった。

実際、クラウドワークスの案件を見てると「AI利用OK・文字単価0.1円」みたいなのがゴロゴロある。以前は0.5〜1.5円が普通だったのに、今は0.02〜0.1円まで落ちてる案件も珍しくない。「AIで書けるんだから安くていいでしょ」って値下げ圧力がすごい。

でも、自分はAIを使って副業してて月14万もらえてる。

同じ「AIを使った副業」なのに、なんでこんなに差がつくのか。2ヶ月やってきてわかったことを全部書く。

最初は自分もAI丸投げだった

副業を始めた最初の2週間、自分もChatGPTに「この案件の記事書いて」って投げて、出てきた文章をほぼそのまま納品してた。

結果、修正依頼が毎回来た。

クライアントからのフィードバック（実際に来たやつ）
「全体的に内容はいいんですが、もう少し具体的なエピソードが欲しいです」
「文章のトーンが少し硬いので、もう少しカジュアルにお願いします」

要は「AIっぽい」って遠回しに言われてたんだと思う。直接「AIで書きましたか？」とは聞かれなかったけど、毎回修正が入るってことは品質が足りてなかった。

で、低単価の案件ばかりだから修正に時間かけると時給が500円切る日もあった。

「バレてない人」と「バレてる人」の差

転換点は、自分のnote記事を書くときに気づいたことだった。

自分はnoteの記事もAI（Claude）を使って書いてる。でもスキ数が20を超える記事もあるし、「AIっぽい」と指摘されたことは1回もない。

一方、CWの納品物は毎回修正が入る。

何が違うのか考えた結果、差は3つだった。

文字単価0.1円の人 → AIの出力をそのまま納品
月14万の自分 → AIの出力に「自分」を入れてから納品

工夫① 一次体験を必ず入れる

AIが絶対に書けないものがある。「自分だけが体験したこと」。

たとえば「業務効率化ツールの使い方」を書く案件なら、AIに全体の構成を作らせた後、自分が実際にそのツールを触って「ここで詰まった」「この設定を変えたら動いた」っていう体験を1つ以上入れる。

AIだけの文章
「このツールを使うと、作業時間を大幅に短縮できます。設定も簡単で、初心者でもすぐに使い始められます」
一次体験を入れた文章
「最初に設定画面を開いたとき、項目が多すぎて正直閉じたくなった。でも実際にいじるのは3箇所だけで、自分は10分で終わった。1週間使ってみたら、毎日の報告書作成が40分→15分になった」

下の方が「この人、本当に使ったんだな」って伝わる。AI判定ツールも、体験談の有無を人間判定の重要基準にしてる。

工夫② AI定型フレーズを全部消す

AIが書いた文章には「AIの癖」がある。これを残したまま納品すると一発でバレる。

自分が毎回チェックして消してるフレーズはこんな感じ。

消すフレーズ（代表例）
「〜と言えるでしょう」→ 断言するか消す
「〜ではないでしょうか」→ 「〜です」にする
「重要なのは」→ 消して具体例を先に出す
「様々な」→ 具体的に3つ挙げる
「積極的に活用」→ 「使ってください」
「シームレスに」→ 何が繋がってるか書く

これを全部消すだけで、文章のAI感がかなり減る。慣れると5分で終わる作業だけど、効果はでかい。

工夫③ 最初からAIに「自分の書き方」を教える

一番効いたのがこれ。AIに文章を書かせるとき、自分が過去に書いた文章をプロンプトに入れて「この文体で書いて」と指示する。

自分の場合、Claudeに過去のnote記事の一部を読ませて「このトーンで」と頼んでる。そうすると最初から「自分っぽい文章」が出てくるから、修正が少なくて済む。

プロンプトの例（自分が実際に使ってるやつ）
「以下の文体サンプルを参考に書いてください。語尾は『〜した』『〜だった』系で統一。1段落は2〜3文。専門用語は初出で1文説明。体験談を必ず1つ以上含めてください」

これだけで出力の質が全然変わる。最初は「プロンプトに時間かけるくらいなら自分で書いた方が早い」と思ってたけど、1回作ったプロンプトは使い回せるから、案件が増えるほど楽になる。

  1. 自分が実際に使ってる「AI定型フレーズ削除チェックリスト」全文
  2. 「声クローン」プロンプトのテンプレート（コピペで使える）
  3. 納品前の品質チェック3ステップ（Before/After実例つき）
  4. CWのAIポリシーの実態と「書いていいこと・ダメなこと」

【有料】AI定型フレーズ削除チェックリスト

自分が毎回の納品前に確認してるリスト。全部で15項目あるけど、慣れると3分で終わる。

曖昧・逃げ表現（必ず消す）

「〜と言えるでしょう」
→ 断言するか、根拠を書く
「〜ではないでしょうか」
→ 「〜です」に変えるか、理由を添える
「〜かもしれません」
→ 確認できないなら「〜は未確認」と正直に
「〜と考えられます」
→ 「〜です」で断言

過剰な褒め言葉（数字に置き換え）

「非常に重要」
→ 「〇〇の点で重要」と具体化
「画期的な」「革新的な」
→ 数字で示す（「従来の3倍速い」等）
「効率的に」「スムーズに」
→ 「40分が5分になった」みたいに数字で

AI感の強い接続詞（段落頭で使わない）

「したがって」「このように」「以上のことから」
→ 段落頭には使わない。どうしても繋げたいなら「で、」「結局」「要は」に変える
「また、〜」が3回以上連続
→ 構造を見直す。列挙したいなら箇条書きにする

【有料】声クローンプロンプト テンプレート

自分がCW案件の納品物を書くときに、Claudeに最初に渡してるプロンプトがこれ。

以下の条件で文章を書いてください。

【文体ルール】
- 語尾は「〜した」「〜だった」「〜だと思う」系で統一
- 「です/ます」は全体の20%以下に抑える
- 1段落は2〜3文。4文以上にしない
- 専門用語は初出で1文説明。2回目以降は略称OK
- 「まず」「次に」「最後に」の三段構成は使わない

【必ず入れるもの】
- 体験談を最低1つ（「自分がやってみたら〜だった」形式）
- 数字を最低2つ（時間・金額・%のどれか）
- つまずきポイントを1つ（失敗or困ったこと）

【絶対に使わないフレーズ】
- 「〜と言えるでしょう」
- 「〜ではないでしょうか」
- 「重要なのは」
- 「積極的に活用」
- 「様々な」「幅広い」
- 「シームレスに」「スムーズに」
- 「いかがでしたか？」
- 「参考になれば幸いです」

このプロンプトを案件ごとにコピペして、案件の内容を足すだけ。1回作れば使い回せる。

【有料】納品前の品質チェック 3ステップ

自分が納品ボタンを押す前に毎回やってること。全部で10分くらい。

ステップ1 — AI定型フレーズスキャン（3分）

上のチェックリストを見ながら、文章全体を1回通して読む。引っかかるフレーズがあったら消すか書き直す。

ステップ2 — 「体験」の確認（2分）

記事の中に「自分が実際にやったこと」が最低1つ入ってるか確認する。入ってなかったら追加する。「3日使ってみた結果」「最初に設定するとき困ったこと」みたいな短いエピソードでいい。

ステップ3 — 声に出して読む（5分）

これが一番効く。声に出して読んで、詰まった箇所は直す。AIの文章は目で読むと自然に見えても、声に出すと不自然な箇所がわかる。「ここ、自分は絶対こう言わないな」って思ったら書き直す。

CWのAIポリシー — 実際のところ

よく「クラウドワークスはAI禁止じゃないの？」って聞かれるけど、実際は違う。

CWの公式AIポリシーでは、AI利用自体は禁止していない。ただし2つのルールがある。

ルール1
AI利用の可否はクライアントに事前確認する。利用していることを意図的に隠さない
ルール2
クライアントの機密情報をAIツールに入力するときは情報の取り扱いに注意する

つまり「AIを使うこと」が問題なんじゃなくて、「AIを使ったことを隠して低品質なものを納品すること」が問題。自分は案件の提案文に「AI（Claude）を活用して作成し、自分で品質確認・修正した上で納品します」と書いてる。これで断られたことは1回もない。

つまずいたこと

最初の頃にやらかした失敗が2つある。

失敗1 — AI判定ツールに振り回された

「AI判定ツール」っていう、文章がAIで書かれたかチェックするサービスがある。最初はこれに通ることばかり気にして、文章を何度も書き直してた。

でも途中で気づいた。クライアントが見てるのは「AI判定ツールの結果」じゃなくて「納品物の品質」。ツールの判定をすり抜けても、中身が薄ければ修正依頼は来る。逆に、体験談が入ってて読みやすければ、AIで書いたかどうかなんて気にされない。

AI判定ツールを意識するのをやめてから、逆に修正依頼が減った。

失敗2 — チェック項目を増やしすぎた

品質を上げようとして、納品前のチェック項目を30個くらい作ったことがある。結果、チェックに1時間かかって時給が下がった。

今は上の3ステップ（10分）に絞ってる。チェック項目は少ない方がいい。「体験が入ってるか」「AI定型フレーズが残ってないか」「声に出して自然か」。この3つだけ確認すれば十分。

結局、差は「AI+自分」か「AIだけ」か

AI副業の単価が崩壊してるのは事実。でもそれは「AIの出力をそのまま納品する人」が増えたから。

自分がやってるのは「AIに80%作らせて、残り20%を自分で仕上げる」こと。この20%が体験談であり、文体調整であり、品質チェック。

たった20%の手間で、文字単価0.1円の世界と月14万の世界が分かれてる。非エンジニアでもできる。自分もコードは書けない。大事なのは「AIを使うこと」じゃなくて「自分の言葉に変えること」。

「AIで書いてるのにバレない」っていうのは、ズルしてるわけじゃない。AIを道具として使って、自分の品質基準で仕上げてるだけ。その工夫の全体像を、もう少し体系的にまとめたのがこっち。
