# 大学生がClaude CodeのFocus Viewを副業で1週間試したら、3案件並行でも頭が混乱しなくなった話

副業を3案件並行したら、画面の情報量で頭がパンクしていませんか？

先週、Claude Codeの画面を眺めながら案件Aのバグを追ってた途中で、別タブの案件Bから修正依頼が来て、そのままCに戻ろうとしたら「今どこ見てたんだっけ」ってなって10分止まった。

これが1日に3回くらい起きてた。ツール呼び出しのログ、中間の思考、ファイル差分、ストリームで流れてくるログ、全部見えてる状態。自分がどこを見るべきかわからなくて、頭の中で案件が混ざる。

結論から言うと、Claude Code v2.1.97（2026-04-07頃リリース）で入ったFocus View（Ctrl+O）を1週間使ったら、この問題がほぼ消えた。

**3案件並行時の切り替えロス 1日30分 → 5分**

## 何に困ってたか

副業で動いてる案件は大体2〜4本。それぞれにClaude Codeのセッションがある。画面を切り替えながら「この案件どこまで進んだっけ」を追う時間が、地味に積み上がっていた。

毎回3〜5分。1日6〜7回切り替えてたので、これだけで1日30分消えてた。時給換算したらちょうど1案件分の利益が溶けてる計算。

## Focus Viewって何ができるのか

Claude Codeのターミナル画面で`Ctrl+O`を押すと、表示が3段階で切り替わる仕組み。通常表示、Transcriptモード、そしてFocus View。

Focus Viewに切り替えると、画面に残るのは自分の最後のプロンプト・ツール実行の1行サマリー・Claude Codeの最終レスポンスの3つだけ。途中の思考ログとか、ツールが何を出したかの生ログは全部畳まれる。画面がスッと静かになる。

## 使い始めて何が変わったか

案件を切り替えるときにまずCtrl+Oを押すのが習慣になった。前のセッションから戻ってきたとき、画面には「自分が最後に頼んだこと」「Claudeが何を変えたか」「どう終わったか」の3つしか映らない。

1週間計測した感じだと、切り替えロスが1日30分→5分。週で約3時間戻ってきた。この3時間は、別案件1本受注に使える時間量。
