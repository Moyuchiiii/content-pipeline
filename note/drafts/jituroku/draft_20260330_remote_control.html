Claude Codeって、PCの前にいないと使えないと思ってた。

案件の途中でClaudeに「ここどうすれば？」って聞きたいとき、外出中はスマホじゃできなくて。電車に乗りながら「あの処理の続き、帰ってからじゃないと無理か」って諦めてた。

2月末にAnthropicがリリースしたRemote Control、最初は正直スルーしてた。でも先週ついに使い始めたら、週10時間あったPC前の"待機時間"が週3時間まで削れた。

移動中にスマホで承認できる。それだけなんだけど、これが思ったより大きかった。

Remote Controlで何が変わるか

一言で言うと「スマホのブラウザから、PCで動いてるClaude Codeのターミナルに接続する機能」。

Claude Codeって自律的に動いてくれるんだけど、ファイルを書き換えたり外部にアクセスしたりするたびに承認が必要で。その承認のたびにPCの前に戻らないといけなかった。

Remote Controlはその承認作業をスマホに転送する仕組み。Claudeが「ここ変更していいですか？」って聞いてきたら、スマホ画面にその通知が届く。OKを押せば作業が続く。

PC側はターミナルを開いたまま放置しておけばいい。

セットアップは1分で終わった

Proプランがあれば追加費用なし。PC側でこれだけ打てばいい。

claude remote-control

省略してclaude --rcでもいける。QRコードが表示されるので、スマホのカメラで読む。接続完了。

自分は初回、QRが読み込めなくて3回くらい試した。WiFiとモバイルデータを切り替えたら繋がった。ローカルじゃなくてAnthropicのサーバー経由で繋がる仕組みなので、ネット接続が必要。

1つだけ注意。PC側のターミナルは開いたままにしておく必要がある。ターミナルを閉じるとセッションが切れる。スリープは問題なかったけど、完全にシャットダウンするとアウト。

副業案件で1週間使ってみた記録

先週、クラウドワークスで受けた案件（Pythonスクリプト改修、¥28,000）で本格的に試した。

月曜日、通学中に承認を3回

授業前の30分、電車の中でスマホを眺めてた。普段なら帰るまで放置してた処理が、その場で「承認→次のステップ」できた。

電車でWi-Fiが切れた瞬間は焦ったけど、自動で再接続してくれた。ネットが戻ればそのまま続きから使える。

水曜日、図書館で勉強しながら承認

PCはカバンの中、スマホだけ机に出してた。Claude Codeがエラーを見つけて「こっちで直していいですか？」って来たので承認。5分後に修正完了の報告が来た。

以前だったら「図書館行ったから今日の作業は無理」で終わってた案件が、その日のうちに進んだ。

金曜日、バイト帰りの電車で最終確認

案件の仕上げ、バイトから帰りながらスマホで承認し続けた。家に着いたら、ほぼ完成状態になってた。納品前の最終チェックだけPCでやって、その日のうちに完了。

1週間のBefore/After

Remote Control導入前
PCの前にいた時間: 週10時間
うち「待機・承認」だけの時間: 4〜5時間
移動中・外出中の稼働: ほぼゼロ
Remote Control導入後（1週間）
PCの前にいた時間: 週3時間
スマホ承認（移動・外出中）: 2〜3時間
PC前での"待機だけ"の時間: ほぼゼロ

PCの前にいる時間が7時間減った。移動時間がそのまま副業の稼働時間になった感じ。

案件の完了速度も上がった。月曜に始めた案件を木曜に納品できたのは、移動中も止まらずに進んでたから。以前なら金〜土にずれ込んでたと思う。

つまずいたこと3つ

ターミナルを閉じてセッションが切れた

2回やらかした。PCをシャットダウンしたら全部切れた。

今はClaude Codeを起動したまま、シャットダウンはしない。スリープだけにしてる。「あ、PC使わないから電源切っとこう」という習慣があると詰まる。

スマホの通知を見逃す

Claudeが承認を求めてきても、スマホにプッシュ通知が来るわけじゃない。ブラウザを開いてる前提の仕組みなので、「後で見よう」と思ったら1時間放置してた、ってことが何度かあった。

今はSafariのタブをピン留めして、30分に1回は見る習慣にしてる。

ファイル送受信はPCでやるしかない

承認はスマホでできるけど、完成したファイルをクライアントに送る操作はPC必須。「移動中にほぼ全部終わったのに、添付ファイルを送るためだけに帰宅後にPC開く」みたいな状況が2回あった。

Remote Controlは「承認のリモコン」だと思っておくといい。ファイル操作の全部を外出先でやろうとすると詰まる。

Dispatchとの違い（よく混同されがち）

似たような機能に「Claude Dispatch」というのがある。自分も最初は同じものだと思ってた。

Remote Control
Claude Codeのターミナルセッションにスマホで接続する。コーダー向け。PC側でClaudeが動いてる前提。
Dispatch
Claude Desktop/Coworkにスマホからタスクを投げる。ノーコード向け。ターミナルとは別の仕組み。

コードを書いてる副業の人はRemote Control。ノーコードでタスクを投げたい人はDispatch、という分け方で合ってると思う。

1週間使ってわかったこと

「承認待ちでPCの前から動けない」という問題を解決する、今のところ一番シンプルな方法だった。

自分は週10時間副業に使ってたうち、4〜5時間が「ただ承認ボタンを押すためだけに画面を眺める時間」だった。これが移動中にできるようになっただけで、生活が結構変わった。授業の空き時間、通学、バイト帰り。そういう"隙間時間"が副業時間に変わった。

完璧な自動化じゃないし、まだスマホで承認できない操作もある。でも「PCの前にいないと副業が止まる」という状況から抜け出せたのは大きかった。
